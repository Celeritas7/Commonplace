6.1 — What an address is

Hi, I’m Bit. Pointers scare a lot of beginners — but only because nobody draws them a picture first. Let’s fix that. Try selecting any text below.

Memory is just a row of numbered boxes

Imagine your computer’s memory as a very long street of identical little boxes, all lined up. Each box can hold one small value, and — this is the key idea — every box has a number painted on the front. That number is its address. The boxes don’t move and the numbers never change while your program runs.

When you write int age = 30;, the computer quietly picks a free box, paints the number 30 inside it, and remembers “the box called age lives at address 4012.” You usually only care about the value, 30. But sometimes you want to talk about the box itself — where it lives — and that’s what a pointer is for.

📖 From your notes A variable name is a convenient label for a box. The address is the box’s real-world location. A program can hand around either one — the contents, or the location.

The address-of operator: &

To ask “which box does age live in?”, you put an ampersand in front of it. The expression &age reads aloud as “address of age.” It does not give you 30; it gives you something like 4012 — the location, not the contents.

// reading the value vs. reading the location
int age = 30;
printf("value:    %d\n", age);   // 30
printf("address:  %p\n", &age);  // e.g. 0x7ffd4012

This is the moment most tutorials go too fast. The single character & flips you from thinking about what’s inside the box to thinking about where the box is. Hold onto that distinction — it’s the whole game.

A pointer is a box that stores an address

So far we have boxes that store numbers. A pointer is simply a box whose painted-on contents are themselves an address — it stores the location of some other box. When we later “follow” that stored address to reach the real box, we call it dereferencing, and we’ll meet that in the next section.

That’s the entire mental model. A pointer points at another box by remembering its number. Everything else in this chapter is detail layered on top of this one picture.
