Overview

The N42's brain is the DME (Digitale Motor Elektronik) — specifically the MS32 variant for N42. It's the master controller that runs VANOS (variable valve timing), Valvetronic (variable valve lift), fuel injection, ignition, and closed-loop fuel control via the oxygen sensors. The page shows the standard ECU architecture (sensors → control unit → actuators), then drills into the VANOS control loop (cam sensor → DME → solenoid valve → phaser) and the dual-O₂ closed loop with both pre-cat and post-cat sensors.

DME identification — MS32 / K-coding / N42

Notebook box top right: "MS32 / DME (Digital motor electronic) / K / N42"

  • MS32 — Bosch hardware/software platform (Motronic Siemens 32).
  • DME — BMW's name for the ECU.
  • K — coding-platform letter (likely BMW's "K" series of vehicles or coding scheme).
  • N42 — engine code this DME is configured for.

A DME from another engine (e.g., N52) won't drop into an N42 — it'd have to be re-coded for the right engine map.

The DME uses the O₂ signal to trim fuel pulse-width on the fly to hold λ = 1. It also uses the same signal (along with crank position, MAP, HFM, etc.) to schedule VANOS and Valvetronic.

Bottom half — diagnostic loop with two sensors

Notebook (paraphrased): "Initially all λ = 1. So two 'Monitor O₂ sensors'. If [reading] = 1, it shows that this system is normal. It's not true — due to mixing of exhaust gases [post-cat] should sit at λ = 1. If λ > 1 reading error message is sent as λ > 1 → DME → 'Error is entered into the error memory.'"

Reading the diagnostic logic:

  1. In normal operation, the pre-cat O₂ sensor swings rapidly between rich and lean (DME deliberately oscillates fuel to keep cat in its working window).
  2. The post-cat O₂ sensor should sit steady at ~ λ = 1 because a healthy catalyst eats up the O₂ swing.
  3. If the post-cat sensor starts swinging like the pre-cat one → catalyst has failed (no more O₂ buffering).
  4. If the post-cat sensor sits at λ > 1 (high voltage / lean) → exhaust is being diluted by air leak after the cat, or the cat is dumping O₂ → fault.
  5. DME stores the error in error memory for OBD readout via DIS / OBS port (see diagnosis chapter).

Voltage convention (λ vs V)

Notebook: "λ = 1 → 1.5 V λ > 1 → > 1.5 V λ < 1 → < 1.5 V"

This convention is for a wide-band O₂ sensor (continuous output linear with λ). Narrow-band sensors switch around 0.45 V (low = lean, high = rich) instead.

Note: The notebook's "λ > 1 → > 1.5 V" reads the right direction for a wide-band sensor that outputs higher voltage for leaner mixtures, but the absolute number (1.5 V at λ = 1) is the trainer's round figure rather than a strict spec — real wide-band sensors are calibrated to output a linear current through an external resistor, and the converted voltage depends on resistor choice. See Open Questions.

Detailed Component Notes

"Initially all λ = 1"

At startup before closed-loop control activates, the DME runs from a base map — fuel quantity from rpm × HFM mass × stoichiometric ratio. Result is approximately λ = 1 right out of the box. Once the O₂ sensor heats up and goes live, DME switches into closed-loop trim and starts oscillating around λ = 1 to keep the catalyst in its working window.

Why monitor O₂ as a check on the cat

The catalyst doesn't just convert pollutants — it has oxygen storage built in. The Pt/Pd/Rh washcoat traps small amounts of O₂ during lean swings and releases it during rich swings, smoothing the post-cat O₂ signal. A healthy cat → flat post-cat signal. A worn cat → post-cat signal starts to look like pre-cat (swinging) → DME concludes cat has failed → DTC stored → MIL on.

Why DME also reads the crank position sensor here

VANOS phasing is meaningful only relative to crank position. The cam position sensor reads cam angle; the crank position sensor reads crank angle; the DME subtracts to know the actual cam-vs-crank phase and compares it to the commanded phase. This is the feedback that closes the VANOS control loop.

"Values set in it"

The DME stores the target VANOS phase angle as a 3D map indexed by engine rpm and load (and modulated by coolant temp, knock, etc.). At any operating point, the DME looks up the target → commands the solenoid → reads back actual phase → adjusts duty cycle.

Error message and memory

Notebook last line: "ERROR MSG as λ > 1 → DME → 'Error is entered into the error memory.'"

OBD-II workflow: - Fault detected (e.g., post-cat sensor reading shifted to lean). - DME assigns a DTC (Diagnostic Trouble Code, e.g., P0420 = catalyst efficiency below threshold). - DTC saved to non-volatile memory. - MIL ("check engine") light commanded on. - Tech reads the code with DIS / ISTA / OBD scanner (see diagnosis chapter).

Key Facts

  • N42 DME = MS32 variant; ECU is BMW-named DME (Digitale Motor Elektronik).
  • ECU architecture template: sensors → control unit → control outputs, plus switches, relay outputs, interfaces.
  • VANOS control loop: cam position sensor → DME → solenoid valve → phaser → cam position sensor (closed).
  • Solenoid valve is the actuator in VANOS; cam-position sensor is the feedback.
  • Wide-band O₂ pre-cat drives fuel closed loop.
  • Monitor O₂ post-cat is OBD diagnostic only — catches catalyst failures.
  • Voltage map (wide-band): λ = 1 → 1.5 V, λ > 1 → > 1.5 V (lean), λ < 1 → < 1.5 V (rich).
  • Healthy cat → post-cat sensor sits flat; faulty cat → post-cat sensor swings like pre-cat → DTC.
  • "Initially all λ = 1" — base map approximation before closed-loop activates.
  • Error memory stores DTCs; read via DIS / OBS scanner (see diagnosis chapter).
