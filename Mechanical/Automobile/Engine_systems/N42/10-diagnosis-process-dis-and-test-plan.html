N42 Engine

Diagnosis Process — DIS, OBS & Test Plan

Source images: IMG20250515003045, IMG20250515003057, IMG20250515003102

Notebook date: 6/8/2013

Overview

The dealer-tech fault-diagnosis workflow for a BMW (built around the N42 platform but generally applicable). Four ordered steps: collect symptoms → mechanical inspection → test drive → electrical diagnosis with BMW's DIS (Diagnosis Information System) via the OBS (On-Board Socket). Closes with the test-plan symbol legend that DIS uses on screen and tips on what each step is actually checking for.

Working Principle

The diagnostic mindset (header)

Notebook: "Diagnosis:- ★ Carry sufficient parts/components like fuses (decide as per diagnosis)."

The tech doesn't start blindly swapping parts. They run a structured 4-step process, narrowing the problem from "customer says car isn't right" to "this specific module/connector/sensor has failed." Only after diagnosis do you bring the replacement parts.

Step 1 — Collect symptoms

Notebook: "Steps: 1. Symptoms — The fault may not be always at the symptoms. - Collect the symptoms (by asking questions) - By driving car - Ask questions to customer"

The trainer's "fault may not be at the symptoms" line is the foundational diagnostic principle: the symptom is not the cause. A misfire might present as poor fuel economy. A failed crank-position sensor might present as a no-start. Listen to the customer, then drive the car yourself.

Step 2 — Mechanical inspection (visual + functional)

Notebook: "A) Visual inspection (look at the car from outside): - how car is standing? - any leakage? - opening bonnet (to detect any change in normal condition) - ignition started — check for any indication on dashboard - car from downside — check for bend or leakage B) Functional inspection: - ignition started — dashboard indication, brake lights, headlights, wipers, AC, radio, etc., every part"

Sub-step What's being checked
How car is standing Suspension sag, ride height (one corner low = spring or shock fail), tyre pressures.
Any leakage Coolant (green), oil (brown), brake fluid (yellow), power-steering fluid (red), AC condensate (clear — normal).
Open the bonnet Look for anything visibly disconnected, broken, leaking, or rubbing on a hot pipe.
Ignition on — dashboard All warning lights illuminate (bulb check)? Any that stay on after start = active DTC.
Car from underneath Bend in subframe (accident), leakage at oil pan / transmission / diff, loose exhaust mounts.
Functional check Every electrical consumer: lights, wipers, washers, horn, HVAC blower, AC, radio, windows, mirrors, etc.

Step 3 — Test drive

Notebook: "3. Test drive — Check for sound to diagnose. - Remove water bottle or chain which can be responsible for wrong result. - Also due to air leak there could be specific sound."

Drive the car, but first clean out the cabin — water bottles in cup-holders, jingling keys, chains hanging from the mirror — anything that rattles will mask the actual fault sound. A specific whistling or hissing during acceleration is often an air leak (intake or boost-pipe joint).

Step 4 — Electrical diagnosis with DIS / OBS

Notebook: "4. Electrical diagnosis: DIS — Diagnosis & Information System (BMW). - Previously this circuit was in bonnet. - OBS (On-board socket) is used to diagnose the car. It sends & receives through radio signals."

Four DIS functions:

Notebook: "1. TIS 2. Diagnosis 3. Measuring system (it can become voltmeter, ammeter) 4. Coding program (All on/off Control units are programmed, so need of coding)"

Function What it does
TIS (Technical Information System) Service manuals, wiring diagrams, torque specs — built-in reference library for every BMW.
Diagnosis Reads DTCs from every ECU on the car (engine DME, ABS, airbag SRS, transmission EGS, etc.), explains them in plain text.
Measuring system The DIS hardware doubles as a voltmeter / ammeter / oscilloscope for live signal checks (CAN bus, sensor voltages, switch states).
Coding program Re-codes ECUs after replacement; sets country-specific options; activates features.

Margin tags clarifying #4: - "prevention from failing" — coding ensures swapped modules don't generate fault codes from mis-match. - "vehicle based" — coding is tied to the specific VIN/chassis, not generic.

Symbol legend (IMG 003102)

Notebook: "✗ : fault in system ⊘ (or !) : No fault ⋯ : The program should be installed"

These symbols are how DIS displays each ECU's status in the diagnostic tree:

Symbol Meaning Action
✗ Fault detected in this ECU/subsystem Drill down to specific DTC; fix per service procedure.
! (or ⊘) No fault — system healthy No action.
⋯ DIS doesn't have the test program for this ECU/version Update DIS software / check version compatibility.

Components

Component Function
DIS (Diagnosis Information System) BMW's dealer diagnostic terminal. Plugs into OBS. Reads DTCs from all ECUs, runs test plans, codes modules, doubles as measuring instrument.
OBS (On-Board Socket) The OBD-II diagnostic port on the BMW. Sends & receives through radio signals (notebook wording) — really sends K-line / CAN signals from the car's diagnostic bus to the DIS handset; DIS itself may be wired or wireless to a workstation.
TIS (Technical Information System) Built-in service manuals on the DIS — wiring diagrams, torque specs, fault-trees, repair procedures.
Measuring System DIS hardware reconfigurable as voltmeter / ammeter / oscilloscope.
Coding program Writes vehicle-specific calibration to replaced ECUs (Valvetronic unit, DME, transmission ECU, etc.).
Error memory (in each ECU) Non-volatile DTC storage. Read by DIS, can be cleared after repair.

Detailed Component Notes

"Previously this circuit was in bonnet"

Pre-OBD-II, BMW (like most manufacturers) had a proprietary diagnostic socket under the bonnet (20-pin round connector on BMWs of the early '90s). With OBD-II mandate (US 1996, EU 2001), the standard 16-pin OBD-II socket moved to the driver's footwell area. Trainer's note marks the transition.

Why coding matters for swapped modules

A new Valvetronic ECU (or DME, or anything on the powertrain CAN bus) ships with generic default calibration. Plug it into a specific car and the DME will see: - Wrong vehicle option codes (e.g., manual vs auto trans). - Wrong adaptation values (idle, fuel trim, throttle position learn). - Wrong country-specific tuning (emissions calibration varies by region).

DIS pushes the vehicle-specific calibration into the new module so it integrates correctly. Without this step, even a brand-new module throws fault codes.

Measuring system

The DIS handset has banana-plug inputs for the leads. The tech can use it as: - DC/AC voltmeter (most common — check supply rails, sensor outputs). - Ammeter (parasitic-draw test). - Oscilloscope (look at injector pulses, ignition primary, crank-sensor waveform).

This avoids needing to bring a separate Fluke for routine work.

Why "DIS" specifically (vs ISTA, GT1, etc.)

Note that DIS is the dealer tool from 2002–2007 era — fits the 2013 training-notebook context loosely. Later replaced by GT1, then ISTA / ISTA-D (current). The diagnostic concept is the same; only the software has updated.

Key Facts

  • 4-step diagnosis workflow: symptoms → mechanical → test drive → electrical (DIS).
  • "Fault may not be at the symptoms" — always distinguish symptom from cause.
  • Mechanical step splits into Visual (look-only) and Functional (operate-every-consumer).
  • Test drive: remove cabin-noise sources first to hear the actual fault.
  • DIS = Diagnosis & Information System; OBS = On-Board Socket (BMW name for the OBD-II port).
  • DIS has 4 functions: TIS (service manuals), Diagnosis (DTC reading), Measuring System (volt/amp/scope), Coding program.
  • DIS status symbols: ✗ = fault, ! = OK, ⋯ = test program missing.
  • Diagnostic-port location moved from under bonnet (pre-OBD-II) to driver footwell (OBD-II era).
  • Coding is required after any module replacement to prevent fault codes.
