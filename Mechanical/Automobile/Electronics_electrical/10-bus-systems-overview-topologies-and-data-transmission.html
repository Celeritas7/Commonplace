Bus Systems — Overview, Topologies, Data Transmission

Source images: IMG20250515002630, IMG20250515002647, IMG20250515002653, IMG20250515002706, IMG20250515002715

Notebook date: 1/8/2013

Overview

A modern car has dozens of ECUs (engine, transmission, ABS, airbag, climate, audio, navigation, instrument cluster, etc.) that must share information. The old approach — running a dedicated wire from each sensor to each ECU that needs it — collapses under its own weight. Bus systems solve this: a single shared communication medium that every ECU can talk on. This page covers why bus systems exist in BMWs, the 6 advantages, the 3 fundamental topologies (linear, ring, star), the master-slave and multi-master arbitration, and the data-transmission types (parallel/serial, bidirectional, synchronous vs asynchronous K-bus).

Working Principle

Why bus systems

Notebook (IMG 002630): "Bus system – AC system, navigation system, telephone [communication system, active-passive system] — expected by customers. – How can these systems be introduced without changing the amount of space??? – Day by day size of electronic component is getting reduced. – DME (Digital motor electronic) is getting reduced. – Components increased so communication is required. – 'BUS SYSTEM' used for communication between components & control modules."

The bus motivation in one line: more features needed + less space available = a shared communication medium becomes essential.

BMW bus families (combined on this page)

Notebook: "It Combines: ① PT-CAN — Power-train network (fast, engine/transmission/ABS) ② K-CAN — Networking of general vehicle electronic modules (slower, body electrics) ③ byteflight — Specially developed for passive safety (airbag SRS) ④ MOST bus — Networking in information & communication areas (audio/video, fibre-optic)"

Margin notes (left side): - Terminator (just know what is in memory) - SGM = Safety & Gateway Module (the "bus translator") - Traffic management annotation - Convey messages / Changes the speed - LED

The four BMW buses, in order of speed and criticality:

Bus Speed Used for
byteflight 10 Mbit/s Safety-critical (airbag SRS) — failsafe, redundant
PT-CAN 500 kbit/s Powertrain — engine, transmission, ABS
MOST 25 Mbit/s Audio/video — fibre-optic (separate technology)
K-CAN 100 kbit/s Body — comfort modules, doors, windows
LIN 19.2 kbit/s Sub-bus — within a single function (e.g., HVAC stepper motors)

"Any control unit can communicate with another"

Notebook: "Any control unit can communicate with another. Mean data input from one sensor can be known by other control unit which is not connected to sensor."

The key benefit: a wheel-speed sensor physically wired only to the ABS module can have its reading read by the DME, EGS, instrument cluster, navigation, and dashboard — all without any additional wiring. Just a single CAN broadcast.

6 advantages of bus systems

Notebook (IMG 002647): "Advantages: ① Less wiring ② Lower cost ③ More functions ④ Less space ⑤ Great reliability ⑥ Efficient diagnosis"

# Why
1 Less wiring: one shared bus replaces dozens of point-to-point wires.
2 Lower cost: less copper, fewer connectors, fewer harnesses.
3 More functions: new features just need a new ECU on the bus, no new wires.
4 Less space: thinner harnesses fit through smaller body cavities.
5 Great reliability: fewer connectors = fewer failure points.
6 Efficient diagnosis: DIS reads every ECU on the bus from one OBD port.

Bus topologies (3)

Notebook (IMG 002647 lower): "Bus structures: ● Linear topology — nodes strung along one wire pair ● Ring topology — nodes connected in a loop ● Star topology — all nodes connected to a central hub. Margin: 'Announcement / Satellite (TV channels from satellite) / Master'"

Topology Layout Pros Cons
Linear Bus wire passes by each node Simple wiring, cheap Single break = network split
Ring Closed loop through every node Redundant (can route around break) Slower, complex
Star All nodes wire to a central hub Fast, easy diagnosis Hub failure = all dead

CAN uses linear topology. byteflight uses star topology (each node has its own wire to the gateway).

Master-Slave communication

Notebook (IMG 002647 bottom + IMG 002653 top): Diagram: "Master ←→ Slave / Slave" Margin: "right to send message"

A master-slave bus has one ECU in charge (master) that initiates all communication; slaves only speak when polled.

Notebook (IMG 002653): "Multi-master system: – There is possibility of more than one master. – So slave can become master & master can become [slave]. – At a time when line of bus system is busy for sending message (like mobile calling)."

Multi-master = any node can initiate a message at any time. Requires arbitration to resolve collisions (covered in CAN chapter).

Master-slave Multi-master
Simple — one master controls all Complex — needs arbitration
Predictable timing Better bandwidth utilisation
Used in: LIN sub-buses Used in: CAN main buses

Data-transmission types

Notebook (IMG 002706): "Parallel data transmission — High speed, many cables (e.g., 8-bit parallel uses 8 wires) Serial data transmission — Save no. of cables (one wire + shield for earthing) Bidirectional data transmission — Data can be sent & received in both directions"

Parallel Serial Bidirectional
Wires Many (8, 16, 32) One (or twisted pair) One pair, bi-directional
Speed Very high Lower per pin but adequate Variable
Cost High Low Low
Where used Inside chips, memory bus Vehicle buses (CAN, LIN, MOST) All modern vehicle buses

Modern vehicle buses are all serial + bidirectional.

Bus message structure

Notebook (IMG 002706 bottom): "Bus system message: – Data – Identifier – Check total"

A bus message is a structured packet: - Identifier — tells receivers what the message is about (priority + content type). - Data — the actual payload (sensor value, command, etc.). - Check total — CRC or checksum so receiver can detect corruption.

CAN messages typically: 11-bit identifier + 0-8 bytes data + 16-bit CRC.

Synchronous data transmission

Notebook (IMG 002715): "Synchronous data transmission: CCC clock signal with data 1001010"

In synchronous transmission, a separate clock line accompanies the data → receiver knows exactly when each bit starts/ends. Clean, but needs extra wire.

In asynchronous transmission, the receiver locks onto each transmission separately using start/stop bits → no extra clock wire needed (uses bus traffic to recover timing).

Margin: "No. of cycles per second" — i.e., the bit rate (or baud rate for raw symbol rate).

K-bus

Notebook: "K-bus: Asynchronous data transmission using single wire."

The BMW K-bus (Karosserie-Bus = body bus) is a slow asynchronous single-wire bus used by older BMW comfort modules — wipers, central locking, instrument cluster. Largely replaced by K-CAN in modern cars but still appears in vintage/older E-chassis.

Components

Component Function
Bus wire (twisted pair) Physical medium — typically 2 wires, twisted to reject electromagnetic noise
Bus terminator Resistor at each end of the bus to prevent signal reflections
Bus transceiver IC (in each ECU) Converts ECU's digital logic to bus-level differential signals and back
Bus controller (often in ECU's μP) Manages timing, arbitration, message ID matching
Gateway (SGM) Bridges two different buses (e.g., PT-CAN ↔ K-CAN) — translates messages
Identifier (in message) First field of each message — tells receivers what's being sent
Data payload The actual bytes of information being transmitted
Check total (CRC) Error-detection field
Arbitration logic Resolves collisions when two nodes try to send simultaneously

Detailed Component Notes

What "Terminator" means

Notebook (margin): "Terminator (just know what is in memory)"

A bus terminator is a 120 Ω resistor at each physical end of a CAN bus that absorbs the signal and prevents reflections. Without proper termination, signals bounce back and corrupt new messages → bus errors. Two terminators per bus (one at each physical end).

The trainer's "just know what is in memory" margin note seems unrelated — possibly a personal note about something else on the page.

Gateway module (SGM)

Notebook (margin): "SGM — Safety & Gateway Module"

The SGM is a multi-bus router — it sits on every bus simultaneously and forwards messages between them as needed. So an ECU on K-CAN can talk to an ECU on PT-CAN through the SGM's translation. This is essential because different buses run at different speeds and use slightly different protocols.

Why two wires (twisted pair) for CAN

Differential signalling: the bus uses two wires (CAN-H and CAN-L) that always have opposite voltages. Receivers measure the difference between them, which automatically cancels out any noise that affects both wires equally → CAN is extremely robust against electromagnetic interference even in the harsh under-bonnet environment.

"Like mobile calling" — the trainer's multi-master analogy

Notebook: "At a time when line of bus system is busy for sending message (like mobile calling)."

Cell phones: when you call someone whose line is busy, you get a busy tone → try again later. Same on a multi-master bus: if your node wants to send when the bus is occupied, you wait until it's clear. Modern CAN uses arbitration — the higher-priority message wins instantly, the lower waits.

Key Facts

  • Bus system = shared communication medium connecting many ECUs.
  • Motivation: more features + less space → can't run point-to-point wires anymore.
  • BMW bus families: PT-CAN (powertrain, 500 kbit/s), K-CAN (body, 100 kbit/s), byteflight (safety, 10 Mbit/s), MOST (audio/video, 25 Mbit/s, fibre-optic), LIN (sub-bus, 19.2 kbit/s).
  • 6 advantages: less wiring, lower cost, more functions, less space, reliable, easy diagnosis.
  • 3 topologies: linear (CAN), ring, star (byteflight).
  • Master-slave (one boss) vs multi-master (any can initiate, needs arbitration).
  • Message structure: identifier + data + check total (CRC).
  • Synchronous (with clock wire) vs asynchronous (start/stop bits).
  • K-bus = old BMW single-wire async body bus.
  • Differential signalling + twisted pair = noise immunity.
  • Bus terminator = 120 Ω resistor at each end of CAN bus.
  • SGM (Safety & Gateway Module) routes between different buses.
