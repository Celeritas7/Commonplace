Control Unit — Internal Modules

Source images: IMG20250515002622

Overview

What's inside an automotive ECU when you open the case: the microprocessor (arithmetic + decision-making brain), input module (reads sensors), output module (drives actuators), storage module (RAM/EEPROM for adapted values + DTCs), interface module (CAN/LIN/MOST bus drivers), special module (custom IC chips for application-specific tasks like ABS), field-effect transistors (the output power-switching MOSFETs), and voltage regulator (creates clean 5 V from messy 12 V battery). Together these eight building blocks make up every BMW DME, EGS, DSC, SRS, IHKA, or any other "K" / "control unit" in the car.

Module-by-module breakdown

Module What it does Implementation
Microprocessor The "brain" — runs the algorithm: reads inputs, decides outputs, manages timing. Specialised automotive μP (e.g., Motorola/NXP, Infineon TriCore) — 8/16/32-bit; runs at 10-200 MHz
Input module Receives signals from sensors → conditions them (filters noise, amplifies tiny signals) → converts to digital values via A/D converter Op-amps + RC filters + ADC chip
Output module Takes digital output commands → converts to current/voltage signals strong enough to drive actuators DAC + driver transistors (BJTs or MOSFETs)
Storage module Stores calibration data (maps for fuel, ignition, etc.), adaptation values (learned trims), diagnostic trouble codes (DTCs) in non-volatile memory EEPROM, flash memory + small SRAM working area
Interface module Drives the vehicle bus (CAN, LIN, MOST) for communication with other ECUs CAN transceiver, LIN slave/master IC, MOST transceiver
Special module Application-specific IC chips — for ABS, airbag triggering, immobiliser, etc. Custom ASICs or dedicated function chips
Field-effect transistors The actual high-current switches that drive injectors, coils, motors, solenoids Power MOSFETs (see chapter 08)
Voltage regulator Converts 12 V battery (which varies 10-14 V) into clean stable rails (5 V for logic, 3.3 V for newer chips) Linear regulator (7805) or switching regulator (buck converter)

Why each is separated as a module

On a real ECU PCB, these aren't always distinct chips — but they're functionally distinct so they're taught as separate modules. A modern automotive ECU might integrate microprocessor + memory + ADC + CAN transceiver onto one SoC (System-on-Chip) to save board space and cost.

Components

Component Function
Microprocessor (μP) Decision-making logic
A/D Converter (input module) Analog sensor → digital number
D/A Converter or PWM (output module) Digital command → analog/PWM output
EEPROM Non-volatile storage of calibration + DTCs
Flash memory Stores program code; reprogrammable for software updates
SRAM Working memory; volatile (lost when power off)

Power MOSFETs Output stage that drives injectors, coils, motors

Detailed Component Notes

Why "Storage module" is highlighted with a star

Notebook ★: "Stores different information, like 'fuel level is low'"

The trainer drew a star on this entry — likely because it's the module that holds: - DTCs (Diagnostic Trouble Codes) — read by DIS during diagnosis (see diagnosis chapter). - Adaptation values — learned by the system over time (idle trim, fuel trim, transmission shift adaptation, etc.). - Coding data — vehicle-specific calibration (country, options, equipment). - Mileage/service intervals — many critical figures.

If the storage module gets corrupted (battery disconnected too long, EEPROM aging), the ECU loses all this and reverts to default behaviour → noticeable change in feel + many warning lights → dealer reflash required.

Why "Special module"

Not every ECU needs every kind of processing power. Sometimes a specific function — like decoding an immobiliser transponder, running the ABS hydraulic solenoid sequence, or firing airbags within microseconds — is more efficiently done by a dedicated IC than a general-purpose microprocessor. The "special module" slot is where those custom chips live.

Examples in real cars: - ABS ECU: hardware sequencer for solenoid pulses. - SRS (airbag) ECU: dedicated accelerometer-to-firing-logic ASIC for safety. - Immobiliser: dedicated crypto chip that verifies the key's rolling code.

Why a separate voltage regulator

The car's electrical bus is noisy (alternator ripple, load switching, cranking sags, etc.) and varies 10–14 V. Logic chips need a clean stable supply (5 V or 3.3 V) → a regulator separates the noise.

The classic LM7805 linear regulator dissipates the excess voltage as heat — fine for low-current loads but wasteful at 5+ A. Modern ECUs use switching regulators (buck converters) that are 90%+ efficient even from a 14 V → 3.3 V conversion.

What microprocessor does BMW DME use

The dealer-training-era DME (MS32 for N42, MS43 for N42/N46, etc.) used Motorola/NXP 32-bit microprocessors (HC12/HCS12 family or PowerPC variants). Later DME (MSV70, MSD81 for N54/N55) moved to Infineon TriCore — a 32-bit dual-core with hardware-level safety features for automotive (ASIL-D rated).

Key Facts

  • 8 functional modules inside any automotive ECU: microprocessor, input, output, storage, interface, special, FETs, voltage regulator.
  • Microprocessor = decision-making brain; reads inputs, decides outputs.
  • Input module filters + ADC-converts sensor signals to digital.
  • Output module drives actuators (via FET power stage).
  • Storage module holds calibration, adaptations, DTCs in EEPROM/flash.
  • Interface module = CAN/LIN/MOST bus driver(s).
  • Special module = application-specific IC (ABS, SRS, immobiliser).
  • FETs = power-switching MOSFETs at the output stage.
  • Voltage regulator = converts 12 V battery noisy supply → 5 V clean logic supply.
  • Modern ECUs integrate multiple modules onto one System-on-Chip (SoC).
