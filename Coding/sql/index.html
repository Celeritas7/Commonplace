Code Sensei · SQL

Interactive SQL Book

Ten hands-on modules, real queries running in your browser. No installation needed — open any module to begin.

Modules

Module 1
Retrieving Data
Three foundational ways to pull data out of a SQL database. Every query you will ever write builds on these three skills.
Module 2
Filtering & Sorting
Narrow the rows you retrieve, decide what order they come back in, and cap how many you see. The three knobs that turn "fetch some data" into "answer a question".
Module 3
Aggregating Data
Stop thinking row-by-row. Start asking questions that collapse many rows into one — totals, averages, counts per group, filtered by what those totals look like.
Module 4
JOINs
Pull data from two or more tables in one query. JOINs are where SQL stops being a query language and starts feeling like working with a network of related ideas.
Module 5
Advanced SQL
The next layer of tools. Functions for the values inside cells, and the ability to nest queries inside queries. After this module you can write almost any real-world query.
Module 6
Creating & Modifying Tables
So far we have only read data. Now you become the one who builds the database — designing tables, putting data in, changing it, and evolving the schema as needs change.
Module 7
Conditional Expressions & Views
The polish. Tools that make queries clearer, safer with NULLs, and reusable. After this module your SQL stops being a sequence of queries and starts being a small library.
Module 8
Importing & Exporting Data
Getting data into and out of a database. CSVs you receive from elsewhere, exports for analysts who don't speak SQL, the routine data hygiene that comes with both.
Module 9
PostgreSQL with Python
SQL lives inside applications most of the time. This module wires PostgreSQL into Python — connecting, reading, writing, and the safety rails (parameters, transactions) that separate scripts that work from scripts that survive contact with real data.
Module 10
Capstone — Tiffinly
One extended exercise that ties everything together. Design a schema for a small tiffin delivery startup, load it with data, answer business questions across modules 1–8, wrap it with views and a Python operationalization layer. No new syntax — just integration.
