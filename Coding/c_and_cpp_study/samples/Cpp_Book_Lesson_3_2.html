switch — Multi-Way Branching

When you're branching on one value across many fixed options — especially an enum — switch is cleaner than a stack of else ifs. We'll cover fall-through, the C++17 [[fallthrough]] attribute, switching on enums, and switch-with-initializer.

1 Why this matters

One value, many destinations 🧠

Some decisions aren't "is this true?" but "which one of these is it?" A menu choice, a game state, a parsed token type. When you're matching a single value against a list of fixed possibilities, switch reads cleaner than a tall stack of else ifs — and in C++ it pairs beautifully with enums, where the compiler can even warn you if you forget a case.

You could write all of this with if / else if. switch doesn't add power — it adds clarity for one shape of decision: comparing one integer-like value against several constants. It carries C's classic quirk (execution falls through until a break), and C++ layers a few modern conveniences on top. Let's get the fundamentals, then the C++ extras.

2 Anatomy of a switch

The parts 🧩

switch (choice) {
    case 1:
        std::cout << "New game\n";
        break;
    case 2:
        std::cout << "Load game\n";
        break;
    default:
        std::cout << "Unknown option\n";
}
  • switch (choice) — C++ evaluates choice once, then jumps to the matching case.
  • case 1: — a label. Case values must be compile-time constant integers, chars, or enum values.
  • break; — exits the switch. Without it, execution falls through into the next case.
  • default: — the catch-all when no case matches. Optional, but usually wise.
Think of the case labels as entry doors, not separate boxes. C++ finds the matching door, walks in, and keeps walking downward through whatever follows — across other case labels — until a break stops it. That walking-downward behavior is the whole story of switch.
3 Fall-through and [[fallthrough]]

The break you can't forget 💥

🚨 Forgetting break = fall-through

Leave out a break and execution spills into the next case. Given choice = 1 with no breaks, all three lines run. This is the #1 switch bug — end every case with break;.

Stacking empty case labels to share code is intentional fall-through, and that's fine:

switch (key) {
    case 'a':
    case 'A':
        moveLeft();
        break;     // 'a' and 'A' share this
}
When you deliberately fall through after running some code, C++17 gives you a way to say so out loud: the [[fallthrough]] attribute. C++17 Place [[fallthrough]]; where a break would go, and the compiler knows you meant it — no "did you forget a break?" warning. It documents intent to both the compiler and the next reader. 
case warning:
    logWarning();
    [[fallthrough]];    // intentional — also do the error steps
case error:
    logError();
    break;
4 Switching on enums

Where switch really shines ✨

The most idiomatic use of switch in C++ is over an enum — a named set of states. The cases become self-documenting, and the compiler becomes your safety net:

enum class Light { Red, Yellow, Green };

switch (signal) {
    case Light::Red:    stop();  break;
    case Light::Yellow: slow();  break;
    case Light::Green:  go();    break;
}
The compiler safety net: if you switch over an enum and forget a case (say you add Light::Flashing later but don't handle it), compilers with -Wswitch (part of -Wall) will warn you about the missing case. That's a huge advantage over an if chain, which can silently ignore the new value. For this reason, many C++ programmers deliberately omit default when switching on an enum — so that adding a new enum value lights up every switch that needs updating.

One more modern touch: like if, switch supports an initializer in C++17 — switch (auto s = getState(); s) { ... } C++17 — scoping the value to the switch alone.

5 switch vs if (and what switch can't do)

Choosing the right tool 🤔

Use switch when…Use if / else if when…
Matching one value against fixed constants or enum valuesTesting ranges (score >= 90)
Branching on an enum state — get compiler warnings freeCompound conditions (x > 0 && y < 10)
The value is an int, char, or enumComparing doubles or… std::string

🚨 You can't switch on a std::string

Beginners coming from other languages often expect switch (name) { case "yes": ... } to work. It doesn't — switch only handles integer-like values, and a string isn't one. For string matching, use if (name == "yes") chains, or a std::map later on.

The mental model: a switch is a computed jump — C++ leaps straight to the matching label and runs forward. That's why every case must be a compile-time constant, why ranges and strings are out, and why fall-through exists at all. When your decision is "pick the matching value from a fixed list," switch is the clean choice.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

💥
Forgetting break Execution falls through into the next case. End each case body with break; — or mark deliberate fall-through with [[fallthrough]]; so the compiler (and readers) know it's intentional.
🔤
Trying to switch on a std::string case "yes": won't compile — switch needs integer-like values. Use if (s == "yes") chains or a std::map for string dispatch.
🧰
Declaring a variable in a case without braces case 1: int y = 5; can fail because the variable's scope crosses other labels. Wrap the case body in its own { } block when you need a local.

Practice exercises 💪

Easy Traffic light. Define enum class Light { Red, Yellow, Green }; and write a switch that prints the action for each. Omit default and turn on -Wall — confirm the compiler is happy when all cases are covered.
Medium Vowel grouping. Read a char and print "vowel" or "consonant" using stacked case labels (case 'a': case 'e': ...) for the vowels — intentional fall-through done right.
Stretch Add a case, catch the warning. Take your traffic-light switch (no default), then extend the enum with Flashing. Compile and watch -Wswitch flag the unhandled case — the compiler safety net in action. Then handle it.

🎓 What to remember

  • switch (value) jumps to the matching case and runs forward; break; exits.
  • Fall-through spills into the next case — accidental is a bug; deliberate gets a [[fallthrough]]; (C++17).
  • Switching on an enum is the sweet spot — -Wall warns about missing cases; often skip default on purpose.
  • Cases must be compile-time constants; you can't switch on a std::string or a range.
  • C++17 switch-init: switch (init; value) scopes a variable to the switch.
