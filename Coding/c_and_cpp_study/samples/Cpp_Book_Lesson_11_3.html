Stream Formatting & Manipulators

By default << prints values plainly. Manipulators are small objects you insert into the stream to control how output looks — decimal precision, field width, alignment, number base, how bool prints. Most are "sticky"; one famous one isn't. And C++20's std::format offers a tidier alternative.

1 Why this matters

Knobs you drop into the stream 🎛️

A bare std::cout << 3.14159 prints with default settings — but real output often needs to look a certain way: money to two decimals, columns lined up, a byte shown in hex. Manipulators let you say so. They're values you insert into a stream with <<; instead of printing text, they change the stream's formatting state. Some you already know — std::endl is a manipulator. The simple flag manipulators (std::fixed, std::hex, std::boolalpha) live in <iostream>; the ones that take an argument (std::setw(n), std::setprecision(n), std::setfill(c)) live in <iomanip>. They work on any stream — cout, a file stream, or a string stream (11.2).

The key mental model: a manipulator either toggles a persistent setting or tweaks the very next value. Knowing which is which is most of the battle.

2 Precision

setprecision & fixed for numbers 🔢

Floating-point output is controlled by two things together: a precision value and a mode. std::setprecision(n) sets the digit count; std::fixed / std::scientific changes what those digits mean:

#include <iomanip>

double pi = 3.14159265;

std::cout << pi;                                   // 3.14159   (default: 6 sig. figs)
std::cout << std::setprecision(3) << pi;            // 3.14      (3 SIGNIFICANT digits)
std::cout << std::fixed << std::setprecision(3) << pi; // 3.142   (3 digits AFTER the point)
Here's the subtlety that catches everyone. With the default float mode, setprecision(n) means "n significant digits in total" — so setprecision(3) on 3.14159 gives 3.14. Add std::fixed and the meaning flips: now setprecision(n) means "n digits after the decimal point," so the same call gives 3.142. For money and most tabular numbers you want std::fixed << std::setprecision(2). std::scientific switches to exponent form (3.142e+00), and std::defaultfloat restores the original behaviour.
3 Width & alignment

setw, setfill, left/right 📐

To line values up into columns, set a minimum field width and choose how to pad it:

std::cout << std::setw(8) << "Bo" << "|";        // "      Bo|"  (right-aligned, default)
std::cout << std::left << std::setw(8) << "Bo" << "|";  // "Bo      |"  (left-aligned)
std::cout << std::setfill('.') << std::setw(8) << 42;   // "......42"  (pad with dots)
std::setw(n) sets a minimum width for the next value: if it's shorter, the stream pads it to n characters (it never truncates a longer value). By default padding goes on the left, so values are right-aligned; std::left flips that, and std::right sets it back. std::setfill(c) changes the pad character from the default space. Together these are how you build aligned tables. But setw has a crucial quirk — it applies to one insertion only, which the next step (and the gotcha) makes precise.
4 Base & booleans

hex, oct, showbase, boolalpha 🔣

Integers can print in a different base, and bool can print as words instead of 1/0:

ManipulatorEffect (on value 255 / true)
std::hex255 → ff (base 16)
std::oct255 → 377 (base 8)
std::showbasewith hex: 255 → 0xff
std::boolalphatrue → true (instead of 1)
 
std::cout << std::hex << std::showbase << 255;   // 0xff
std::cout << std::dec;                            // back to base 10
std::cout << std::boolalpha << (1 < 2);           // true
std::hex, std::oct, and std::dec switch the integer base (these are sticky — once set, every following integer uses that base until you change it back). std::showbase adds the 0x/0 prefix, and std::uppercase makes hex digits capital (FF). std::boolalpha prints bools as true/false rather than 1/0 — much friendlier in logs — and std::noboolalpha turns it back off. Because these stick, remember to reset (e.g. std::dec) when you're done, or later integers will surprise you.
5 Sticky vs one-shot

The one rule to remember 📌

Almost every manipulator is sticky: it changes the stream and stays in effect until you change it again. The big exception is std::setw, which is one-shot — it applies to the next value only, then resets to 0:

std::cout << std::setw(5) << 42 << 7;   // "   427"
//                ^^^^^ width 5 → "   42", then width is gone → "7"

🚨 setw resets after one value; everything else sticks

Two failure modes flow from this. First, people expect setw to format a whole row, but you must repeat it before every column: cout << setw(8) << a << setw(8) << b;. Writing setw(8) once only pads a. Second — the mirror image — the sticky manipulators keep going even when you didn't intend it: set std::hex and forget, and every later integer prints in hex; set std::fixed and your unrelated numbers all gain trailing zeros. The habit that saves you: apply a sticky setting, do the work, then reset it (std::dec, std::defaultfloat, std::noboolalpha). C++20's std::format sidesteps the whole issue (next step).

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

📌
Expecting setw to stick setw applies to one insertion, then resets. To pad every column of a row, repeat std::setw(n) before each value — not just once at the start.
🔢
Forgetting std::fixed with money setprecision(2) alone means 2 significant digits (so 1234.5 → 1.2e+03). For two decimal places use std::fixed << std::setprecision(2).
♻️
Leaving a sticky flag on Set std::hex (or fixed, boolalpha) and every later value keeps that format. Reset when done: std::dec, std::defaultfloat, std::noboolalpha.

The modern alternative — std::format ✨

Since C++20, std::format (header <format>) replaces manipulator soup with a Python-style format string — no sticky state to leak: std::format("{:>8}", "Bo") right-aligns in width 8, std::format("{:.2f}", price) gives two decimals, std::format("{:#x}", 255) gives 0xff. It returns a std::string, so it pairs naturally with the file and string streams from 11.1–11.2. When your compiler supports it, prefer std::format for new code; you'll still read manipulators everywhere, so it's worth knowing both.

Practice exercises 💪

Easy Money format. Print 1234.5 as 1234.50 using std::fixed and std::setprecision(2). Then print 255 in hex with a 0x prefix.
Medium A lined-up table. Print three rows of name (left-aligned, width 10) and score (right-aligned, width 6) so the columns line up. Remember to repeat setw for each field.
Stretch Two ways. Format a small table once with stream manipulators and once with std::format (if your compiler supports C++20). Compare the readability, and note how std::format needs no resetting of sticky state.

🎓 What to remember

  • Manipulators are inserted with << to control formatting; argument ones (setw, setprecision, setfill) need <iomanip>.
  • Precision: setprecision(n) = significant digits by default; with std::fixed it's digits after the point (use fixed << setprecision(2) for money).
  • Alignment: setw(n) sets a minimum width, setfill(c) the pad char, left/right the side.
  • Base/bool: hex/oct/dec, showbase, boolalpha — all sticky, so reset them.
  • One rule: setw is one-shot; everything else sticks. And C++20 std::format is the cleaner modern path.
