Class Templates

Last lesson templated a function. You can template a whole class too — a blueprint for a type, parameterised by the type it holds. Write Box<T> once and you get Box<int>, Box<std::string>, and any other on demand. This is exactly how std::vector<T> and the rest of the STL containers are built.

1 Why this matters

A blueprint for a type 📦

A function template (9.1) generates a function per type. A class template does the same for a whole class — a stack, a pair, a container — parameterised by the type(s) it works with. Without it you'd write IntBox, StringBox, DoubleBox by hand, all identical but for the element type. With it you write Box<T> once, and the compiler stamps out a real class for each T you use. This is the single most important use of templates: every standard container — std::vector<T>, std::map<K,V>, std::unique_ptr<T> (you've been using these!) — is a class template.

Same generic idea as last lesson, now applied to data structures instead of single functions.

2 The syntax

template<typename T> class Box 📐

Put template<typename T> before the class, then use T for the member type, parameters, and return types:

template<typename T>
class Box {
    T value;                       // a member of the parameter type
public:
    Box(T v) : value(v) {}          // constructor takes a T
    T get() const { return value; }   // returns a T
    void set(T v) { value = v; }
};
T is the type parameter, exactly as with function templates. Inside the class, use it anywhere a concrete type would go — as a data member's type, a constructor parameter, a return type. The class doesn't commit to int or string; it's a recipe waiting for you to supply T. (You can have more than one parameter, like template<typename K, typename V> class Map — that's how std::map<K,V> is declared.)
3 Using it

You name the type 🏷️

The big difference from function templates: when you create an object, you usually state the type in angle brackets — there are no constructor arguments yet for the compiler to deduce T from at the declaration:

Box<int>    a(42);            // T = int
Box<std::string> b("hi");    // T = std::string
std::cout << a.get();           // 42
std::cout << b.get();           // hi
Box<int> and Box<std::string> are two different, fully separate classes the compiler generates from the one template — each with its own int or string member. The <int> is part of the type's name. (Since C++17, the compiler can often deduce a class template's argument from the constructor — Box b(42); infers Box<int> — called CTAD. But spelling out Box<int> always works and is clearer while you're learning.)
4 Defining members outside

The verbose-but-normal part 📝

If you define a member function outside the class body, the template syntax tags along — both on the function and on the class name:

template<typename T>
class Box {
    T value;
public:
    Box(T v);
    T get() const;
};

template<typename T>              // repeat the template line
T Box<T>::get() const { return value; }   // note Box<T>::
Each out-of-class definition needs its own template<typename T> line, and the class is referred to as Box<T> (with the parameter), not just Box. It's wordy, which is why small template classes are often written entirely inline inside the class body — and why, as the trap notes, the whole thing usually lives in a header. You don't have to memorise this; just recognise the shape when you see it.
5 It's how the STL is built

You already use class templates 🧰

Every container you've met is a class template — the angle brackets you've been writing are template arguments:

You wroteIt's a class template
std::vector<int>vector<T> with T = int
std::unique_ptr<Widget>unique_ptr<T> with T = Widget
std::map<std::string, int>map<K,V> with two parameters

🚨 Each instantiation is a separate type — and templates live in headers

Two things to keep straight. First, Box<int> and Box<double> are unrelated types — you can't assign one to the other, and a function taking Box<int> won't accept a Box<double> (to accept any, the function itself must be a template). Second, like function templates, a class template is normally defined entirely in a header, not split across .h/.cpp: the compiler needs the full definition wherever it instantiates the class, so a separate .cpp typically causes link errors. And a member function of a class template is only actually compiled when you use it for a given T — so an unused method with a T-specific bug can lurk until something calls it.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

🏷️
Forgetting the <T> when naming the type Box alone isn't a type — Box<int> is. Outside the class, write the class as Box<T> (e.g. Box<T>::get). (C++17 CTAD can deduce it at construction, but the type still has an argument.)
🔀
Mixing instantiations Box<int> and Box<double> are different, unrelated types — you can't assign or pass one as the other. A function that accepts any Box must itself be a template.
📄
Splitting the template across .h/.cpp The compiler needs the full template definition where it's used. Put class templates in headers, not a separate .cpp, or you'll hit link errors.

Practice exercises 💪

Easy Generic Box. Write template<typename T> class Box with a get/set. Create a Box<int> and a Box<std::string> from the one template and print each.
Medium A tiny Pair. Write template<typename A, typename B> class Pair holding a first and second of two different types. Make a Pair<std::string, int> (like a name and an age) and print both.
Stretch A Stack<T>. Build template<typename T> class Stack backed by a std::vector<T> with push, pop, and empty. Test it with a Stack<int> and a Stack<std::string> — one definition, two working stacks (Rule of Zero, thanks to the vector member).

🎓 What to remember

  • A class template (template<typename T> class Box) is a blueprint for a whole type, parameterised by T.
  • Name the type with its argument — Box<int> — when you create objects; the compiler generates a real class per type.
  • Each instantiation is a distinct, unrelated type — Box<int> ≠ Box<double>.
  • Out-of-class members repeat template<typename T> and use Box<T>::; keep templates in headers.
  • The whole STL is class templates — vector<T>, map<K,V>, unique_ptr<T> — you've been using them all along.
