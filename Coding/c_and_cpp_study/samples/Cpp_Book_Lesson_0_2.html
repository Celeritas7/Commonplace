What Computers Actually Do

Open up the hood and look inside. You'll meet the CPU, memory, and the simple loop that powers everything from web browsers to Mars rovers.

1 Why this matters

Lift the hood for a minute 🔧

In the last lesson you learned a program is a list of instructions a computer carries out. That raises the obvious next question: what does the computer actually do with those instructions?

You don't need to understand every transistor — that's electrical engineering. But you do need a cartoon picture of what's going on inside, because every C++ concept you'll meet later (variables, references, RAII, smart pointers) only makes sense once you have that picture.

Here's the whole thing in one sentence: a computer is a fast worker (the CPU) that reads tiny instructions from a wall of numbered boxes (memory), one at a time, forever. Let's unpack each piece.

2 Meet the CPU

The worker that does the work 👷

The CPU (Central Processing Unit) is the chip that actually runs your program. It can only do a small number of tiny things — but it does them extremely fast, billions of times per second.

3 Meet memory

A long row of numbered boxes 📦

Memory (or RAM, for "Random Access Memory") is where the CPU keeps everything it's currently working with. Picture a giant wall of tiny boxes, each one numbered from 0 upward. Each box holds a small value — a number, a letter, a piece of an instruction.

Two things make memory powerful, and they'll come up again and again as you learn C++:

  • Every box has an address — a number that uniquely identifies it. The CPU uses the address to say "fetch box #42" or "write to box #1000."
  • Access is "random" — the CPU can jump directly to any box without scanning the ones in between. That's where the "R" in RAM comes from.

When you write int x = 5; later, what you're really saying is "set aside a box in memory, label it x in my head, and put the number 5 inside it." Variables are just human-friendly names for memory boxes. This is also why C++ has references and pointers — they're tools to talk directly about those memory addresses.

One more thing worth knowing: memory is volatile. When you turn off the computer, everything in RAM is wiped. That's why files have to be saved to disk — disks (or SSDs) keep their contents even with the power off, but they're much slower than RAM. Modern computers shuttle data between the two constantly.

4 The loop that powers everything

The fetch-execute cycle 🔁

Here's what the CPU does, billions of times per second, from the moment you press the power button until you shut down:
  1. Fetch — go to memory, read the next instruction.
  2. Decode — figure out what kind of instruction it is (load? add? jump?).
  3. Execute — actually do it (load the value, add the numbers, jump to a new address).
  4. Repeat — go back to step 1.

That's it. Every program on your computer is just this loop, running fast enough to feel continuous. A 3 GHz CPU does this around 3 billion times per second. The reason a game feels smooth, or a video plays without skipping, or your typing appears instantly — all of it is that loop running far faster than you can perceive.

The CPU keeps a tiny number called the program counter that says "next instruction is at address X." After each instruction, the counter ticks forward by one — unless a jump instruction tells it to go somewhere else. That's literally how loops, function calls, and decisions work at the lowest level.

One reason C++ is famous for being fast: it compiles to machine code that maps very directly onto these CPU operations. There's no interpreter, no garbage collector, no virtual machine in between you and the chip.

7 Common mistakes

Three traps to avoid 🚧

🙅
"Memory and disk are the same thing." They're not. RAM is fast but loses everything when the power goes off. Disk (or SSD) is slower but keeps your files forever. They're two different physical things in your computer.
🙅
"The CPU is smart." It really isn't. It only does the handful of operations we listed — load, store, add, compare, jump. All the "intelligence" you see (web search, games, GPT) is millions of those simple steps stitched together by clever programmers.
🙅
"A variable is a label." Close, but more precisely: a variable is a name your program gives to a memory address. The CPU doesn't know about names — it only knows addresses. The C++ compiler is what converts your friendly names into the numeric addresses the CPU actually uses. This is exactly why C++ has tools like references and pointers — to talk about those addresses directly.

🎓 Quick recap

  • A computer has two main parts you care about right now: the CPU (the worker) and memory / RAM (the wall of numbered boxes).
  • The CPU only knows a handful of operations: load, store, add, compare, jump, repeat.
  • It runs the fetch–decode–execute loop billions of times per second.
  • Memory is addressable (every box has a number) and volatile (it wipes when the power goes off).
  • A variable in C++ is a friendly name your program gives to a memory address. The CPU only sees addresses; the compiler does the renaming.
  • C++ compiles directly to machine code that maps onto these CPU operations — no interpreter, no VM in between. That's where its speed comes from.
