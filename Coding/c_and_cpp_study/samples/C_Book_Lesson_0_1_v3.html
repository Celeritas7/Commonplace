What is Programming?
What is C?

Before you write a single line of C, let's get clear on what programming actually is — using the way you already explained it in your notes.

1 What is Programming?

Talking to a computer 💬

Computer Programming is a medium for us to communicate with computers. Just like we use "Hindi" or "English" to communicate with each other, programming is a way for us to deliver our instructions to the computer.

That's the one-sentence answer, straight from your notes. Let's unpack it a little.

When you and a friend talk in Hindi or English, your friend understands you because both of you agreed (long ago) on what each word means. A computer doesn't speak Hindi or English. It only "speaks" in extremely simple instructions — "add these two numbers," "store this value here," "jump to that step." A programming language like C is a shared agreement between you and the computer: you write in C, a tool translates it into the simple instructions the computer actually understands.

2 What is C?

The language you're about to learn

C is a programming language.
C is one of the oldest and finest programming languages.
C was developed by Dennis Ritchie at AT&T's Bell Labs, USA in 1972.

Three things to file away from that:

  • It's old — older than most software you use. That's not a problem; it's a sign that it works.
  • It's still everywhere — operating systems, embedded devices, game engines, databases. You'll see why in the next section.
  • It's small — the whole language fits in your head once you've learned it. Compare that to languages with hundreds of features bolted on.

The journey from your file to the CPU 🚀

Compilation is the process of translating high-level source code written in programming languages like C into machine code, which is the low-level code that a computer's CPU can execute directly. Machine code consists of binary instructions specific to a computer's architecture.

7 Common mistakes

Three traps to avoid 🚧

🙅
"The CPU understands C." It doesn't. As your notes say, the compiler translates C into machine code. That machine code — not your C — is what the CPU runs.
🙅
"C is outdated." It's old, but it's still inside the OS you're reading this on, the browser you're using, and probably your microwave. "Old" and "outdated" aren't the same thing.
🙅
"Machine code is the same on every computer." Your notes flag this: machine code is specific to a computer's architecture. A binary built for your laptop won't run on a Raspberry Pi without recompiling.

🎓 Quick recap

  • Programming is a way to deliver instructions to a computer, just like Hindi or English is a way to talk to a friend.
  • C is a programming language designed by Dennis Ritchie in 1972 — old, small, and still everywhere.
  • C is used in operating systems, drivers, embedded systems, and games — anywhere speed and direct hardware access matter.
  • You don't write machine code by hand. You write C, and the compiler translates it into machine code the CPU can run.
  • Machine code is architecture-specific — a binary for one chip doesn't necessarily run on another.
