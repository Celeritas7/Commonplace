🎯 PART VI · POINTERS

Pointers to Pointers & Function Pointers

Two more pointer ideas, briefly — both things you'll meet in real code even if you rarely write them. A pointer to a pointer (int **) is just an address of a pointer, and it's how a function can change your pointer. A function pointer holds the address of a function, so one variable can call different functions — the basis of callbacks.

1 Why this matters

Two pointer power-tools 🔌

You've mastered the everyday pointer. Two variations round out the toolkit. A pointer to a pointer (int **) sounds scary but is just "the address of a pointer variable" — and it's the standard way a function modifies a pointer the caller owns (or builds a list of strings, like the char **argv from command-line arguments). A function pointer stores the address of a function, letting you pass behaviour around as data — call different functions through one variable, register callbacks, build dispatch tables. This is a brief tour: enough to recognise both and use the common patterns.

Same rules you know (& takes an address, * follows one) — applied one level deeper, and to functions.

2 Pointer to a pointer

An address of an address 🪆

If int *p holds the address of an int, then int **pp holds the address of p. You follow it with * one extra time:

int  x  = 5;
int *p  = &x;     // p points to x
int **pp = &p;     // pp points to p

*pp        // → p          (the int*)
**pp       // → x's value  (5)  — dereference twice
Read the type right-to-left: int **pp is "a pointer to a pointer to int." Each * you write peels off one layer: *pp gives you back p (an int *), and **pp follows that to reach x itself. It's the same address-of / dereference dance you already know, just nested one level. You don't reach for this often — but two patterns make it essential.
3 Why you'd use it

Changing the caller's pointer 🔧

The classic use: a function that needs to modify the caller's pointer (not just what it points to) — for example, to allocate memory and hand the address back:

void make_array(int **out, int n) {
    *out = malloc(n * sizeof int);   // write through to the caller's pointer
}

int *arr = NULL;
make_array(&arr, 10);   // pass the ADDRESS of arr; now arr points to the memory
Remember from 6.6: to change a caller's variable, a function takes a pointer to it. So to change a caller's int *, the function takes a pointer to that pointer — an int **. Passing &arr lets make_array set arr for you. The other place you've already met ** is char *argv[] (i.e. char **) — an array of string pointers (Lesson 9.5). Same idea: a pointer that points at pointers.
4 Function pointers

A variable that holds a function ⚡

A function has an address too, and you can store it in a function pointer. The type spells out the signature; then you call through it like a normal function:

int add(int a, int b) { return a + b; }
int sub(int a, int b) { return a - b; }

int (*op)(int, int) = add;   // op points to add
op(2, 3);                       // → 5   (calls add)
op = sub;                       // now op points to sub
op(2, 3);                       // → -1  (calls sub)
The declaration int (*op)(int, int) reads "op is a pointer to a function taking two ints and returning int." The parentheses around *op are essential (more on that in the trap). Assign any matching function's name to it — a function name is its address, so no & needed — and then op(2, 3) calls whatever it currently points to. One variable, swappable behaviour.
5 Where function pointers shine

Callbacks & dispatch 🎛️

Function pointers let you pass behaviour as an argument. The standard library's qsort is the classic example — you hand it a comparison function:

int cmp(const void *a, const void *b) {
    return *(const int*)a - *(const int*)b;
}
qsort(arr, n, sizeof arr[0], cmp);   // pass cmp as a callback

🚨 The parentheses matter: int (*op)(...) vs int *op(...)

Declaration syntax is the tricky part. int (*op)(int, int) is a pointer to a function returning int. Drop the parentheses — int *op(int, int) — and you've instead declared a function named op that returns int *. Completely different! The (*name) grouping is what makes it a function pointer. Two more notes: function pointers are how you build dispatch tables (an array of function pointers indexed by a command), and how libraries take callbacks (qsort, event handlers, threads). When the syntax gets dense, a typedef helps: typedef int (*BinOp)(int, int); lets you write BinOp op = add;.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

📐
Dropping the parentheses in a function pointer int (*op)(int,int) is a function pointer; int *op(int,int) is a function returning int*. Keep the (*op) grouping — or use a typedef.
🪆
Wrong number of stars To change a caller's int * you need int ** and must pass &ptr. Passing the pointer by value lets the function change only what it points to, not the pointer itself.
⚠️
Mismatched function-pointer signature The pointer's parameter and return types must match the function exactly. Assigning a function with a different signature is an error (or undefined behaviour if forced).

Practice exercises 💪

Easy Double dereference. Set up int x, int *p = &x, int **pp = &p. Print **pp, then do **pp = 42; and confirm x changed to 42.
Medium A tiny calculator. Write add, sub, mul. Make an array int (*ops[])(int,int) = {add, sub, mul}; and call ops[i](a, b) based on a user's choice — a dispatch table.
Stretch Allocate through a pointer-to-pointer. Write void make_array(int **out, int n) that mallocs an int[n], fills it, and stores the address in *out. Call it with &arr, use the array, then free it. Bonus: sort it with qsort and your own comparator.

🎓 What to remember

  • int **pp is a pointer to a pointer — *pp is the int*, **pp is the int. Read types right-to-left.
  • Pass int ** when a function must change the caller's pointer (e.g. allocate and hand back the address); call it with &ptr.
  • A function pointer stores a function's address: int (*op)(int,int) = add; then op(2,3) — keep the (*op) parentheses.
  • Function pointers enable callbacks & dispatch tables — that's how qsort takes a comparator.
  • A typedef tames the syntax: typedef int (*BinOp)(int,int); → BinOp op = add;.
