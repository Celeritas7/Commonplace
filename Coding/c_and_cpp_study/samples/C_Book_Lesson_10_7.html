Capstone: A Word Counter (wc)

The last build, and a fitting one: a clone of the Unix classic wc, which counts lines, words, and characters. It reads from a file or straight from stdin, and the word count uses a neat little state machine — a few lines that capture a surprising amount of cleverness. Finish this and you've finished the C book.

1 What we're building

Lines, words, characters 🏁

wc ("word count") is one of the oldest, most-used Unix tools: give it text and it reports how many lines, words, and characters it contains. Ours will do the same, reading either from a filename on the command line or from stdin (so it works in a pipe). It's a perfect finale because it's tiny but touches the whole back half of the book: reading a stream character by character (Part IX), a while loop with a small state machine for words (Part III), argc/argv to pick the input (10), and stdin for pipes (10). Master this loop and you understand how a huge family of text tools works.

The clever bit is the word count — and it's only a few lines once you see the trick.

2 The counting idea

One pass, three counters 🔢

Read the input one character at a time. Characters and lines are easy; words take a tiny bit of state:

  • Characters: add 1 for every character read.
  • Lines: add 1 each time you see a '\n'.
  • Words: add 1 each time you cross from whitespace into a non-whitespace character.
The word rule is the insight. A "word" is a run of non-blank characters, so instead of trying to find where words end, you count where they begin — the moment you step from a space (or newline/tab) onto a real character. To detect that moment you remember whether you were already "inside a word" with a flag, in_word. That single boolean turns word-counting into a two-state machine: outside a word or inside one, with a count bumped on each outside→inside transition.
3 The state machine

in_word: the whole trick ⚙️

Here's the core loop. getchar returns each character (or EOF at the end); isspace (from <ctype.h>) tests for blanks:

#include <stdio.h>
#include <ctype.h>

void count(FILE *f, long *lines, long *words, long *chars) {
    int c, in_word = 0;
    while ((c = fgetc(f)) != EOF) {
        (*chars)++;                       // every char counts
        if (c == '\n') (*lines)++;        // newline ends a line
        if (isspace(c)) {
            in_word = 0;                  // we're between words now
        } else if (!in_word) {
            in_word = 1;                  // whitespace → word: a NEW word
            (*words)++;
        }
    }
}
Walk the logic: every character bumps chars; a '\n' bumps lines. For words, if the character is whitespace we set in_word = 0 (we're outside a word); otherwise, if we were not already in a word, we've just started one — set in_word = 1 and bump words. Crucially, once in_word is 1, more letters don't re-count — only the first letter of each run does. That's why multiple spaces between words, or leading spaces, never inflate the total. The counters are passed by pointer (Part VI) so count can update the caller's variables.
4 File or stdin

Pick the input 🔀

The same count function works on any FILE * — so main just chooses what to hand it: a file named on the command line, or stdin if none was given:

int main(int argc, char *argv[]) {
    long l = 0, w = 0, ch = 0;

    if (argc < 2) {
        count(stdin, &l, &w, &ch);          // no filename → read stdin (pipes!)
    } else {
        FILE *f = fopen(argv[1], "r");     // filename given → open it
        if (!f) { perror(argv[1]); return 1; }
        count(f, &l, &w, &ch);
        fclose(f);
    }
    printf("%ld %ld %ld\n", l, w, ch);     // lines words chars (like real wc)
}
This is the Unix-tool shape from 10.x: with no arguments, read stdin so the program slots into a pipe (cat big.txt | ./wc); with a filename, open that file. Because count takes a FILE *, both cases reuse the exact same logic — stdin is just another FILE *. Real wc loops over multiple filenames and prints a totals line; that's a natural extension once this works.
5 Edge cases & building

The fiddly bits 🧪

A few details decide whether your counts match real wc exactly:

Inputlines · words · chars
"hello world\n"1 · 2 · 12
"a b " (two spaces, trailing)0 · 2 · 5
"" (empty)0 · 0 · 0

🚨 Lines = newlines counted, so a last line with no '\n' isn't counted

The classic surprise: wc counts newline characters, not "lines of text." So "a\nb" reports 1 line (one '\n'), even though you'd say there are two lines — the final b has no newline after it. That matches real wc. Two more details: use int c (not char) to hold fgetc's result, because EOF doesn't fit in a char and you'd never detect end-of-file; and multiple/leading/trailing spaces never over-count words thanks to the in_word flag (whitespace just resets it). Build it with the Makefile habit from 10.3: cc -Wall -g wc.c -o wc, then test against the real wc to confirm your numbers agree.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

🔡
Using char c instead of int c fgetc/getchar return an int so they can return EOF. Store it in a char and the EOF test can fail — the loop may never end (or end early).
🔁
Counting every non-space as a word Without the in_word flag you'd count each letter, not each word. Bump words only on the whitespace→non-whitespace transition.
↩️
Expecting the last line without \n to count wc counts newline characters, so a final line with no trailing '\n' isn't counted as a line. That's correct behaviour — match it.

Build it & extend 💪

Easy Make it real. Put the code into wc.c, build with cc -Wall -g wc.c -o wc, and run ./wc < somefile.txt and echo "hi there" | ./wc. Compare your output to the system wc.
Medium Filename + flags. Accept a filename argument (open it, with a clean error if it won't open), and add -l/-w/-c flags that print only the requested count — parsing argv as in 10.5… er, 9.5.
Stretch Multiple files + totals. Loop over several filenames, printing each file's counts and a final total line, exactly like real wc. Run it under AddressSanitizer to confirm it's clean.

🎓 What to remember

  • Read once, char by char with fgetc in a while (… != EOF) loop — store the result in an int.
  • chars = every character; lines = each '\n'; words = each whitespace→non-whitespace transition.
  • The in_word flag is the whole word-counting trick — count where words begin, so extra spaces never over-count.
  • One count(FILE *, …) works for both a file and stdin — pick the source in main via argc/argv.
  • Match real wc's edge cases — lines are newline counts; a final line without '\n' isn't counted.
