📦 PART V · THE STL

The Algorithms

The STL ships dozens of ready-made, battle-tested functions — sort, find, count, transform, accumulate — that work on any container through the iterator range you met in 5.5. Stop writing the loop; call the algorithm.

1 Why this matters

Stop reinventing the loop 🧠

Sorting a list, finding a value, counting matches, summing numbers — you could write a hand-rolled loop for each. But these are solved problems. The <algorithm> library gives you correct, fast, named functions for them. Your code says what you want ("sort this"), not the mechanical how — and it's far less likely to hide an off-by-one bug.

This is where everything in Part V clicks together: containers hold your data, iterators describe ranges within them, lambdas (from 4.5) describe custom behaviour — and the algorithms tie all three together. Master a handful and you'll reach for them constantly.

2 They all take a range

(begin, end) — every time 🎯

Almost every algorithm takes a half-open range [begin, end) — the exact pair of iterators from 5.5. You pass v.begin(), v.end() to mean "the whole container," and they're declared in the <algorithm> header:

#include <algorithm>

std::vector<int> v = {5, 2, 8, 1};
std::sort(v.begin(), v.end());   // now {1, 2, 5, 8}
Because they speak iterators, the same algorithm works on a vector, a std::array, or a plain array — anything that provides begin/end. Passing a sub-range is just as easy: hand over different iterators (e.g. v.begin(), v.begin() + 3) and the algorithm only touches that slice.
3 The everyday three

sort, find, count 🧰

AlgorithmWhat it doesReturns
sort(b, e)orders elements ascending, in placenothing (void)
find(b, e, x)locates the first element equal to xan iterator (== e if absent)
count(b, e, x)how many elements equal xa number
 
auto it = std::find(v.begin(), v.end(), 8);
if (it != v.end())                   // found it
    std::cout << "found 8\n";

int n = std::count(v.begin(), v.end(), 2);  // how many 2s?
The big surprise is find: it returns an iterator, not a bool or an index. You check success by comparing to end() — exactly the "one past the last" marker from 5.5. If you need a position, subtract: it - v.begin().
4 Producing values

transform & accumulate ➗

Two more earn their keep daily. transform applies a function to every element and writes the results somewhere. accumulate (from <numeric>) folds a whole range down to a single value — a sum by default:

#include <numeric>   // accumulate lives here

int total = std::accumulate(v.begin(), v.end(), 0);  // sum, starting from 0

std::transform(v.begin(), v.end(), v.begin(),
               [](int x){ return x * 2; });   // double every element
accumulate's third argument is the starting value (use 0 for a sum, 1 for a product). transform's third argument is where to write the results — passing v.begin() again means "overwrite the original in place." Both lean on the lambda you pass to describe the per-element work.
5 Where lambdas pay off

The _if family ⚡

Many algorithms have an _if cousin that takes a lambda predicate — a little function returning true/false — so you decide the condition. And sort takes an optional comparator to order things your way:

// count elements greater than 4
int big = std::count_if(v.begin(), v.end(),
                        [](int x){ return x > 4; });

// sort descending with a custom comparator
std::sort(v.begin(), v.end(),
          [](int a, int b){ return a > b; });

🚨 Passing the container instead of iterators

Writing std::sort(v) does not compile in classic C++ — the algorithms want a pair of iterators, so it's std::sort(v.begin(), v.end()). (C++20 adds std::ranges::sort(v) that does take the container, but the iterator form is what you'll see everywhere and should learn first.)

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

🔎
Treating find like a bool or index std::find returns an iterator. Check success with it != v.end(), and get a position with it - v.begin() — don't expect true or a number directly.
📦
Passing the container, not the range std::sort(v) doesn't compile pre-C++20. Give it the iterator pair: std::sort(v.begin(), v.end()). (Or opt into std::ranges::sort(v) in C++20.)
📥
Forgetting the right header Most algorithms need #include <algorithm>; accumulate lives in <numeric>. Miss the include and you'll get a confusing "not declared" error.

Practice exercises 💪

Easy Sort and show. Put a few unsorted numbers in a std::vector<int>, call std::sort(v.begin(), v.end()), then print them with a range-for. Confirm they come out ascending.
Medium Count the evens. Use std::count_if with a lambda [](int x){ return x % 2 == 0; } to count how many elements are even. Print the count.
Stretch Product and squares. Use std::accumulate(v.begin(), v.end(), 1, [](int a, int b){ return a * b; }) to compute the product (note the 1 start value). Then use std::transform to build a new vector holding each element squared.

🎓 What to remember

  • Algorithms live in <algorithm> (and <numeric> for accumulate) and work on iterator ranges [begin, end).
  • sort orders in place; find returns an iterator (== end() if absent); count counts matches.
  • transform writes a computed value into each slot; accumulate folds a range to one value (3rd arg = start).
  • The _if versions take a lambda predicate — count_if, find_if — and sort takes a custom comparator.
  • Pass begin(), end(), not the container itself (until C++20 ranges).
