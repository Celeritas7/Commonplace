🎯 PART VI · POINTERS

Pointers as Parameters

C always passes arguments by value — a function gets its own copy, so it can't change your variables. The fix is the most important pointer idiom there is: pass a pointer to the variable, and the function can reach back and modify the original. It's how swap, scanf, and "return more than one value" all work.

1 Why this matters

Functions get copies 🧠

In C, when you pass a variable to a function, the function receives a copy of its value — this is "pass by value." Whatever the function does to that copy, your original is untouched. Usually that's exactly what you want. But sometimes you need a function to change the caller's variable: to swap two values, to hand back more than one result, or to fill in a value the caller is waiting for. Pointers make that possible.

You've seen the symptom before in 4.2 (value vs reference) and 5.6 (arrays getting modified). Now we'll make the idiom deliberate: pass &x, write through *p, and the function reaches your variable.

2 The problem

Pass by value can't reach back 📋

Here's a function that tries to change its argument — and fails, because it only ever touched a copy:

void tryAdd(int n) {   // n is a COPY of the caller's variable
    n = n + 10;          // changes the copy only
}

int main(void) {
    int x = 5;
    tryAdd(x);
    printf("%d\n", x);  // 5  — x never changed!
}
tryAdd bumped n to 15, but n was a private copy that vanished when the function returned. The caller's x was never in the room. To actually change x, the function needs its address, not its value.
3 The fix — pass a pointer

Hand over the address 🎯

Make the parameter a pointer, pass &x at the call, and write through the pointer with *. Now the function reaches the caller's variable directly:

void addTen(int *n) {   // n holds the ADDRESS of the caller's variable
    *n = *n + 10;         // write through it → changes the original
}

int main(void) {
    int x = 5;
    addTen(&x);           // pass the address of x
    printf("%d\n", x);  // 15  — x changed!
}
Two small things make it work, and both are easy to forget: the & at the call (give the address, not the value) and the * inside (follow the pointer to the real variable). This pattern — pass a pointer so a function can modify the caller's data — is often called pass by reference, simulated with pointers. C doesn't have true references (that's a C++ feature); pointers are how C does it.
4 The classic example

swap — impossible without pointers 🔄

Swapping two variables is the textbook case. With plain values it can't work (you'd only swap copies); with pointers it's clean:

void swap(int *a, int *b) {
    int tmp = *a;   // save the value a points to
    *a = *b;        // copy b's value into a's variable
    *b = tmp;       // put the saved value into b's variable
}

int x = 1, y = 2;
swap(&x, &y);       // now x == 2, y == 1
Because a and b point at the caller's x and y, the swaps land on the real variables. Try writing swap(int a, int b) with plain values and you'll see it do nothing outside the function — proof that this is exactly the problem pointers exist to solve.
5 Beyond modifying

Multiple results, and efficiency 📤

A function can only return one value — but with pointer parameters it can hand back several, by filling in variables the caller provides (called out-parameters):

void minmax(int a, int b, int *lo, int *hi) {
    *lo = (a < b) ? a : b;
    *hi = (a > b) ? a : b;
}

int lo, hi;
minmax(3, 8, &lo, &hi);   // lo = 3, hi = 8 — two results out
Pointers also let you pass big things cheaply: instead of copying a large struct into a function, you pass a pointer to it (just an address). If the function shouldn't change it, mark the parameter const — const struct Big *p — which says "I'll read through this pointer but never write." (More on const and pointers next lesson.)

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

📍
Forgetting the & at the call The function wants an address: call addTen(&x), not addTen(x). Pass the value and the types won't match (int vs int *) — the compiler will warn you.
⭐
Forgetting to dereference inside Writing n = 99; changes the local pointer, not the target. To change what it points to, dereference: *n = 99;.
🔣
*n++ vs (*n)++ Precedence trap: *n++ increments the pointer then dereferences. To increment the value it points to, use (*n)++.

Practice exercises 💪

Easy Add ten. Write void addTen(int *n) that adds 10 to *n. Call it as addTen(&x) and print x before and after — watch it change.
Medium Swap. Write void swap(int *a, int *b) and use it to exchange two variables. Print them before and after. Then try a plain-value version and confirm it does nothing.
Stretch Two results. Write void divmod(int a, int b, int *q, int *r) that stores the quotient in *q and the remainder in *r. Call it with &q, &r — a function that hands back two values at once.

🎓 What to remember

  • C passes by value — functions get copies, so they can't change your variables directly.
  • Pass a pointer (&x) and write through it (*n = …) to let a function modify the caller's variable.
  • This "pass by reference" idiom powers swap, out-parameters (multiple results), and scanf's &.
  • Remember both halves: the & at the call and the * inside — each is easy to forget.
  • Pass a pointer (often const) to hand big data to a function cheaply, without copying it.
