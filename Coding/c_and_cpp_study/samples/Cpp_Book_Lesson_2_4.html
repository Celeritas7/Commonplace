Input and Output — cin, cout, getline

Your programs need to talk to the user. C++ gives you streams: std::cout for output (you've used it), std::cin for input, and std::getline for reading whole lines including spaces. Today: meet all three and the famous whitespace gotcha.

1 Why this matters

From printing to reading 🗣️

Up to now, every program you've written has only spoken to the user — std::cout prints, the user reads. To make programs that actually respond to input — a calculator, a greeting that knows the user's name, a quiz — you need to read what the user types. That's std::cin's job.

The C++ stream model is symmetric: same idea, opposite arrows. cout << x sends x into the output stream. cin >> x pulls a value out of the input stream and into x. Once you see that pattern, the rest is just learning the gotchas.

2 std::cout — a quick recap + manipulators

Output, with a few new tricks ✏️

You've been using std::cout since lesson 1.3. The new pieces today: manipulators like std::endl (newline + flush) and "\n" (just newline), and the fact that you can chain dozens of << calls.
std::cout << "Hello" << ", " << "world!" << "\n"; std::cout << "Age: " << age << "\n"; std::cout << "Pi ≈ " << 3.14159 << std::endl; // "\n" vs std::endl: // "\n" just inserts a newline character. // std::endl inserts a newline AND flushes the buffer (slower). // Use "\n" by default. Use std::endl only when you need immediate output.

One common modern convention: stick with "\n" instead of std::endl unless you need an immediate flush (like before a prompt where you want the user to see something right now). The performance difference is real in tight loops.

3 std::cin and the >> operator

Reading values from the user 👂

The >> operator (called extraction) pulls one token at a time from the input stream into your variable. The compiler figures out the type — if you extract into an int, it parses an integer; into a double, a decimal; into a std::string, a single word.
#include <iostream> #include <string> int main() { int age; std::cout << "Enter your age: "; std::cin >> age; // reads an int from input std::string name; std::cout << "Enter your name: "; std::cin >> name; // reads ONE WORD (stops at whitespace!) std::cout << "Hi " << name << ", you are " << age << "\n"; }

You can also chain >> like <<: std::cin >> a >> b >> c; reads three values, separated by any amount of whitespace (spaces, tabs, newlines — all the same to >>).

The whitespace trap

std::cin >> name reads one whitespace-separated token. If the user types "Aniket Mangaonkar", only "Aniket" goes into name; "Mangaonkar" sits in the input buffer waiting for the next read.

To read a whole line including spaces, you need std::getline — coming up next.

4 std::getline — for whole lines

When the input has spaces in it 📄

For names, addresses, sentences — anything that may contain spaces — use std::getline(std::cin, your_string). It reads everything up to the next newline (the user pressing Enter), and stores it in your std::string.
#include <iostream> #include <string> int main() { std::string fullName; std::cout << "Enter your full name: "; std::getline(std::cin, fullName); // reads everything including spaces std::cout << "Hi " << fullName << "\n"; }

Note the syntax difference: getline isn't a method on cin — it's a free function that takes cin as its first argument. Slightly awkward, but it's how C++ does it.

The mixed gotcha: if you do cin >> age; and then getline(cin, name); right after, name will be empty! That's because >> leaves the newline (the Enter key) sitting in the input buffer, and getline immediately reads up to that newline — which is right there, so it reads nothing. Fix: call std::cin.ignore(); in between, or read everything with getline and parse numbers separately. We'll see the cure in the simulator.

7 Common mistakes

Three classic I/O traps 🚧

🙅
"cin >> name reads a whole line." It doesn't. >> reads one whitespace-separated token. For whole lines, use std::getline(std::cin, name). This trips up every C++ beginner once — usually when they try to ask for a person's full name.
🙅
"I can mix cin >> and getline freely." You can — but only if you flush the leftover newline first. After cin >> age;, there's still a '\n' sitting in the buffer from when the user pressed Enter. A direct getline after will read that newline as an empty line. Fix: std::cin.ignore(std::numeric_limits<std::streamsize>::max(), '\n'); between them. Or just use getline for everything and parse the strings yourself.
🙅
"What happens if the user types abc when I asked for an int?" cin goes into a failure state. The variable stays at whatever value it had before. Any subsequent cin >> calls also fail until you clear the error: std::cin.clear() then std::cin.ignore(...). For robust input, you check if (std::cin >> x) { ... } — the stream evaluates to true on success, false on failure.

🎓 Quick recap

  • std::cout << x outputs. std::cin >> x inputs. Same idea, opposite arrows.
  • cin >> x reads one whitespace-separated token. For ints/doubles, it parses; for strings, it reads one word.
  • std::getline(std::cin, s) reads a whole line into s, including spaces, stopping at newline.
  • Use "\n" instead of std::endl unless you need an immediate buffer flush.
  • Mixing cin >> with getline? Insert std::cin.ignore(...) between them to flush the leftover newline.
  • On bad input (e.g., user types "abc" for an int), cin enters a failure state. Check with if (std::cin >> x) for robust input.
