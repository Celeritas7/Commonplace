What is a Program?
What is C++?

Before we write a single line of C++, let's get clear on what we're actually doing when we "write a program" — and what makes C++ a particularly interesting language to do it in.

1 What is a program?

Teaching a machine to follow steps 💬

A program is just a list of instructions for a computer to carry out, one after another. The instructions are extremely simple on their own — "add these two numbers," "compare these values," "store this here," "jump to step 12." Strung together by the billions, they become a web browser, a game, an operating system.

You don't write those tiny low-level instructions by hand. That would take forever and you'd lose your mind. Instead, you write in a programming language — a more human-friendly form — and a tool translates it down to what the machine actually runs. C++ is one of those human-friendly forms.

2 What is C++?

The language with two faces

A C++ program is a collection of commands or statements.

That one-liner is in your book — and it's true, but it undersells C++. Let me unpack what makes C++ special:

C++ was created by Bjarne Stroustrup at Bell Labs in 1979. He started with C (which was already popular) and added the ability to model real-world things as objects with their own data and behavior. Over the decades, the language has evolved through standards labeled by year — C++11, C++14, C++17, C++20, C++23 — gaining features like lambda functions, smart pointers, ranges, and concepts.

The "two faces" of C++:
• High-level: classes, templates, the Standard Library — write expressive, readable code.
• Low-level: direct memory access, manual control over how things are laid out — squeeze every cycle out of the hardware.

Almost no other mainstream language gives you both.

4 Where C++ sits among languages

The same task in three languages

Here's "add two numbers and print the result" in three popular languages. The point isn't the code itself — you don't need to read it. The point is to see how C++ looks in context.
Python INTERPRETED
a = 3
b = 4
print(a + b)
C++ COMPILED
#include <iostream>
int main() {
  int a = 3, b = 4;
  std::cout << a + b;
}
Java COMPILED→VM
class M {
  public static void main(String[] a) {
    System.out.println(3 + 4);
  }
}

Python runs through an interpreter — it reads each line and executes it on the spot. Java compiles to a portable middle-format that runs on a virtual machine. C++ compiles all the way down to native machine code for your specific CPU. That's why C++ tends to be fastest — and why a C++ program built on Windows won't run on a Mac without being recompiled.

8 Common mistakes

Three traps to avoid 🚧

🙅
"The computer understands C++." It doesn't. The compiler turns your C++ into native machine code; the CPU runs that machine code. C++ is for you, not for the chip.
🙅
"C++ is just C with classes added on." It started that way in 1985 but modern C++ has its own personality — templates, RAII, the STL, lambdas, move semantics. We'll lean into that modern style from day one.
🙅
"If it compiles, it's correct." A compiler only checks that your code is grammatically valid. It can't check that it does what you meant. We'll meet bugs that compile happily and crash on the first run.

Stretch Spot the abstraction. A C++ compiler is itself a program — often written in C++. How is that possible? You don't need the full answer yet — just sit with the puzzle for a minute. (The trick has a name: bootstrapping.)

🎓 Quick recap

  • A program is a list of instructions a computer carries out.
  • A programming language like C++ is a way for humans to write those instructions clearly.
  • C++ was created by Bjarne Stroustrup in 1979 — a high-level and low-level language at the same time.
  • The CPU does not read C++. The compiler translates your C++ source into native machine code, and only that runs.
  • C++ powers browsers, games, trading systems, robotics — places where performance and control matter.
