Where to Go Next

You've reached the last page. From "what is a program?" to a JSON parser and an expression evaluator, you've learned a real, modern slice of C++. This finale looks back at what you can now do, points to the big areas still ahead — concurrency, networking, build systems, the wider STL — and leaves you with habits and resources to keep growing.

1 Look how far you've come

The whole journey 🎉

Take a moment. You started not knowing what a program was, and you now understand variables, control flow, and functions; arrays, strings, and the STL containers; object-oriented programming with classes, inheritance, and polymorphism; templates and concepts; modern resource management with RAII and smart pointers; the full error-handling toolkit — exceptions, noexcept, and std::expected; and file and stream I/O in every flavour. Then you put it together into three working programs: a persistent to-do app, a JSON-like parser, and an expression evaluator. That's not a beginner's checklist any more — that's a foundation you can build a career on.

Real fluency comes from using this, so the rest of this page is about momentum: what to learn next, what habits to keep, and where to find help.

2 What you can do now

Your toolbox 🧰

It's worth naming the capabilities explicitly — these are things you can sit down and write today:

  • Model a problem with your own types (struct/class), collections (std::vector, std::map), and clean interfaces.
  • Manage resources safely with RAII and smart pointers — no manual new/delete, no leaks.
  • Handle failure deliberately: throw and catch exceptions, or return std::optional/std::expected when failure is routine.
  • Read and write data — files, strings, formatted output, even raw binary.
  • Write reusable code with templates and constrain it with concepts.
  • Build a small program end-to-end, from data model to a command loop with persistence.
Notice the through-line: nearly all of this leans on a few big ideas applied over and over — let types own their resources, make illegal states hard to represent, and lean on the standard library instead of hand-rolling. Those instincts matter more than memorising syntax, and you've been building them the whole way.

4 Habits worth keeping

What good C++ programmers do 🌱

♻️
Let RAII own everything Tie every resource — memory, files, locks, sockets — to an object's lifetime. If you're typing delete, ask whether a smart pointer or container should do it instead.
🧱
Prefer the standard library Before writing a loop, check <algorithm>. Before a raw array, reach for std::vector. The STL is faster, safer, and clearer than most hand-rolled code.
🚦
Turn on warnings & sanitizers Compile with -Wall -Wextra and treat warnings as bugs. Run with -fsanitize=address,undefined to catch memory and UB errors the compiler can't see (recall Part X / 10.5).
📐
Be const-correct & precise Mark things const by default, pass big objects by const&, name things clearly, and keep functions small. Future-you (and your reviewers) will thank you.

None of these are new — they're the habits this book has been quietly modelling. Keeping them is what separates code that merely works from code that lasts.

5 How to keep learning

Momentum > cramming 🚀

The single best thing you can do now is build things. Reading teaches you what's possible; writing teaches you what's true. A few ways to keep the gradient steep:

  • Pick a real project you actually want — a game, a tool, a small server — and let its needs pull you into new topics.
  • Read good code. Browse a well-regarded open-source C++ library and notice how it's structured.
  • Use the reference. Keep cppreference open while you code; look things up rather than guessing.
  • Experiment cheaply on Compiler Explorer — try a snippet, see the assembly, compare compilers.
  • Revisit this book. Concepts you skimmed will click differently once you've hit the problems they solve.
And be patient with yourself. C++ is famously large, and nobody holds all of it in their head — experienced engineers look things up constantly. You don't need to know everything; you need to know enough to start, and the judgement to find the rest. You already have both. The interactive's Resources tab collects the handful of places worth bookmarking.

7 Quick check

One last question 🎯

Across this whole book, one idea underpins safe C++ more than any other: tie each resource (memory, files, locks) to an object's lifetime so it's released automatically. What's it called?
Hint: it's the idea behind smart pointers, std::vector, and file streams alike.
