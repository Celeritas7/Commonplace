Struct Pointers & the -> Operator

In 8.1 you saw that passing a struct copies the whole thing. The fix — and the key to dynamic data — is a pointer to a struct. To reach a member through it you'd write (*p).x… but C gives you a cleaner shorthand: the arrow operator, p->x. It's the tool behind every linked structure you'll build.

1 Why this matters

Point at a struct, don't copy it 🧠

Back in 8.1 you hit a wall: passing a struct to a function copies it, so the function can't change your original — and copying a big struct is wasteful. The answer is the same one you learned for everything else in Part VI: take a pointer to it. A pointer lets a function reach into and modify the caller's struct, avoids copying large records, and — crucially — lets structs link to each other (a node pointing to the next node). The one new piece of syntax you need is how to reach a member through a pointer. That's the arrow operator.

This lesson is the bridge from "a struct" to "a data structure" (next, the linked list).

2 A pointer to a struct

Same idea, bigger target 👉

A pointer to a struct works like any other pointer — & to take its address, * to dereference. To reach a member you dereference first, then use the dot:

struct Point pt = {3, 4};
struct Point *p = &pt;     // p points at pt

(*p).x          // reach member x THROUGH the pointer → 3
// the parentheses are REQUIRED:
*p.x            // ✗ this is *(p.x) — wrong, and won't compile
The parentheses matter because the dot binds tighter than *: *p.x means *(p.x), which tries to use p as a struct (it isn't — it's a pointer). You have to write (*p).x: "dereference p to get the struct, then take its x." Correct, but clunky to type all day — which is exactly why the arrow exists.
3 The arrow operator

p->x means (*p).x ➡️

The arrow operator -> does the dereference-and-dot in one clean step. These two lines are exactly the same:

struct Point *p = &pt;

(*p).x = 10;     // the long way
p->x = 10;       // the same thing, much nicer ✨

printf("%d %d\n", p->x, p->y);   // read members through the pointer
Whenever you have a pointer to a struct, use -> to get at its members; when you have the struct directly (a value), use the dot. That's the whole rule: value uses ., pointer uses ->. The arrow is overwhelmingly the form you'll see, because real programs pass structs around by pointer.
4 Modifying through a pointer

Reach the real struct ✏️

This is the fix promised in 8.1. Pass a pointer to a struct and the function edits your original, not a copy:

void move(struct Point *p, int dx, int dy) {
    p->x += dx;     // writes through to the caller's struct
    p->y += dy;
}

struct Point pt = {1, 1};
move(&pt, 5, 3);          // pass its address
// pt is now {6, 4} — the original really changed ✅
Compare this with 8.1: there, bump(pt) took a copy and your original stayed put. Here, move(&pt, …) passes the address, so p->x += dx reaches all the way back to pt. Passing by pointer is the standard way C functions modify a struct — and, as a bonus, it avoids copying the whole record on each call.
5 Structs on the heap

malloc a struct 🧱

You can also allocate a struct dynamically. malloc hands back a pointer, so you use -> throughout — and free it when done:

struct Point *p = malloc(sizeof *p);   // room for one Point; sizeof *p = sizeof(struct Point)
if (!p) return;          // always check malloc
p->x = 3;
p->y = 4;
// ... use it ...
free(p);                  // give it back when finished

🚨 The pointer must point somewhere valid first

Writing p->x = 3 only works if p already points at real storage — a struct you took the address of, or a successful malloc. On an uninitialized or NULL pointer it's a crash (recall 6.8: wild and null pointers). And after free(p), using p->x is a use-after-free. The arrow is just access — it doesn't create the thing being pointed at.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

🔣
Writing *p.x for (*p).x The dot binds tighter than *, so *p.x parses as *(p.x). Use p->x (or parenthesize: (*p).x).
💥
Arrowing through a bad pointer p->x = 3 crashes if p is NULL or uninitialized. Point it at a real struct (with &) or a successful malloc first.
🧹
Mixing up . and -> Use . when you have the struct value, -> when you have a pointer to it. Using the wrong one is a common compile error.

Practice exercises 💪

Easy Two ways in. Make a struct Point pt = {3, 4}; and a pointer p = &pt;. Print p->x and (*p).y and confirm they read 3 and 4.
Medium Modify in place. Write void scale(struct Point *p, int k) that multiplies both members by k using ->. Call it with &pt and confirm the original pt changed.
Stretch Heap it. malloc a single struct Point, fill it via ->, print it, and free it. Run it under a sanitizer (7.6) to confirm there's no leak. This is exactly the pattern the linked list will use next.

🎓 What to remember

  • A pointer to a struct lets you modify the original and avoid copying — and link structs together.
  • p->x is shorthand for (*p).x — dereference, then member. The arrow does both.
  • Value uses ., pointer uses -> — that's the whole rule.
  • Pass struct T * to a function to change the caller's struct (the fix from 8.1).
  • The pointer must be valid first — a real address or a successful malloc; mind NULL and use-after-free.
