Multi-dimensional Arrays

An array of arrays — a grid. Where a 1D array is a row of boxes, a 2D array is a whole table: rows and columns, reached with two indices, m[row][col]. It's how C holds matrices, game boards, and spreadsheets.

1 Why this matters

Some data is a table 🧠

A single array is a line of values. But lots of data is naturally two-dimensional: a tic-tac-toe board (3×3), a chess board (8×8), a month's calendar, a grid of pixels, a matrix of numbers. For these you want rows and columns — and that's a 2D array, written int m[rows][cols] and indexed with two numbers.

You already have everything you need: a 2D array is just an "array of arrays," and a pair of nested loops (from lesson 3.6) walks every cell. This lesson adds the two-index syntax, how to lay out a grid in memory, and the one rule you must remember when passing a 2D array to a function.

2 Declaring & accessing

Two sizes, two indices 🔲

Declare a 2D array with two bracketed sizes — rows then columns — and reach a single cell with two indices, again row first, column second:

int m[3][4];          // 3 rows, 4 columns → 12 ints

m[0][0] = 7;        // top-left corner
m[2][3] = 9;          // bottom-right (last row, last column)
printf("%d\n", m[1][2]); // row 1, column 2
Read m[r][c] as "row r, column c." Both indices are still zero-based, so an m[3][4] has rows 0–2 and columns 0–3. The top-left is m[0][0] and the bottom-right is m[2][3] (that's m[rows-1][cols-1]). Mixing up the order — writing m[c][r] or using a comma like m[r, c] — is the classic 2D bug.
3 Initializing

Braces inside braces 🎁

You can fill a 2D array with a nested braced list — one inner { } per row — which lines up nicely with how the grid reads:

int m[2][3] = {
    {1, 2, 3},     // row 0
    {4, 5, 6}      // row 1
};

int zeros[3][3] = {0};   // every cell becomes 0
Just like 1D arrays, any cells you don't fill become 0, so {0} zeroes the whole grid. The inner braces are optional (C will fill row by row from a flat list), but using them makes the row structure obvious and is much easier to read. You can also let C infer the first dimension from the initializer: int m[][3] = {{1,2,3},{4,5,6}} is a 2×3.
4 Looping over a grid

Nested loops: rows × columns 🔁

To visit every cell, nest two loops — the outer over rows, the inner over columns. This is exactly the nested-loop pattern from lesson 3.6, now with a real grid behind it:

int m[2][3] = {{1, 2, 3}, {4, 5, 6}};

for (int r = 0; r < 2; r++) {       // each row
    for (int c = 0; c < 3; c++) {   // each column in that row
        printf("%d ", m[r][c]);
    }
    printf("\n");                   // newline after each row
}
The outer loop picks a row, the inner loop sweeps across that row's columns, and the printf("\n") between the loops ends each row — print the grid and it comes out as a neat table. This row-then-column order is also how a 2D array actually sits in memory, which matters next.
5 Memory & passing to functions

Row-major, and the column rule 🧩

In memory a 2D array is stored row by row (row 0's cells, then row 1's, …) — this is called row-major order. It's why, when you pass a 2D array to a function, C needs to know the number of columns: that's how it figures out where each row starts.

// the column count is REQUIRED; the row count is just data
int sum(int m[][3], int rows) {   // [][3] — columns must be given
    int total = 0;
    for (int r = 0; r < rows; r++)
        for (int c = 0; c < 3; c++)
            total += m[r][c];
    return total;
}

🚨 Leaving out the column size when passing a 2D array

You can write int m[][3] (skip the row count) but you cannot skip the column count: int m[][] won't compile. Without the number of columns, C can't compute where row 1 begins, so m[r][c] can't be located. The rule: when passing a 2D array, always spell out every dimension except the first.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

🔀
Swapping row and column m[r][c] is row then column. Writing m[c][r] reaches a different cell (and may go out of bounds if the grid isn't square). Keep the order straight: row first.
,
Using a comma: m[r, c] That's not how C indexes a 2D array — and it compiles to something surprising (the comma operator), not m[r][c]. Always use two separate brackets: m[r][c].
🧩
Dropping the column size in a parameter A function parameter needs the column count: int m[][3], never int m[][]. Without it, C can't locate m[r][c]. The first dimension can be left blank; the rest cannot.

🎓 What to remember

  • A 2D array is a grid: int m[rows][cols], accessed with two indices m[row][col].
  • Row first, column second, both zero-based — top-left is m[0][0], bottom-right m[rows-1][cols-1].
  • Initialise with nested braces (one per row); unfilled cells become 0.
  • Nested loops walk it — outer over rows, inner over columns.
  • Passing to a function: give every dimension except the first — int m[][cols] (it's stored row-major).
