Variables and Identifiers

A variable is a named place where your program remembers a value. Today: what variables are, how to declare them, and the rules for naming them. Your existing notes have the perfect analogy — we'll use it.

1 Why this matters

Programs need to remember things 🧠

In lesson 0.2 you met memory — the wall of numbered boxes the CPU works with. Variables are the way you, the programmer, give those boxes friendly names. Instead of saying "the number in box #4F2A," you say "the number in age." The compiler does the address-bookkeeping; you just think in names.

This is one of the most fundamental ideas in all of programming. Every program you'll ever write uses variables — to hold inputs, to count, to remember intermediate results, to track state. Today we name the thing and learn the rules.

2 What a variable is

The kitchen-container analogy 🫙

A variable is an entity whose value can be changed. A variable is a container which stores a "value." In a kitchen, we have containers storing rice, dal, sugar etc. Like that, variables in C store values.

Your notes have one of the best explanations of variables I've ever read. Three pieces in that quote:

  • A container. Like a rice canister — it has a fixed shape, holds one specific kind of thing.
  • A label. You write "Rice" on the canister so you can tell it apart from the sugar one. That's the variable's name.
  • Contents that change. Today the rice canister has 2 kg of rice; tomorrow it has 5 kg. The container stays put; the value inside changes. That's why it's called a variable — its value varies.

Three corresponding examples, again from your notes:

a = 3;   // a is assigned 3
b = 4.7;   // b is assigned 4.7
c = 'A';   // c is assigned 'A'

Each line creates a labeled container and puts a value inside. a holds a whole number, b holds a decimal number, c holds a single character. Three different kinds of value — and C wants you to be specific about which kind, which is what types are for.

3 Types — what kind of thing fits in the container

C's basic types 📦

Unlike some languages (Python, JavaScript), C needs you to declare up front what kind of value each variable will hold. The rice canister is for rice and only rice — never sugar. This strictness is one reason C is so fast.

The five basic types you'll meet right away:

int

Whole numbers
int age = 25; int count = -7;

float

Decimal numbers (single precision)
float price = 9.99f; float pi = 3.14f;

double

Decimal numbers (more precision)
double avogadro = 6.022e23;

char

A single character
char grade = 'A'; char letter = 'z';

_Bool

True or false (0 or non-zero)
_Bool done = 1; // often written 'bool' // via <stdbool.h>

Each type takes a specific amount of memory and has a fixed range of values. We'll get into the exact sizes in lesson 2.2. For now: pick the type that matches what you're storing.

4 Declaring and assigning

Three small steps 📝

Using a variable is three steps that you often do in one line:
  1. Declare — tell the compiler "I want a variable of this type with this name."
    int age;
  2. Assign — put a value into it.
    age = 25;
  3. Use — read it, do math with it, print it.
    printf("%d", age);

You can combine step 1 and 2 on a single line — that's called initializing:

// declare + initialize in one go
int age = 25;
float price = 9.99f;
char grade = 'A';

// declare now, assign later
int count;
count = 10;
count = count + 1;   // now count is 11

That last line — count = count + 1 — is a very common pattern. It reads the current value of count, adds one, and stores the result back. The container is the same; the value inside changed.

One critical rule: declaring a variable doesn't put any meaningful value in it. int x; with no initialization leaves x holding whatever junk was in that memory box before. Always initialize when you declare, unless you're about to assign on the very next line.

5 Naming rules + Try it

What's a legal variable name? 🔤

Rules for naming variables in C:
• First character must be an alphabet or underscore (_)
• No commas, blanks allowed
• No special symbol other than (_) allowed

Your notes nailed it. Add three more practical rules from modern C:

  • Subsequent characters can be letters, digits, or underscores.
  • Case matters — Age, age, and AGE are three different variables.
  • You can't use C keywords as names (int, if, return, while, etc. — about 30 of them).

7 Common mistakes

Three traps with variables 🚧

🙅
"I declared it, so it has a value." int x; declares x but leaves it with garbage — whatever bytes happened to be in that memory cell. Reading an uninitialized variable is undefined behavior and can produce different results every run. Always initialize: int x = 0;.
🙅
"Age and age are the same." Not in C. Case matters. Age, age, AGE, aGe are all different variables. Most C codebases use snake_case (e.g. user_name) or camelCase (e.g. userName) — pick one style and stick to it.
🙅
"I'll name everything x, y, z." Single-letter names are fine for loop counters and quick math. For anything else, spend the keystrokes on a real name. user_age beats a. Future-you reading this code in three months will thank present-you.
8 Your turn

Variables in your own code 💪

Easy Modify hello.c to use variables. Replace the literal "Hello, world!" with three variables: a name (string — use char name[] = "Aniket";), an age (int), and a height in meters (float). Print them in a sentence like "Aniket is 30 years old and 1.78 meters tall." using printf with format specifiers %s, %d, %.2f.
Medium Name ten variables for a contacts app. Imagine you're building a contacts app. Write down 10 variable names you'd need (name, phone, email, etc.). For each, pick a type. Make every name legal C and meaningful — no single letters except where it'd be natural. Bonus: which would be char[] (string) vs int vs other?
Stretch The uninitialized-variable trap. Write a program: int x; printf("%d\n", x); (no initialization). Compile with gcc -Wall hello.c. Read the warning. Then run the program a few times — note that the printed value might be different each run. Now add = 0 after the declaration. Recompile, rerun, note it's reliably 0. This is exactly why we initialize.

🎓 Quick recap

  • A variable is a named container in memory. The name is for you; the address underneath is for the CPU.
  • Each variable has a type declared up front. The five basic types are int, float, double, char, and _Bool.
  • Three steps: declare → assign → use. Combine the first two with initialization: int age = 25;.
  • Naming rules: start with letter or _; rest can be letters/digits/_; case matters; can't be a C keyword.
  • Always initialize. Uninitialized variables hold garbage and produce undefined behavior.
