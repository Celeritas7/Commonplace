🎯 PART VI · POINTERS

What an Address Is

Pointers are the heart of C — and the whole idea rests on one simpler concept: the address. Memory is a long row of numbered boxes, every variable lives in some of them, and the & operator tells you exactly where. Get this, and pointers become easy.

1 Why this matters

Pointers start here 🧠

Pointers have a scary reputation, but they're built on one humble idea: every piece of data your program stores sits somewhere in memory, and that "somewhere" has a number — its address. A pointer is nothing more than a variable that holds an address. So before we touch pointers at all (next lesson), let's get crystal clear on what an address actually is.

This is the part of C that other languages hide from you. In C you can ask "where does this variable live?" and get a real answer. That power is why C is used for operating systems and embedded devices — and it all begins with addresses.

2 The mental model

Memory is a row of numbered boxes 📦

Picture your computer's memory as an enormously long street of identical little boxes, lined up in a row. Each box holds one byte, and each box has its own house number — that number is its address. The boxes are numbered 0, 1, 2, 3, … all the way up.

When you declare a variable, the program parks it in one of these boxes (or several boxes in a row, for bigger types). The variable's address is the number of the first box it occupies. A char takes 1 box; an int usually takes 4 boxes side by side — but its address is still just the number of that first box.
3 Finding the address — the & operator

&x means "the address of x" 📍

You don't have to guess where a variable lives — C will tell you. Put the address-of operator & in front of a variable's name and you get its address:

int age = 30;

age   // the VALUE   → 30   (what's inside the box)
&age  // the ADDRESS → e.g. 0x7ffe… (which box it's in)
Keep the two ideas apart: age is what's inside the box (30), while &age is where the box is (its address). You've actually used & before — scanf("%d", &age) passes the variable's address so scanf knows where to put the number it reads. Now you know what that & was doing.
4 An address is just a number

Print it with %p 🔢

An address is an ordinary number — you can even print it. The format specifier for an address is %p (for "pointer"), and addresses are usually shown in hexadecimal, like 0x7ffee3a4:

int age = 30;
printf("value:   %d\n", age);          // 30
printf("address: %p\n", (void *)&age); // 0x7ffee3a4bc (varies!)
Two things to notice. First, the actual number is huge and looks random — that's normal; you almost never care about the exact value, only that it identifies a box. Second, it changes every time you run the program: the operating system places your variables wherever it likes. So never write an address down and reuse it — always ask with &.
5 Why we care

Addresses let you point 🎯

Here's the payoff. Once you can name where a variable lives, you can hand that location to someone else — a function, or another variable — so they can reach the original. That stored location is precisely what a pointer is, and it's the whole game from here on:

ExpressionReads asGives you
age"the value in age"30
&age"the address of age"a box number
Different variables always live in different boxes, so they always have different addresses — that's how the machine keeps them apart. In the next lesson we'll make a variable whose job is to hold one of these addresses: a pointer. Everything you do with pointers — pass-by-reference, dynamic memory, linked lists — is just storing and following box numbers.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

🔀
Confusing the value with the address x is what's inside the box; &x is where the box is. They're different things — one is your data, the other is a location.
🎲
Expecting the same address every run The OS places your variables in different boxes each time the program runs. Addresses look random and change run-to-run — never hard-code one; always get it with &.
📐
Thinking a variable is only one box Bigger types span several boxes — an int is usually 4 bytes. Its address is just the number of the first box; the rest sit right after it.

Practice exercises 💪

Easy Show the addresses. Declare three variables and print each one's address with printf("%p\n", (void *)&x);. Notice they're all different — each variable lives in its own box.
Medium Count the boxes. Print sizeof(char), sizeof(int), and sizeof(double). Each number is how many boxes (bytes) that type occupies — relate it back to the row-of-boxes picture.
Stretch Neighbours in a row. Make an int arr[3] and print &arr[0], &arr[1], &arr[2]. The addresses climb by sizeof(int) each step — proof the array's boxes really are side by side. (A sneak peek at pointer arithmetic.)

🎓 What to remember

  • Memory is a long row of byte-sized boxes, each with its own numbered address.
  • Every variable lives at an address — the number of its first box (an int spans 4 boxes).
  • The & (address-of) operator gives a variable's address: &x.
  • An address is just a number (print it with %p) — and it changes from run to run.
  • That "where it lives" number is exactly what a pointer stores — coming up next.
