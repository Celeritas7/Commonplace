struct — Grouping Related Data

So far every variable has stood alone. But real things have several attributes at once: a point has an x and a y; a student has a name, an age, a GPA. A struct bundles related values into a single named type you can pass, copy, and return as one unit — the foundation of every custom type you'll build in C.

1 Why this matters

Some things belong together 🧠

To represent a single point you'd need an x and a y. To represent a student: a name, an age, a GPA. You could keep these as loose, separate variables — but then nothing ties them together, you can't pass "a point" to a function in one go, and an array of 100 students becomes a tangle of parallel arrays. A struct solves this: it defines a new type that groups these fields into one value. From here on, "a Point" or "a Student" is a thing you can name, store, copy, and hand around as a unit. This opens Part VIII — Building Your Own Types.

You've informally bumped into the idea before; now it gets its proper introduction.

2 Defining a struct

A blueprint for a new type 🏷️

A struct definition lists the members (the fields) the type contains. It's a blueprint — it defines a type, it doesn't create any data yet:

struct Point {     // define a new type called "struct Point"
    int x;
    int y;
};              // ← the semicolon is REQUIRED

Then you create variables of that type. In C, the type's full name is struct Point:

struct Point p;          // one Point variable (members uninitialized)
struct Point q = {3, 4};  // initialize x=3, y=4 in order
Writing struct Point every time is a little verbose — in the next lesson you'll meet typedef, which lets you call the type just Point. For now, the key idea: the definition is a recipe, and each variable you declare is a fresh instance with its own copy of every member.
3 Reaching the members

The dot operator 🎯

You read and write a member with the dot (.): variable.member. Each member behaves exactly like a normal variable of its type:

struct Point p;
p.x = 3;              // set member x
p.y = 4;              // set member y
printf("%d, %d\n", p.x, p.y);   // reads them → 3, 4

// or initialize all at once, in declaration order:
struct Point q = {3, 4};   // q.x = 3, q.y = 4
The dot is the whole interface to a struct: p.x is just "the x inside p." You can assign to it, read it, pass it, take its address — anything you'd do with an int. A brace initializer {3, 4} fills the members in the order they were declared, so order matters.
4 Structs are values

They copy as a whole 📦

Unlike arrays, a struct is a true value. Assigning it copies every member; passing it to a function copies the whole thing; returning one works too:

struct Point a = {1, 2};
struct Point b = a;   // b is a full COPY: b.x=1, b.y=2
b.x = 99;             // changes b only — a.x is still 1

void bump(struct Point pt) { pt.x = 100; }  // gets a COPY
bump(a);              // a.x is STILL 1 — the copy was changed, not a

🚨 Pass-by-value copies the struct

Because a struct is passed by value, a function receives its own copy — edits inside don't reach the caller's variable. To let a function modify the original (or to avoid copying a big struct), pass a pointer to it: void bump(struct Point *pt), then reach members with (*pt).x or the shortcut pt->x. That arrow operator gets its own lesson next in this part.

5 Building bigger shapes

Arrays and nesting 🧩

Structs compose. You can make an array of structs, and a struct can contain other structs as members:

struct Point poly[3] = { {0,0}, {1,2}, {3,4} };
printf("%d\n", poly[1].x);   // 1  — index, then dot

struct Line {
    struct Point start;       // a struct inside a struct
    struct Point end;
};
struct Line seg = { {0,0}, {3,4} };
printf("%d\n", seg.end.y);   // 4  — chain the dots
An array of structs replaces a mess of parallel arrays (one for each field) with a single tidy list of whole records — poly[1].x reads "the x of element 1." Nesting lets you build layered types and reach deep members by chaining dots: seg.end.y. This composability is what makes structs the building block for everything from a coordinate to a whole linked list (later this part).

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

⛔
Forgetting the semicolon after } A struct definition ends with }; — leaving the semicolon off produces a confusing cascade of compiler errors on the lines that follow.
🟰
Comparing structs with == You can assign structs (b = a) but you can't compare them with == — it won't compile. Compare member by member: a.x == b.x && a.y == b.y.
📦
Expecting a function to change a struct you passed Pass-by-value gives the function a copy. To modify the caller's struct, pass a pointer (struct Point *p) and use p->x — the arrow operator, coming up next.

Practice exercises 💪

Easy Make a Point. Define struct Point { int x; int y; };, create one with {3, 4}, and print x + y. Then set p.y = 10 with the dot and print again.
Medium A Student record. Define a struct Student with a char name[32], an int age, and a double gpa. Fill one in, then write a function void print_student(struct Student s) that prints all three fields.
Stretch An array of records. Make an array of three struct Student, fill them, and loop to find the highest GPA. Notice how much cleaner one array of structs is than three parallel arrays.

🎓 What to remember

  • A struct defines a new type that groups related members — and the definition ends with };.
  • Reach members with the dot: p.x = 3; reads/writes the x inside p.
  • Brace initializers {3, 4} fill members in declaration order.
  • Structs are values — assigning, passing, or returning one copies the whole thing; pass a pointer to modify the original.
  • They compose: arrays of structs and nested structs build bigger types — the basis for everything in Part VIII.
