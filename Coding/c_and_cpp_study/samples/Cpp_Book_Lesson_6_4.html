Constructors

A constructor is a special method that runs automatically the instant an object is created — its job is to put the object into a valid starting state. C++ gives you three kinds to know: the default, the parameterized, and the copy constructor.

1 Why this matters

Born ready 🧠

So far you've made an object and then set its members one by one with the dot. That's fragile — forget a line and the object starts life half-built, holding garbage. A constructor fixes this: it's code that runs automatically when the object is created, guaranteeing every object begins in a sensible, complete state. No object can exist without one running.

You've leaned on constructors already without naming them: std::string s("hi"); and std::vector<int> v(3, 0); both call constructors. Now you'll write your own.

2 What a constructor is

A special method 🏗️

A constructor is a member function with two distinctive rules: it has the same name as the class, and it has no return type (not even void). It runs once, automatically, when an object is created:

struct Dog {
    std::string name;
    int age;

    Dog() {                  // constructor: name of the class, no return type
        name = "(unnamed)";
        age = 0;
        std::cout << "a Dog was born!\n";
    }
};

Dog d;   // the constructor runs right here, automatically
You never call a constructor by hand like d.Dog() — writing Dog d; is the call. By the time that line finishes, the constructor has run and d is fully set up. Think of it as the object's "setup routine," fired at birth.
3 The default constructor

No arguments 🆓

A constructor that takes no parameters is the default constructor — it's the one that runs for Dog d;. If you write no constructors at all, the compiler quietly generates an empty one for you, which is why your earlier classes worked:

struct Point {
    int x = 0, y = 0;     // default member values
    // no constructor written → compiler supplies an empty default one
};

Point p;   // fine: p.x and p.y are 0
Combine a default constructor with default member values (int x = 0;) and every object starts clean with zero effort. The moment you need custom setup logic — validation, a message, computed fields — you write the constructor body yourself.
4 The parameterized constructor

Set it up with arguments 🎛️

Far more useful day-to-day: a constructor that takes arguments, so you build an object with its values in one step:

struct Dog {
    std::string name;
    int age;

    Dog(std::string n, int a) {   // parameterized constructor
        name = n;
        age = a;
    }
};

Dog d("Rex", 3);   // name = "Rex", age = 3 — set at birth
One line builds a ready-to-use Dog. (There's a tidier way to write this with a member initializer list — Dog(std::string n, int a) : name(n), age(a) {} — which you'll meet in 6.9; the assignment form above does the same job for now.) A class can have several constructors with different parameters — that's just overloading, which you saw with functions in 4.3.
5 The copy constructor

A new object from an old one 📋

The third kind makes a new object as a copy of an existing one. You trigger it by initializing one object from another:

Dog a("Rex", 3);
Dog b = a;      // copy constructor: b is a duplicate of a
// b.name == "Rex", b.age == 3 — but b is independent
You don't have to write a copy constructor — the compiler provides one that copies each member across, which is exactly right for simple classes like this. The copy is independent: changing b.age later doesn't touch a. (When a class manages a resource like heap memory, the default member-wise copy can be wrong — that's a topic for RAII and the "rule of three" later in the book.)

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

🚫
Expecting a default after writing another constructor Once you define any constructor, the compiler stops giving you the free default. If you still want Dog d; to work, add Dog() {} (or Dog() = default;) yourself.
↩️
Giving a constructor a return type Constructors have no return type — not even void — and must share the class's name exactly. void Dog() {...} is just an ordinary method named Dog, not a constructor.
🗑️
Leaving members uninitialized Without a constructor or default member values, members hold garbage. A constructor (or int x = 0; defaults) guarantees a valid starting state — rely on it.

Practice exercises 💪

Easy A parameterized type. Write struct Rect with a constructor Rect(int w, int h) that sets two members. Create Rect r(3, 4); and print width and height.
Medium Both constructors. Add a default constructor that sets a 1×1 rectangle. Create one Rect each way (Rect a; and Rect b(5, 2);) and print both — confirm a is 1×1.
Stretch Copy and diverge. Make Rect b = a; (copy), print b to confirm it matches a, then change b's width and print both — show that a is unchanged, proving the copy is independent.

🎓 What to remember

  • A constructor is a special method (same name as the class, no return type) that runs when an object is created.
  • The default constructor takes no arguments — the compiler supplies one only if you define no constructors.
  • A parameterized constructor takes arguments to initialize the object: Dog d("Rex", 3);.
  • The copy constructor builds a new object as a copy of an existing one (Dog b = a;); the default copies each member.
  • Define any constructor and the free default disappears — add Dog() {} if you still need it.
