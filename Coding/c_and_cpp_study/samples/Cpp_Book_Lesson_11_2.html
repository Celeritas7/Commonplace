String Streams

A string stream gives you the whole stream interface — << and >> — but reads from and writes to a std::string held in memory. Use an ostringstream to build a formatted string, and an istringstream to parse one into fields and numbers.

1 Why this matters

A stream backed by a string 🧵

You've now used three kinds of stream: the console (cout/cin), files (ofstream/ifstream, 11.1), and now string streams — streams whose backing store is a std::string in memory. Same << and >>, no disk involved. They live in <sstream> and come in three flavours: std::ostringstream (build a string by writing into it), std::istringstream (parse a string by reading out of it), and std::stringstream (both). They're the natural tool for two everyday jobs: assembling a formatted message from mixed pieces, and pulling apart a line of text into its fields.

If files are "streams to disk," string streams are "streams to a variable" — all the stream skills you have, aimed at a std::string.

2 Building strings

std::ostringstream — assemble text ✍️

Write into an ostringstream with << just like cout, then call .str() to get the finished std::string:

#include <sstream>

std::ostringstream os;
os << "User " << name << " has " << count << " points";

std::string msg = os.str();   // "User Alice has 42 points"
This is the clean way to combine text and non-text values into one string. You can't just write "User " + name + " has " + count — that won't compile, because count is an int and there's no + that glues an int onto a string. The ostringstream handles it: << knows how to format every built-in type (and anything with an operator<<), turning numbers into their text form automatically. When you're done, .str() hands back the accumulated string. It's also where stream formatting (precision, width, hex — next lesson) pays off, since those manipulators work here too.
3 Parsing strings

std::istringstream — pull apart text 📖

Construct an istringstream from a string, then read fields out of it with >> — exactly as you'd read from cin:

std::istringstream is("Alice 42 9.5");

std::string name;
int score;
double rating;
is >> name >> score >> rating;   // "Alice", 42, 9.5
>> does the parsing for you: it skips whitespace, reads one token, and converts it to the target type — a std::string gets the next word, an int gets a parsed integer, a double gets a parsed floating-point number. So a whole line of mixed data unpacks into typed variables in one expression. Each extraction can fail (wrong format, or nothing left), which puts the stream in a fail state and tests as false — the same rule as file and console input, so the looping idiom you learned in 11.1 carries straight over.
4 The classic combo

getline + istringstream 🔗

The most common real use: read a file line by line with getline, then parse each line with an istringstream. This cleanly separates "get a line" from "split the line":

std::string line;
while (std::getline(in, line)) {       // in is an ifstream (11.1)
    std::istringstream ls(line);       // wrap the line in a stream
    std::string name;  int age;
    if (ls >> name >> age)              // parse the fields
        records.push_back({name, age});
}
Why two steps instead of reading straight from the file? Because getline gives you exactly one record (one line) at a time, and the inner istringstream lets you parse within that line without accidentally running into the next one. It also makes per-line error handling easy: a line that doesn't parse just fails the inner if and you skip it, without derailing the whole file read. For numbers specifically, you can also reach for std::stoi/std::stod (string→number) and std::to_string (number→string) — handy for a single value, while a string stream wins when there are several fields to handle together.
5 Reusing & resetting

std::stringstream and a reset gotcha 🔄

A std::stringstream can both read and write. If you want to reuse one for a fresh string, you must reset both its contents and its state:

std::stringstream ss;
ss << "100";
int a; ss >> a;          // a = 100, now at end-of-stream

ss.str("");              // reset the CONTENTS to empty
ss.clear();             // reset the STATE flags (eof/fail)
ss << "200";
int b; ss >> b;          // b = 200

🚨 str("") and clear() are two different resets

This trips up almost everyone. ss.str("") replaces the stream's text buffer with an empty string. ss.clear() resets the stream's error/state flags (like eofbit and failbit) back to "good". They're unrelated, and you usually need both to reuse a stream: after the first read hits end-of-stream, the eofbit/failbit are set, so even if you put new text in with str(...), the next >> will refuse to read until you've also called clear(). Note clear() does not empty the text (that name is misleading), and str("") does not fix the flags. When in doubt, it's often simplest to just declare a fresh istringstream per line, as in the combo above.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

🔄
Reusing a stream without clear() After a read hits end-of-stream the fail/eof flags stay set. Calling only str("") won't let you read again — you also need ss.clear() to reset the state. (Or just make a fresh stream.)
➕
Trying to + numbers onto strings "score: " + 42 doesn't compile (and pointer arithmetic if it did would be a disaster). Use an ostringstream, or std::to_string(42), to turn the number into text first.
📥
Ignoring extraction failure is >> n can fail if the text isn't a number. Check the stream (if (is >> n)) before trusting n — an unchecked failure leaves it unchanged (or zero-initialised in C++11+).

Practice exercises 💪

Easy Build a label. Given std::string name = "Bo"; int n = 7;, use an ostringstream to build the string "Bo scored 7" and print it.
Medium Sum a line of numbers. Put "3 5 8 13" in an istringstream and loop while (is >> x) to add them up. Print the total (29).
Stretch Parse records. Read a file where each line is name age city. For each line use getline then an istringstream to extract the three fields; skip any line that doesn't parse. Store the results in a std::vector of structs and print them.

🎓 What to remember

  • String streams are streams over a std::string (in <sstream>): ostringstream builds, istringstream parses, stringstream does both.
  • Build with os << pieces… then os.str() — the clean way to format mixed text and numbers into one string.
  • Parse with istringstream is(text); is >> field… — >> skips whitespace and converts each token to its type.
  • The combo: getline a line from a file, wrap it in an istringstream, extract fields — easy per-line parsing and error handling.
  • To reuse a stream reset both: str("") (contents) and clear() (state flags). For single values, std::stoi/std::to_string are simpler.
