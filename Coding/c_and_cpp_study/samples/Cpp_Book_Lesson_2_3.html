std::string — Your First Proper String Type

C has char arrays — error-prone, fixed-size, manual length tracking. C++ gives you std::string — growable, knows its own length, works with + for concatenation, == for comparison. Today: meet your text best friend.

1 Why this matters

Text done right 📝

In C, "string" means char array — a fixed-size buffer where you manually track length and pray you don't overflow it. In C++, you have std::string — a real type that handles all that for you, supports natural syntax like name + " " + surname, and grows as needed.

You'll use std::string in essentially every C++ program from here on. Names, file paths, error messages, configuration values, user input — all text. Get comfortable with it now and the rest of the book gets easier.

2 std::string vs char arrays

The reason C++ added a string type 🎯

Here's the same task — concatenate two names and print the result — done both ways. Notice how much shorter and safer the right column is.
🚨 The C way (also legal in C++)
#include <stdio.h>
#include <string.h>

char first[20] = "Aniket";
char last[20] = "Mangaonkar";
char full[50];

strcpy(full, first);
strcat(full, " ");
strcat(full, last);

printf("%s\n", full);
✨ The C++ way
#include <iostream>
#include <string>

std::string first = "Aniket";
std::string last = "Mangaonkar";
std::string full = first + " " + last;

std::cout << full << "\n";

// No size limits to worry about.
// No buffer overflows possible.
// No strcpy/strcat dance.

The C version's char full[50] is a fixed size buffer — if your name is too long, you overflow and corrupt memory. The C++ version's std::string grows automatically to fit whatever you put in it. Plus, + just works.

For text, always prefer std::string in modern C++. Use C-style char arrays only when interfacing with old C APIs (and even then, you can usually convert at the boundary).

3 Creating and initializing

The ways to make a string 🧱

Include <string> and you have std::string. It works just like the variable types we met in 2.1 — with all the same initialization syntax options.
#include <string> // All four create a string holding "Hello" std::string a = "Hello"; // classic init std::string b{"Hello"}; // brace init (preferred — narrowing safe) std::string c("Hello"); // function-call init auto d = std::string{"Hello"}; // auto deduces std::string // Empty string std::string empty; // "" — zero length, safe to use // Fill constructor — N copies of one char std::string stars(10, '*'); // "**********"

Notice the trap with auto: auto x = "Hello"; gives you a const char*, not a std::string (we saw this in lesson 2.1's stretch exercise). If you want a std::string, write std::string x = "Hello"; or use the brace form.

4 The methods you'll use most

The std::string toolkit 🧰

std::string has dozens of methods. These ten cover 95% of everyday use. Each is short, predictable, and standard across every C++ compiler.
s.length() / s.size()

How many characters? Returns a size_t (unsigned).

→ size_t (unsigned)
s.empty()

True if length is zero. Prefer this over s.size() == 0.

→ bool
s + t   or   s += t

Concatenate two strings. Result is a new string (or modifies s).

→ std::string
s == t, s != t, <, >

Compare strings lexicographically. Works exactly like comparing numbers, but on text.

→ bool
s[i]

Access the i-th character (zero-indexed, no bounds check). s[0] is the first char.

→ char&
s.at(i)

Same as s[i] but throws std::out_of_range if i is invalid. Safer.

→ char&
s.substr(start, len)

Returns a copy of len chars starting at start. "Hello".substr(1, 3) → "ell".

→ std::string
s.find("xy")

Returns position of first match, or std::string::npos if not found.

→ size_t
s.replace(start, len, "new")

Replace len chars at start with the new string.

→ std::string&
s.c_str()

Returns a const char* — for interop with C functions that need a C string.

→ const char*

7 Common mistakes

Three traps with std::string 🚧

🙅
"find returns -1 when nothing's found." Not in C++. std::string::find returns std::string::npos — which is the maximum value of size_t (a huge unsigned number, often 18446744073709551615). Compare against std::string::npos, not -1: if (s.find("x") != std::string::npos). The -1 assumption silently breaks because comparing an unsigned to signed -1 gets converted.
🙅
"c_str() gives me a pointer I can hold onto." It returns a pointer to the string's internal buffer. That pointer becomes invalid the moment the string changes or goes out of scope. Use the result immediately: printf("%s", s.c_str()); — don't store it for later. (This is one of the most-asked C++ interview questions.)
🙅
"std::string handles Unicode and emoji properly." Mostly no. std::string stores bytes, not characters. For ASCII text it works perfectly (each char is one byte). For UTF-8 text (emojis, Chinese, accented characters), one logical character may take 1-4 bytes — so s.length() returns byte count, not character count, and s[i] indexes bytes. For real Unicode work, you need a library like ICU or boost::locale. For learning C++ basics, ASCII is fine.
8 Your turn

Real string work 💪

Easy Build a greeting. Write a program that declares three std::string variables: greeting = "Hello", name = "Aniket", punct = "!". Use the + operator to build "Hello, Aniket!" in a single statement. Print the result.
Medium Word counter. Take a std::string like "The quick brown fox". Count spaces using a loop: for (char c : s) if (c == ' ') count++;. Print "word count: 4" (one more than the space count). Then try with " hello world " — does your counter handle it correctly? Why not? (This is why std::istringstream or splitting libraries exist — but we'll meet those later.)
Stretch The c_str() lifetime trap. Write this:
const char* danger;
{
  std::string temp = "hello";
  danger = temp.c_str();
}
printf("%s\n", danger); // BUG: temp is gone
Compile with -fsanitize=address (on Linux/Mac). Run. AddressSanitizer should report use-after-free. This is exactly the kind of bug C++ tooling helps you catch.

🎓 Quick recap

  • std::string from <string> is C++'s proper string type — growable, length-aware, safe.
  • Beats C-style char arrays on every front: no buffer overflows, no manual size tracking, + for concat, == for compare.
  • The core ten methods: length, empty, +/+=, ==, [], at, substr, find, replace, c_str.
  • find returns std::string::npos (a huge unsigned number) when not found — not -1.
  • c_str() returns a pointer to the internal buffer — use immediately, don't store.
  • For ASCII text, std::string is great. For Unicode work, you need extra libraries.
