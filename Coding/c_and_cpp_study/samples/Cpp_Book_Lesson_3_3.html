while, do-while, and for

A loop runs the same block again and again until you tell it to stop. C++ gives you three classic shapes: while (check first), do-while (check after, run once), and for (the counting loop with everything in its header). Let's learn all three and when to pick each.

1 Why this matters

Repeating without copy-paste 🧠

To print the numbers 1 to 100, you won't write 100 std::cout lines. A loop lets you write the repeated work once and tell C++ "keep doing this while a condition holds." Counting, summing, processing each item, retrying until success — all loops.

All three loops in this lesson do fundamentally the same thing: run a block repeatedly while a condition is true. They differ only in shape — where the setup, the test, and the update live. while and do-while keep those parts spread out; for gathers them into one tidy header. Master one and the others are small variations. And in every case, you are responsible for making the condition eventually become false — or the loop never stops.

2 while and do-while

Check first, or check after 🔁

A while loop checks its condition before each run — so it can run zero times:

int i = 1;
while (i <= 5) {
    std::cout << i << ' ';
    i++;
}                       // 1 2 3 4 5

A do-while checks after, so the body always runs at least once — perfect for menus and "ask until valid" prompts:

int choice;
do {
    std::cout << "1) Play  2) Quit\n";
    std::cin >> choice;
} while (choice != 2);   // note the semicolon!
Every loop needs three things: initialize a variable, a condition to keep going, and an update that pushes toward stopping. In while/do-while these are scattered — init before the loop, update somewhere in the body. Forget the update and the condition never changes: an infinite loop. The key difference to remember: while can run 0 times, do-while runs at least 1.
3 The for loop

Everything in the header 🎯

The for loop is built for counting. It gathers all three parts — init, condition, update — into one line, so you can see the whole loop's shape at a glance:

for (int i = 0; i < 5; i++) {
    std::cout << i << ' ';
}                       // 0 1 2 3 4

The header has three slots separated by semicolons: for (init; condition; update). This for is exactly equivalent to the while from the last section — same three pieces, just collected together instead of spread out.

The execution order trips people up, so burn it in: the init runs once at the very start. Then each pass is condition → body → update → condition → … The update happens after the body, not before. And the loop exits the moment the condition is false — without running the body that time. The for-Stepper below highlights each part as it fires.

A bonus that makes for the go-to counting loop: declaring the counter in the header (int i = 0) scopes it to the loop. After the loop ends, i is gone — it can't leak or clash with anything else. That's cleaner than a while, where the counter lives on in the surrounding scope.

4 Choosing among them

Which loop when 🤔

LoopReach for it when…
forYou're counting a known number of times, or iterating with an index
whileYou loop until a condition changes but there's no simple counter (reading input, converging)
do-whileThe body must run at least once before the test (menus, prompts)
They're interchangeable in principle — any for can be rewritten as a while and vice versa. Choose the one whose shape matches your intent, so the next reader instantly sees what kind of loop it is. "Counting to N?" → for. "Until the user types quit?" → while. "Show the menu, then maybe again?" → do-while.

One more note for C++: when you're simply visiting every element of a container (a std::vector, an array, a std::string), there's an even cleaner loop coming next lesson — the range-for. For now, the classic for with an index is your workhorse.

5 The two big dangers

Infinite loops & off-by-one 💥

🚨 The infinite loop

If the condition never becomes false, the loop never ends — usually a missing or wrong update. In a for, a common slip is putting the update in the wrong place or forgetting it in the header. In a while, it's forgetting i++ in the body. Also beware the stray semicolon: for (...); or while (...); makes the body empty.

🚨 Off-by-one

Starting at 0, i < n runs n times (0…n-1); i <= n runs n + 1 times (0…n). One character changes the count, and indexing past the end of a container is a real bug. When a loop does one too many or too few, check < vs <= and the starting value first.

Both bugs reduce to one discipline: trace the first and last iterations in your head before trusting a loop. What's the counter at the start? At exactly what value does the condition flip to false? The for-Stepper lets you watch that play out part by part.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

♾️
Forgetting the update No change to the loop variable means the condition never flips — infinite loop. In a for, keep the update in the header; in a while, don't forget i++ in the body.
📏
Off-by-one with < vs <= From 0, i < n runs n times; i <= n runs n + 1. With containers, i <= size() walks one past the end — a real bug.
🔀
Confusing the for execution order It's init → (condition → body → update) repeating. The update fires after the body, and the condition is rechecked before each body run — including the very first.

Practice exercises 💪

Easy Times table. Use a for loop to print the 7× table from 7×1 to 7×10, one per line. Choose your condition so it stops exactly at 10.
Medium Same loop, three ways. Print 1 to 10 with a for, then again with a while, then a do-while. Confirm all three give identical output — proof they're interchangeable.
Stretch Sum and average. Read 5 numbers with a for loop, accumulate the sum, then print the average. Then convert it to read until the user enters -1 — and notice that this version is better as a while, because there's no fixed count.

🎓 What to remember

  • while (cond) checks before (may run 0×); do { } while (cond); checks after (runs ≥1×).
  • for (init; cond; update) gathers all three loop parts into one header — ideal for counting.
  • for execution order: init once, then condition → body → update, repeating. Update fires after the body.
  • A counter declared in the for header is scoped to the loop — it can't leak afterward.
  • Watch the two dangers: missing update (infinite) and < vs <= (off-by-one).
