Operators Part 2 — Logical, Increment, Ternary

The operators you need to write real decisions: &&, ||, ! for combining conditions; ++ and -- for counting; and ?: — the one-line shortcut for tiny ifs.

1 Why this matters

Real programs make compound decisions 🧠

Last lesson you learned to compare two things: age >= 18. Real programs rarely check just one thing. "Is the user logged in and are they an admin?" "Is it raining or is the temperature below 5°C?" You need to glue conditions together — that's what logical operators do.

You'll also meet two of C's most beloved-and-cursed shortcuts: ++ (add one) and the ternary ?: (one-line if-else). They're everywhere in real code, and each one has a famous gotcha.

2 Logical operators

AND, OR, NOT — the three you need 🔗

🚨 Don't confuse && with &

One ampersand & is the bitwise AND operator (covered later). Two ampersands && is the logical AND. Using the wrong one usually compiles fine, but gives wrong results: 2 & 1 is 0, while 2 && 1 is 1. Same trap for | vs ||.

3 Short-circuit evaluation

C is lazy — and that's a feature ⚡

Here's the magic property: && stops as soon as it sees a false, and || stops as soon as it sees a true. The right-hand side may never be evaluated.

Why? Because the answer is already determined. If the left side of && is false, the whole expression is false — no need to look further. If the left side of || is true, the whole expression is true. C exploits this to skip work and — more importantly — to let you write safe checks like:

PatternWhat it does
if (p != NULL && *p == 5)Only dereferences p if it's non-null. Reversed order would crash.
if (count == 0 || total / count > 10)Avoids divide-by-zero by checking the denominator first.
if (debug && expensive_check())Skips the expensive call when debug is off.

This means the order of conditions matters — not just for readability, but for correctness. Put your safety check first.

4 Increment and decrement

++ and -- — and the pre/post puzzle 🧮

These add (or subtract) 1 from a variable:

FormNameWhat it does
i++Post-incrementReturns old i, then adds 1.
++iPre-incrementAdds 1, then returns new i.
i--Post-decrementReturns old i, then subtracts 1.
--iPre-decrementSubtracts 1, then returns new i.
If you use them on their own line — like i++; — pre and post are identical. They both just add 1. The difference only shows when you use them inside an expression.

Example. Given int i = 5;:

Codex ends asi ends as
int x = i++;56
int x = ++i;66

Read i++ as "give me i's current value, then increment". Read ++i as "increment first, then give me the new value". You'll play with this in the stepper below.

🚨 Don't use ++ on the same variable twice in one expression

Code like i = i++ + ++i; or a[i] = i++; is undefined behavior in C — the compiler can produce any result. Don't be clever. Split it into separate statements.

5 The ternary operator

?: — the one-line if-else 🎀

You'll often write tiny ifs like this:

The long wayThe ternary way
int max;
if (a > b) max = a;
else max = b;		 int max = (a > b) ? a : b;

The syntax is condition ? value_if_true : value_if_false. C evaluates the condition; if true, the whole expression equals the part before :; otherwise, the part after.

When to use it: picking between two values to assign or pass. Things like printf("%d apple%s\n", n, n == 1 ? "" : "s");
When NOT to use it: for side effects, multi-step logic, or anything that won't fit on one line. Just write the if — it's clearer.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

⚡
Using single & or | where you meant && or || The single forms do bitwise math, not logic. 4 & 1 is 0; 4 && 1 is 1. Both compile, and the wrong one silently gives the wrong answer. Always use the doubled forms for if-style conditions.
🔁
Side effects in short-circuited expressions If you write if (cheap_check() || expensive_thing_with_side_effect()), the side effect may never happen. That's the point of short-circuit — but it's a bug if you were counting on the side effect to run.
⚠️
Multiple ++/-- on the same variable in one expression i = i++ + ++i; is undefined behavior — different compilers print different answers, and you can't reason about it. If you need increments, put them on their own lines: int a = i; i++; int b = ++i;

Practice exercises 💪

Easy Leap-year check. Write the condition for "is year a leap year?" using && and ||. Rule: divisible by 4 AND (not divisible by 100 OR divisible by 400). Print the result for 1900, 2000, 2024.
Medium Safe divide with ternary. Read two integers a and b. Print a / b if b != 0, otherwise the string "undefined". Use a ternary inside a single printf.
Stretch Count vowels with short-circuit. Loop over a string. Inside the loop, count a character as a vowel if it matches 'a', 'e', 'i', 'o', or 'u' using a chain of ||s. Notice how short-circuit makes the chain efficient — once a match is found, the rest don't run.

🎓 What to remember

  • Logical: && (AND), || (OR), ! (NOT). Return 1 or 0. Don't confuse with bitwise & and |.
  • Short-circuit: && stops at first false; || stops at first true. Order your conditions accordingly — safety checks first.
  • Increment: i++ returns old value, ++i returns new value. Identical on their own line; different inside expressions.
  • Ternary: cond ? a : b — the one-line if. Great for tiny choices, terrible for complex logic.
  • Don't be clever: never increment the same variable twice in one expression. That way lies undefined behavior.
