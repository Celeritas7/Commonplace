🛠️ PART I · SETTING UP

Installing a C Compiler

You need exactly one thing on your machine before we can write real C: a compiler. This lesson gets one onto your computer in about 15 minutes — on Windows, Mac, or Linux.

1 Why this matters

The tool that makes C real 🛠️

Compilation is the process of translating high-level source code written in programming languages like C into machine code, which is the low-level code that a computer's CPU can execute directly.

You wrote that yourself in your notes, and it's exactly right. The tool that does this translation is called a compiler. Without one installed, the .c files you write are just text — interesting to read, but the computer can't do anything with them.

This lesson is the one-time setup: install a compiler, run one command to check it works, and you're ready for every lesson after this one. We'll come back to the compiler so often that putting in 15 minutes to set it up properly now saves hours of friction later.

The compiler we'll use is called GCC (the GNU Compiler Collection). It's free, open-source, and the same compiler that builds the Linux kernel, the Python interpreter, and most of the embedded systems in your microwave and car. If it's good enough for Linus Torvalds, it's good enough for our hello world.

2 What you need

The two pieces of your setup 🧩

We will use VS Code as our code editor to write our code and install MinGW gcc compiler to compile our C program. We can install VS Code and MinGW from their respective websites.

Two things, exactly as you noted:

  • A text editor — to write your .c files. We'll set up VS Code in the next lesson (1.2). For now, any text editor on your system will do — even Notepad.
  • A C compiler — to turn your .c file into a runnable program. That's what we install today.
Your notes mentioned MinGW (which is one way to get GCC on Windows). The modern, more reliable equivalent is MSYS2 — it's MinGW's successor and what we'll use below. On Mac and Linux, GCC comes from the system tools.
3 Pick your operating system

Three paths, one destination 🎯

Click your operating system below. Each path is well-trodden by millions of developers. Each command has a copy button — paste it into your terminal exactly as shown.
1
Download MSYS2

Go to msys2.org and click the installer link (something like msys2-x86_64-...exe). Run it, accept the defaults. Takes about 2 minutes.

2
Open the MSYS2 terminal and install GCC

From the Start menu, open MSYS2 MINGW64 (a small terminal window). Paste this and press Enter:

$ pacman -S --needed mingw-w64-x86_64-gcc

It'll ask you to confirm — type Y and Enter. Downloads a few hundred MB; takes 2–5 minutes.

3
Add GCC to your Windows PATH

So that gcc works from any terminal, not just the MSYS2 one. Open Settings → System → About → Advanced system settings → Environment Variables. Under User variables, edit Path, click New, and add:

C:\msys64\mingw64\bin

Click OK on every dialog. Close any open terminals and open a fresh one.

4
Verify it works

Open a normal Windows command prompt (Win+R → cmd → Enter) and run:

> gcc --version
gcc.exe (Rev1, Built by MSYS2 project) 13.2.0
Copyright (C) 2023 Free Software Foundation, Inc.

If you see a version number, you're done. 🎉

1
Open Terminal

Press Cmd + Space, type Terminal, hit Enter. A black-ish window opens.

2
Install Xcode Command Line Tools

This brings GCC (technically Apple's clang, which masquerades as gcc — same difference for our purposes). Paste this and press Enter:

$ xcode-select --install

A dialog pops up. Click Install. Wait 5–10 minutes (downloads several hundred MB). When it finishes, the terminal will be ready again.

3
Verify it works

Back in the terminal:

$ gcc --version
Apple clang version 15.0.0 (clang-1500.3.9.4)
Target: arm64-apple-darwin23.0.0

You'll see Apple clang in the output — that's normal. On Mac, gcc is a friendly alias for Apple's clang compiler. For our lessons, it works identically. 🎉

1
Open a terminal

Press Ctrl+Alt+T on most distros, or find Terminal in your application menu.

2
Install GCC via your package manager

Pick the one for your distro:

# Ubuntu / Debian / Mint
$ sudo apt update && sudo apt install build-essential
# Fedora / RHEL
$ sudo dnf install gcc gcc-c++ make
# Arch / Manjaro
$ sudo pacman -S base-devel

Enter your password when prompted. Takes 1–3 minutes.

3
Verify it works
$ gcc --version
gcc (Ubuntu 13.2.0-23ubuntu4) 13.2.0
Copyright (C) 2023 Free Software Foundation, Inc.

If you see a version number, you're done. 🎉

4 The "did it work?" check

One command, one answer ✅

The single command that tells you whether installation succeeded is gcc --version. Open any terminal on your computer and run it. The output should look something like this:

✅ Expected output

gcc (Ubuntu 13.2.0-23ubuntu4) 13.2.0
Copyright (C) 2023 Free Software Foundation, Inc.
This is free software; see the source for copying conditions.

The exact version, distro, and copyright line will differ — but as long as you see "gcc" and a version number, you've got a working compiler. Anything from GCC 9 onward (or any modern clang) works fine for this book.

If instead you see 'gcc' is not recognized as an internal or external command or command not found — the install didn't fully work, or your PATH isn't picking it up. Most often this means closing and reopening the terminal fixes it. If not, the next section has the fix for the most common cause.

5 Plan B — the online fallback

If install gives you trouble 🚧

Installation should work for 95% of you, but sometimes — corporate laptops, locked-down machines, network blocks — it just won't. You can still read and complete this book without a local install, using free online C compilers.

The best of them for our purposes:

  • onlinegdb.com — Full C compiler in the browser, plus a built-in debugger. Free, no signup needed to try it.
  • godbolt.org (Compiler Explorer) — Shows you the assembly your C compiles to, side-by-side. More advanced; great for later lessons.
  • replit.com — Free signup; gives you a saveable, sharable workspace. Slightly slower start-up than the others.

For early lessons, any of these works perfectly. We'll revisit local install in Part II if you start with an online one — there are good reasons to have a local compiler for the bigger projects in Parts VI onward, but you don't need to fight an install today.

7 Common mistakes

What trips beginners here 🚧

🙅
"I added it to PATH but it still doesn't work." Almost always: you didn't close and reopen the terminal. Environment variables are read at terminal startup, so any open terminal still has the old PATH. Close every terminal window, open a fresh one, try again.
🙅
"On Mac, gcc showed Apple clang — did it install wrong?" No. Apple aliases gcc to their clang compiler on purpose. Clang understands the same C as GCC and produces the same machine code for our lessons. If you specifically want real GCC on Mac, install it with brew install gcc — but you don't need to.
🙅
"MinGW or MSYS2 — which is it?" MinGW was the old way; MSYS2 is the modern replacement that packages MinGW-w64 GCC. Just use MSYS2 (per Step 1 above) and you're using the current best option. If you find old tutorials saying "install MinGW," they're not wrong, but MSYS2 is easier to maintain.
8 Your turn

Do the install 💪

Easy Install it. Follow the OS picker above. Run gcc --version at the end. Screenshot the output — it's your first real proof of a working dev environment.
Medium Read the help. Run gcc --help in your terminal. Scroll through. You don't need to understand any of it yet — but skim the section about -o (output file) and -Wall (warnings). We'll meet both in lesson 1.3.
Stretch Two compilers. Most systems can have both GCC and clang installed. Try installing the "other" one for your OS (on Mac that's real GCC via Homebrew; on Linux it's clang via your package manager). Run both gcc --version and clang --version. Why might you eventually want both?

🎓 Quick recap

  • You need a compiler installed on your computer to turn .c files into runnable programs. We're using GCC.
  • Three install paths, picked by OS: MSYS2 on Windows · Xcode Command Line Tools on macOS · build-essential (or equivalent) on Linux.
  • The one command that verifies success: gcc --version. If it prints a version, you're done.
  • On Mac, gcc is an alias for Apple's clang — same behavior for our purposes.
  • If install gives you trouble, onlinegdb.com is a perfectly fine fallback for the first few parts of this book.
