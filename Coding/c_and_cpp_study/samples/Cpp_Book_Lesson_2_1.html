Variables, Types, and auto

A variable is a named place where your program remembers a value. C++ gives you the same idea as C — plus one big modern addition: auto, which asks the compiler to figure out the type for you.

1 Why this matters

Programs need to remember things 🧠

In lesson 0.2 you met memory — the wall of numbered boxes the CPU works with. Variables are how you give those boxes friendly names. Instead of "the int at address 0x7ffe8a3c," you say "the variable age." The compiler does the address-bookkeeping; you just think in names and types.

Every C++ program uses variables — for inputs, counters, intermediate results, state. Today: declare them, name them, and meet C++'s signature improvement over C: auto for type inference.

2 What a variable is

The container analogy 🫙

A variable is a named container in memory that stores a value of a specific type. Think of it like a labeled spice jar: the label is the name, the shape determines what fits inside (only one kind of thing), and the contents can be refilled.

The "of a specific type" part matters more in C++ than most languages. Where Python lets you put anything anywhere — x = 5 then x = "hello" with no fuss — C++ insists you declare up front: this variable holds integers, and only integers. That strictness is one of the things that makes C++ fast and safe.

The simplest possible variable in C++:

int age = 25;       // container labeled "age" holds the integer 25
double price = 9.99;   // container labeled "price" holds the decimal 9.99
char grade = 'A';     // container labeled "grade" holds the character 'A'

Three different kinds of value, three different types. Pick the type that matches the data — that's how C++ thinks.

3 The basic types

What fits in each container 📦

C++ inherits C's basic types, and adds a proper bool type, plus std::string for text (which we'll meet properly in lesson 2.3). Here are the five you'll see most.

int

Whole numbers
int age = 25; int count = -7;

double

Decimal numbers (default float type in C++)
double price = 9.99; double pi = 3.14159;

bool

true or false (a real type, not 0/1)
bool done = true; bool ok = false;

char

A single character
char grade = 'A'; char letter = 'z';

std::string

Text (a sequence of characters)
std::string name = "Aniket";

Two differences from C worth flagging: bool (real type, with true and false as keywords — much nicer than C's _Bool) and std::string (a proper string type that knows its own length, can grow, and supports + concatenation — far easier than C's null-terminated char arrays). We use both freely from here on.

4 Declaring and initializing

Three steps, usually in one line 📝

The pattern is the same as C: declare → assign → use. C++ adds a few new ways to initialize — pick whichever you find readable. 
// 1. The C-style way (still works)
int a = 25;

// 2. The "uniform initialization" syntax (C++11) — preferred by many
int b{25};
double c{3.14};
std::string d{"hello"};

// 3. Function-call style (also works)
int e(25);

// All three of these declare an int with value 25.
// The {} form has one nice property: it catches narrowing conversions.
int f{3.14};   // compiler ERROR — can't narrow double to int silently
int g = 3.14;  // compiles fine, silently truncates to 3 😬

The {} form (called brace initialization or uniform initialization) is the modern recommendation — it works for every type the same way, and it catches a class of silent bugs by refusing narrowing conversions. We'll use it throughout the rest of the book.

Always initialize. Like in C, declaring a variable without giving it a value leaves it holding garbage. The empty {} form is also valid and means "zero-initialize" — int x{}; gives x = 0 reliably.

5 The auto trick

Let the compiler write the type for you 🌟

auto — type inference

C++11 introduced the auto keyword. It tells the compiler: "figure out the type from the value on the right; don't make me write it." The compiler still picks one specific type — there's no runtime cost. It's just a typing-saving feature.

Compare these two equivalent lines:

// Without auto — you write the type explicitly
std::vector<std::string>::iterator it = names.begin();

// With auto — the compiler figures it out
auto it = names.begin();   // it's exactly the same type, less typing

// More everyday examples:
auto age = 25;            // auto = int
auto price = 9.99;         // auto = double
auto name = std::string{"Aniket"};  // auto = std::string
auto letter = 'A';          // auto = char

When to use auto: use it freely when the type is obvious from the right-hand side, and especially when the type would be long or annoying to spell out (iterators, lambda types, template results). Don't use it when it would obscure the type and confuse a reader (auto x = compute(); — what's that?).

One subtle catch: auto x = 3.14; gives you a double, not a float — because in C++, unsuffixed decimal literals default to double. If you specifically wanted a float, write auto x = 3.14f;.

6 Naming rules + Try it

What's a legal variable name in C++? 🔤

C++ naming rules are essentially identical to C:
• First character: letter or underscore
• Subsequent: letters, digits, or underscores
• Case matters
• Can't be a reserved keyword (C++ has more keywords than C — class, template, auto, nullptr, constexpr, etc.)

8 Common mistakes

Three traps with C++ variables 🚧

🙅
"auto x; — that should give me an auto-typed variable." It can't. auto means "infer the type from the initializer" — without an initializer, there's nothing to infer from. The compiler errors. Always write auto x = ...; with a value, or use an explicit type like int x;.
🙅
"auto x = 3.14; gives me a float." It gives you a double. In C++, decimal literals default to double. If you specifically want a float, use the f suffix: auto x = 3.14f;. (You'll rarely want float over double anyway — double is fine for almost everything.)
🙅
"int x = 3.14; is fine." Compiles, but silently truncates to 3, losing the decimal. The brace-init form int x{3.14}; would refuse to compile and force you to make the truncation explicit. This is one reason modern C++ prefers brace initialization — it catches a whole class of silent bugs.

Variables in your own C++ 💪

Easy Modify hello.cpp to use variables. Replace the literal in std::cout with three variables: a std::string name, an int age, a double height in meters. Print: "Aniket is 30 years old and 1.78 meters tall." using chained <<. Hint: std::cout << name << " is " << age << " years old..."
Medium Try every initialization syntax. Declare the same variable four different ways and print all four to confirm they hold the same value:

int b2{25};

All four should print 25. Then try int d{3.14}; and read the error.
Stretch The auto surprise. On godbolt.org, write auto x = "hello";. What type is x? It's not std::string — it's const char* (a C-style string). Now write auto y = std::string{"hello"}; — what's y's type? Use typeid(x).name() from <typeinfo> to confirm, or just hover over x in VS Code and read the tooltip. Understanding what auto deduces is half of using it well.

🎓 Quick recap

  • A variable is a named container in memory. Each one has a fixed type declared up front.
  • Five basic types you'll meet most: int, double, bool, char, std::string.
  • Three initialization syntaxes: int a = 25;, int a{25}; (preferred), int a(25);. The {} form catches narrowing conversions.
  • auto (C++11) lets the compiler infer the type from the initializer. Free, no runtime cost. Use it freely when the type is obvious.
  • Naming rules: start with letter or _; rest can be letters/digits/_; case matters; can't be a C++ keyword.
  • Always initialize. Use int x{}; for "zero-initialize" — beats undefined garbage.
