break and continue

Sometimes you need to bail out of a loop early, or skip just one round and keep going. break exits the loop; continue jumps to the next iteration. They work in every loop — including the range-for — and each has one subtlety worth knowing.

1 Why this matters

Loops with an exit and a skip 🧠

A normal loop only stops when its condition turns false. But real code often needs to bail out sooner — "stop searching the moment you find the match" — or skip a single round — "process every item except the invalid ones." break and continue give you that control without contorting your loop condition.

They're a loop's two escape hatches. break is the emergency exit: it abandons the loop entirely and jumps to the code after it. continue is the skip button: it abandons just the current iteration and moves to the next. Both work in while, do-while, the classic for, and the range-for you just learned. Let's take them one at a time — including the one trap each carries.

2 break

The emergency exit 🚪

break immediately ends the loop it's in. Execution jumps to the first line after the loop — no more iterations, no final check. It shines in searches, and reads beautifully with a range-for:

// Stop as soon as we find the target
for (const auto& item : items) {
    if (item == target) {
        std::cout << "Found it!\n";
        break;          // done — leave the loop now
    }
}
The classic use is searching: once you've found what you want, there's no point checking the rest. break stops right there. You already met this keyword in lesson 3.2 — it's the same one that ends a switch case. (In fact, break inside a loop that contains a switch only breaks out of the switch, not the loop — something to keep in mind.)
3 continue

The skip button ⏭️

continue abandons the rest of the current iteration and jumps ahead to the next. The loop keeps running — you just skip whatever came after the continue this round:

// Sum only the positive scores
int total = 0;
for (int s : scores) {
    if (s < 0) {
        continue;       // negative? skip it, go to the next
    }
    total += s;            // only reached for non-negative s
}
continue is for filtering — handle most items, skip the ones that don't qualify. The idiom is "if this one's no good, continue" near the top of the loop, so the main work stays un-indented instead of buried inside a big if. In a range-for, continue simply moves on to the next element — clean and safe.
4 Where continue jumps

The infinite-loop trap 💥

Here's the subtlety. continue doesn't restart from the top — it jumps to the loop's next step. In a for or range-for, that means the advance to the next iteration still happens automatically. But in a while loop, you control the advance — and continue can skip right over it.

🚨 continue in a while loop can loop forever

If the variable that advances the loop is updated after the continue, then continue skips it — and the condition never changes:

int i = 0;
while (i < 5) {
    if (i == 2) continue;  // skips i++ → i stuck at 2 forever!
    std::cout << i;
    i++;
}

The fix: advance the loop variable before any continue, or use a for / range-for, where moving to the next iteration is automatic.

5 Nesting, return, and labels C++ doesn't have

What they reach — and what they don't 🎯

  • They affect only the innermost loop. A break inside two nested loops exits just the inner one; the outer loop keeps going.
  • break vs return. break leaves the loop but stays in the function; return leaves the whole function. If finding your answer means you're done, return can be cleaner.
  • No labeled break. Unlike some languages (Java, for instance), C++ has no break outer; to jump out of several loops at once. To do that, pull the loops into a helper function and return, or use a boolean flag.
A good rule of thumb: break for "I'm done here" (searching, hitting a limit) and continue for "skip this one" (filtering). When you need to escape nested loops, the cleanest C++ answer is almost always to extract the loops into a function and return from inside — it's clearer than juggling flags, and it reads as "find the thing and hand it back."

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

♾️
continue in a while that skips the update If i++ sits after the continue, it never runs and the loop hangs. Advance the variable before any continue, or use a for / range-for where advancing is automatic.
🪆
Expecting break to exit all nested loops break leaves only the innermost loop, and C++ has no labeled break. To leave several at once, extract the loops into a function and return, or use a flag.
🔀
Forgetting break in a switch inside a loop A break inside a switch that's inside a loop breaks out of the switch, not the loop. If you meant to leave the loop, you need a different structure (a flag or return).

Practice exercises 💪

Easy First match. Given a std::vector<int>, use a range-for to find and print the first element greater than 100, then break. If there is none, print "not found" after the loop.
Medium Skip the blanks. Loop over a std::vector<std::string> and print only the non-empty strings, using continue to skip the empty ones. Use const auto& so you don't copy.
Stretch Escape a grid. Write nested loops over a 2-D grid (a vector<vector<int>>) searching for a target value. Since break only exits the inner loop, refactor the search into a function that returns the moment it finds the target — the idiomatic C++ way to "break out of both loops."

🎓 What to remember

  • break exits the loop entirely and jumps to the code after it — great for searching.
  • continue skips the rest of the current iteration and moves on — great for filtering.
  • The while trap: continue can skip the advance, causing an infinite loop. In for / range-for the advance is automatic.
  • Only the innermost loop is affected, and C++ has no labeled break — extract to a function and return to escape nested loops.
  • Use sparingly: one clear early exit aids readability; a swarm of jumps hurts it.
