🧰 PART IV · FUNCTIONS

Writing Your First Function

Last lesson was the why. Now the how: the exact shape of a function definition, how parameters carry values in, and how return hands a value back. By the end you'll be able to write and call your own.

1 Why this matters

From idea to actual code ✍️

In 4.1 you saw why functions are worth making. This lesson is the hands-on part: the precise syntax. A function definition is just four pieces in a fixed order — once you can read those four pieces, you can write any simple function and call it.

We'll build one tiny function, take it apart piece by piece, and watch a value flow in as an argument and come back as a return value. Everything here stays at the level of values going in and a value coming out — pointers and arrays come later. Master this shape and you've got the backbone of every C program.

2 The shape

Four pieces in order 🔧

Here is a complete function. Read the first line as a sentence, then look at the body:

int triple(int n)   // return type · name · parameter list
{                  // body begins
    return n * 3;  // compute a value and hand it back
}                  // body ends
  • Return type — int. The kind of value handed back. void if nothing comes back.
  • Name — triple. What you call it by (a verb is ideal).
  • Parameter list — (int n). Named inputs with types, in parentheses. (void) or () if there are none.
  • Body — the { … }. The statements that run, ending with a return when there's a value to give back.
That first line — int triple(int n) — is the function's header. It's a contract: "give me one int, I'll give you back one int." The body is how the contract is kept. Get the header right and the rest is just ordinary statements you already know.
3 Parameters carry values in

Arguments become parameters 📥

When you call the function, the values in the parentheses are arguments. Each argument is copied into the matching parameter — they line up by position:

int n = triple(5);   // argument 5 is copied into parameter n
                       // inside triple, n == 5
Two words that sound alike but aren't: the parameter is the name in the function's header (n); the argument is the actual value you pass at the call (5). In C, the argument is copied into the parameter — the function works on its own copy. So changing n inside triple never touches the caller's variable. (That copy behaviour is exactly what the C++ book digs into; for C, parameters are always copies until you learn pointers.)
4 return hands a value back

One value out, and you're done ↩️

The return statement does two things at once: it sends a value back to the caller, and it ends the function immediately. The whole call expression then becomes that value:

int triple(int n) {
    return n * 3;
}

printf("%d\n", triple(5));   // triple(5) becomes 15 → prints 15
int big = triple(5) + 1;       // 15 + 1 → big is 16
The return type must match what you hand back: triple returns an int, so return n * 3; gives an int. A void function returns nothing — it can still use a bare return; to stop early, but it can't return a value. And a non-void function must hit a return with a value on every path, or you'll get garbage.
5 Putting it together

A whole little program 🧩

Here's triple living in a real program. Notice it's defined above main so the compiler meets it first:

#include <stdio.h>

int triple(int n) {     // defined first…
    return n * 3;
}

int main(void) {
    int result = triple(7);   // …so main can call it. result is 21
    printf("%d\n", result);
    return 0;
}

🚨 Calling before the compiler has met the function

If triple is written below main, older C compilers silently assume it returns int and take whatever arguments you pass — an "implicit declaration" that hides real bugs (modern compilers warn or error). The simple rule for now: define functions above main. Lesson 4.3 shows the alternative — a prototype — so you can put definitions anywhere.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

🚪
A non-void function with no return value If triple reaches its end without returning an int, the caller gets garbage. Every path of a value-returning function must end in a return value;.
🔁
Expecting changes to a parameter to reach the caller In C the argument is copied into the parameter. Doing n = n + 1; inside the function changes only the copy — the caller's variable is untouched. (Pointers, later, are how you reach back out.)
🔢
Mismatched return type If a function is declared int but you return 3.9;, the value is truncated to 3. Make the return type match the value you actually intend to hand back.

Practice exercises 💪

Easy Write square. Define int square(int n) that returns n * n. Call it from main and print the square of 6. Remember to define it above main.
Medium Two parameters. Write int max2(int a, int b) that returns the larger of a and b (use an if). Test it with a few pairs from main.
Stretch A void helper. Write void print_line(int width) that prints width dashes followed by a newline, using a loop. It returns nothing — call it to underline a heading. Notice you call it as a statement, not inside a printf.

🎓 What to remember

  • A definition has four pieces: return type, name, parameter list, body.
  • Arguments are copied into parameters by position — the function works on its own copy.
  • return value; hands one value back and ends the function; the call becomes that value.
  • void = nothing returned; a bare return; just stops early.
  • Define above main (or use a prototype — lesson 4.3) so the compiler knows it first.
