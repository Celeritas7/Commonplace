printf and scanf — Format Specifiers

You've used printf("Hello\n") since lesson one. Now learn the % placeholders that let you print numbers, control decimals and width, and read input back from the keyboard with scanf — including its one infamous gotcha.

1 Why this matters

The format string is a fill-in-the-blanks template 🧠

printf takes a format string with little placeholders in it, then fills each placeholder with a value you supply. printf("Score: %d\n", 42) prints Score: 42. The %d is a blank that says "put an integer here." Everything else in the string is printed literally.

This is most of how a C program shows its work — and the matching function scanf uses the same placeholder language in reverse, to read values the user types. The catch is that the placeholder has to match the type of the value. Use %d for an integer, %f for a floating-point number. Get it wrong and C doesn't warn you at runtime — it just prints garbage. So this lesson is really about speaking the placeholder language fluently.

2 The specifiers you'll actually use

The everyday placeholders 🔤

SpecifierForExample value → output

Two that trip up beginners: %f prints a double with six decimals by default (3.140000), and %c with a number prints the character at that ASCII code, not the number itself. To print the digits of a char's code, use %d.
3 Width, precision, and flags

Controlling how it looks 📐

Between the % and the letter you can add modifiers: %flagswidth.precisionletter. The two you'll use constantly are width (minimum number of columns) and precision (digits after the decimal):

These are how you make tidy aligned tables of numbers, money with exactly two decimals, or zero-padded IDs. You don't need to memorize them — you need to recognize the pattern and reach for the right one. The printf Playground below lets you try any combination live.

4 Reading input with scanf

scanf — the mirror of printf ⌨️

scanf uses the same specifiers to read values the user types and store them into your variables. The crucial difference: you must pass the address of each variable with &, so scanf knows where to put the value.

CodeWhat it does
int age;
scanf("%d", &age);		Read an integer into age
double p;
scanf("%lf", &p);		Read a double — note %lf, not %f!
scanf("%d %d", &a, &b);Read two integers separated by spaces

🚨 Forgetting the & in scanf

scanf("%d", age); — without the & — passes the value of age instead of its address, so scanf writes to a random location. It often crashes. Rule: printf takes values, scanf takes addresses. (The exception is strings with %s into a char array, because the array name already is an address — more on that in the arrays part.)

Also mind the asymmetry for doubles: printf uses %f for a double, but scanf needs %lf. And scanf returns the number of items it successfully read — checking that return value (e.g. if (scanf("%d", &n) != 1)) is how you detect bad input instead of trusting it blindly.
5 Matching the type

The golden rule 🎯

The single most important habit: the specifier must match the argument's type. C trusts your format string completely — it reads the bytes of each argument exactly as the specifier dictates, with no type checking at runtime.

  • printf("%d", 3.14) — %d expects an int, but you gave a double. Result: undefined behavior, usually a garbage number.
  • printf("%f", 5) — %f expects a double, but 5 is an int. Also undefined. Write 5.0 or use %d.
  • printf("%s", 42) — %s expects a string address, but 42 isn't one. Likely a crash.
Modern compilers can catch many of these if you turn on warnings (-Wall with GCC or Clang). The compiler reads your literal format string, checks each specifier against the matching argument, and warns on a mismatch. Compile with warnings on — it turns a silent runtime bug into a friendly compile-time message.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

🔀
Wrong specifier for the type printf("%d", 3.14) prints garbage — %d is for int, not double. Match the specifier to the value, and compile with -Wall so the compiler flags mismatches for you.
📍
Dropping the & in scanf scanf("%d", n) writes to a random address and usually crashes. scanf needs the address: scanf("%d", &n). printf takes values; scanf takes addresses.
🔢
Using %f for scanf doubles For printf a double is %f, but scanf needs %lf. Mixing them up means scanf reads into the wrong-sized slot. Remember: scanf doubles take %lf.

Practice exercises 💪

Easy Money formatter. Print a price stored in a double as exactly two decimals with a dollar sign — e.g. $3.50. Test it in the Playground with %.2f and a value like 3.5.
Medium Aligned table. Print three rows of name (left-aligned, width 10) and score (right-aligned, width 5) so the columns line up. Figure out which flags and widths you need, then verify in the Playground.
Stretch Robust input. Write a scanf that reads an integer and checks its return value: if the user types non-numeric input, print "That's not a number" instead of using a garbage value. Explain why checking the return value matters.

🎓 What to remember

  • The format string is a template: literal text plus % placeholders that get filled by your arguments.
  • Core specifiers: %d int, %f double (6 decimals by default), %c char, %s string, %% literal percent.
  • Width & precision: %.2f sets decimals, %5d sets minimum width, %-5d left-aligns, %05d zero-pads.
  • scanf takes addresses (&x), reads doubles with %lf, and returns how many items it read.
  • Match the specifier to the type — mismatches are undefined behavior. Compile with -Wall.
