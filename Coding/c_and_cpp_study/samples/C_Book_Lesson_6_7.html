🎯 PART VI · POINTERS

const and Pointers

A pointer involves two things — the data it points to and the pointer itself — and const can lock either, both, or neither. That's four combinations. The trick to reading them is simple: go right to left around the *.

1 Why this matters

Two things to protect 🧠

For a plain variable, const has one meaning: "you can't change this." But a pointer is two things at once — the value it points to and the pointer variable itself — so const has two places it can go, giving four combinations. The most common one, "pointer to const," is everywhere in real C: it's how you say "this function reads through the pointer but won't write." Knowing all four (and how to read them) makes pointer declarations stop looking like punctuation soup.

You already used "pointer to const" implicitly in 6.6 (const struct Big *p). Now we'll make all four combinations explicit, with a reliable way to decode any of them.

2 The reading rule

Read right to left 🔄

The position of const relative to the * decides what's locked. Read the declaration from the variable name backwards:

const int *p;   // p is a pointer to (int that is const) → can't change the DATA
int *const p;   // p is a const pointer to int      → can't change the POINTER
The * is the pivot. const to the left of * applies to the data being pointed at. const to the right of * applies to the pointer itself. (And const int means the same as int const — the data is const either way; only the side of the * matters.) Once you spot which side the const sits on, the meaning falls out.
3 Pointer to const

const int *p — the read-only view 🛡️

This is the workhorse. const int *p means you can't modify the data through p, but you can re-aim p at something else:

int x = 5, y = 9;
const int *p = &x;

*p = 10;   // ❌ error — can't write through a pointer-to-const
p = &y;    // ✅ ok — re-aiming the pointer is fine
It's a read-only window onto the data. This is exactly what you want for a function that should look but not touch — void print(const int *a, int n), or the standard const char * for strings you won't modify. The pointer can move around; the thing it points at is off-limits for writing through it.
4 Const pointer

int *const p — fixed aim 🔒

Swap the side and you get the opposite. int *const p means you can modify the data through p, but you can't re-aim p — it's stuck pointing where it started:

int x = 5, y = 9;
int *const p = &x;   // must be initialised — it can never move

*p = 10;   // ✅ ok — change the data it points to
p = &y;    // ❌ error — can't re-aim a const pointer
Because the pointer can never move, you must point it at something when you declare it. This combination is less common than pointer-to-const, but it shows up when a pointer must always refer to one fixed thing (for example, a pointer to a specific hardware register). The data stays writable; the aim is frozen.
5 All four at a glance

The full grid 🎚️

DeclarationChange data *p?Re-aim p?
int *p✅ yes✅ yes
const int *p❌ no✅ yes
int *const p✅ yes❌ no
const int *const p❌ no❌ no
Two independent switches: one for the data (const left of *), one for the pointer (const right of *). const int *const p locks both — a read-only window that can't move — while plain int *p locks neither. In day-to-day C you'll reach for pointer-to-const constantly and the others rarely, but reading any of them is now just "which side is the const on?"

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

🔄
Reading const left to right const int *p and int *const p are opposites. The * is the pivot: const before it locks the data, after it locks the pointer. Read right to left around the star.
🪟
Thinking const int *p makes x constant It only blocks changing x through p. The variable x itself is still writable by its own name — p is just a read-only window onto it.
📖
Using a plain pointer for read-only access If a function only reads, take a const pointer-to-const-data (const T *p). It documents intent and lets the compiler stop accidental writes — the professional default.

Practice exercises 💪

Easy Pointer to const. Declare const int *p = &x;. Try *p = 9; (error) and p = &other; (ok). Confirm which line the compiler rejects.
Medium Const pointer. Declare int *const p = &x; and try the opposite: *p = 9; (ok) and p = &other; (error). Note it must be initialised at declaration.
Stretch Read-only parameter. Write void printArr(const int *a, int n) that prints all elements. Try sneaking in a[0] = 0; and watch the compiler stop you — the read-only idiom in action.

🎓 What to remember

  • A pointer has two things const can lock: the data, the pointer, both, or neither — four combinations.
  • Read right to left around the *: const left of * locks the data; right of * locks the pointer.
  • const int *p (pointer to const): can re-aim, can't write *p — the read-only idiom (const char *, const T *).
  • int *const p (const pointer): can write *p, can't re-aim.
  • Use pointer-to-const for read-only access — it documents intent and lets the compiler catch mistakes.
