Arrays

So far each variable held one value. An array holds many values of the same type under a single name — a row of numbered boxes you reach into by position. It's how C stores lists: scores, pixels, the characters of a word.

1 Why this matters

One name for a whole list 🧠

Imagine storing five test scores. With plain variables you'd write score1, score2, … score5 — and good luck looping over them. An array solves this: int score[5]; gives you five integer slots under one name, reachable by a number. Now a single loop can touch every one.

An array is a fixed-length run of same-type values sitting side by side in memory. Picture a row of numbered lockers: same name (score), different positions (score[0], score[1], …). This lesson covers the three things you do with one — declare it, fill it, and reach into it — plus the famous gotcha that the position numbers start at 0, not 1.

2 Declaring & initializing

Make the row of boxes 📦

Declare an array with a type, a name, and a size in square brackets. You can leave the boxes empty, or fill them at the same time with a braced list:

int score[5];                       // 5 ints, values undefined
int score[5] = {90, 85, 70, 60, 100}; // 5 ints, filled
int score[]  = {90, 85, 70};          // size inferred: 3
int zeros[5] = {0};                   // first is 0, the rest become 0 too
A few handy rules: the size in [ ] must be a constant the compiler knows up front. If you give an initializer list but no size, C counts the items for you (int score[] = {90,85,70} is size 3). And if you give fewer values than the size, the rest are filled with 0 — which is why int zeros[5] = {0}; is the classic "all zeros" trick. Leave an array uninitialised, though, and its contents are garbage until you set them.
3 Accessing elements

Reach in by index — starting at 0 🔢

You read or write a single box with its index in square brackets. The catch that trips up every beginner: indices start at 0, so a 5-element array has valid indices 0 through 4:

int score[5] = {90, 85, 70, 60, 100};

printf("%d\n", score[0]);   // 90  — the FIRST element
printf("%d\n", score[4]);   // 100 — the LAST (size 5 → last index 4)
score[1] = 95;                // write: change the 2nd element
Say it once and it sticks: the first element is score[0], and the last is score[size - 1]. An int score[5] has indices 0, 1, 2, 3, 4 — five of them, but the highest is 4, not 5. Off-by-one mistakes here (using 1 to 5, or going up to 5) are the single most common array bug.
4 Looping over an array

The loop and the array are made for each other 🔁

Arrays and for loops are a perfect match: the loop counter is the index. This is where arrays pay off — one loop processes the whole list:

int score[5] = {90, 85, 70, 60, 100};
int total = 0;

for (int i = 0; i < 5; i++) {   // i goes 0,1,2,3,4
    total += score[i];          // add each element
}
printf("Average: %d\n", total / 5);
Notice the loop condition: i < 5, not i <= 5. With < 5, i takes the values 0 to 4 — exactly the valid indices. Using <= 5 would make i reach 5 and read score[5], which doesn't exist. The pattern for (i = 0; i < size; i++) is the canonical "visit every element" loop — burn it into memory.
5 The big trap

C does not check the boundaries 🚧

Here's the part that makes C arrays sharp-edged: there is no bounds checking. If you read or write score[5] (or score[-1], or score[100]), C doesn't stop you — it just touches whatever memory happens to be there.

🚨 Going out of bounds

Accessing an array outside 0 … size-1 is undefined behaviour. It might read garbage, silently corrupt another variable, or crash — and it may "work" by luck on your machine and fail elsewhere. The compiler won't warn you. You are responsible for keeping every index in range. This is exactly why the C++ book starts with std::vector, which can check for you — but in C, the discipline is yours.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

1️⃣
Off-by-one: treating the last index as the size In int a[5], the last valid index is 4, not 5. Loop with i < 5 (or i <= 4), never i <= 5.
🚧
Reading an uninitialised array int a[5]; with no initializer holds garbage. Read a[2] before writing it and you get a random value. Initialise (even to {0}) or fill before you read.
📏
Hard-coding the size in many places If the array size changes, every loop bound must change too. A common fix is a named size: #define N 5 (or const int N = 5;), then use N everywhere. Less to get wrong.

Practice exercises 💪

Easy Sum and average. Make int a[5] = {4, 8, 15, 16, 23};, loop to compute the total, and print the total and the average.
Medium Find the maximum. Given an array of integers, loop through it to find and print the largest value. Start by assuming a[0] is the max, then compare each later element.
Stretch Reverse in place. Print an array of 6 numbers backwards (last to first), then swap elements so the array itself is reversed: swap a[0]↔a[5], a[1]↔a[4], a[2]↔a[3]. Mind the indices!

🎓 What to remember

  • An array holds many values of one type under a single name: int a[5];.
  • Initialise with a braced list; missing values become 0, and int a[] = {…} infers the size.
  • Index from 0: the first element is a[0], the last is a[size − 1].
  • Loop with for (i = 0; i < size; i++) to visit every element.
  • No bounds checking — going outside 0 … size−1 is undefined behaviour. Keeping indices in range is your job.
