Recursion

A function is allowed to call itself. It sounds like a trick, but with one small example it clicks: give the function a way to stop, and a way to shrink the problem each call, and it solves naturally self-similar tasks in a few lines.

1 Why this matters

When a problem contains a smaller copy 🧠

Some tasks are self-similar: the factorial of n is n times the factorial of a smaller number; counting down from 5 is "say 5, then count down from 4." For these, a loop works, but a function that calls a smaller version of itself mirrors the problem exactly. That's recursion, and it's the last big idea in this Functions chapter.

You already know a function can call other functions. Recursion is simply a function calling itself — with a slightly smaller job each time — until the job is tiny enough to answer outright. The whole concept fits in two parts, and the call stack you'll watch below is what makes it work.

2 The two parts

Base case + recursive case 🧩

Every recursive function needs a base case (a condition that stops and returns an answer directly) and a recursive case (which does a bit of work and calls itself on a smaller input). Here's the small example that makes it click — a countdown:

void countdown(int n) {
    if (n == 0) { printf("Liftoff!\n"); return; }  // base case: stop
    printf("%d ", n);
    countdown(n - 1);                              // recursive case: smaller n
}
// countdown(3) prints: 3 2 1 Liftoff!
Trace it in your head: countdown(3) prints 3, then calls countdown(2), which prints 2, then countdown(1), then countdown(0) — which hits the base case and stops. The recursive call must move toward the base case (here n - 1 shrinks n), or it never stops. That's the whole pattern.
3 Returning a value

Factorial — the classic ✖️

Recursion really shines when each call returns something the caller combines. The textbook example is factorial — n! = n × (n−1)!:

int fact(int n) {
    if (n <= 1) return 1;        // base case
    return n * fact(n - 1);       // recursive case
}
// fact(4) = 4 * 3 * 2 * 1 = 24
fact(4) = 4 * fact(3)
        = 4 * (3 * fact(2))
        = 4 * (3 * (2 * fact(1)))
        = 4 * (3 * (2 * 1))     // base case
        = 24
Read fact as its own definition: "the factorial of n is n times the factorial of n − 1, and the factorial of 1 is just 1." Notice the return in front of the recursive call — without it, the multiplied answer would be thrown away. The Call Stack tool below shows how these paused multiplications wait, then finish.
4 The call stack

Calls wait, then unwind 📚

How can fact(4) finish before fact(3)? It can't — so it waits, paused mid-multiplication, while fact(3) runs, which waits on fact(2), and so on. Each paused call is stacked on the call stack. Once the base case returns, the answers flow back down and each waiting call completes.

This is the same call stack behind every function call — recursion just stacks the same function many times. Each call keeps its own n (locals are per-call, from lesson 4.4), which is why fact(4) still knows its n is 4 when control finally returns to it. Recursion isn't magic; it's ordinary function calls, paused and resumed.
5 Watch out

No base case, no mercy 💥

If the recursion never reaches its base case, each call stacks another frame until the stack runs out of memory — a stack overflow, and the program crashes.

🚨 Missing or unreachable base case

int bad(int n) { return n + bad(n - 1); } has no if to stop it, so it recurses forever (until it overflows the stack and crashes). Always include a base case, and make sure every recursive call moves toward it. And remember: anything recursion does, a loop can do too — reach for recursion when the self-similar shape makes the code clearer, not as a rule.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

💥
No base case (or you can't reach it) Without a stopping condition — or if the recursive call doesn't shrink toward it — the function recurses forever and overflows the stack. Check: does every path move toward the base case?
🔙
Forgetting to return the recursive result In a value-returning recursion, write return n * fact(n - 1);. Calling fact(n - 1) without return discards the answer and the result is wrong.
🐢
Recursing when a loop is simpler Recursion is elegant for self-similar problems, but a plain loop is often clearer (and avoids stack growth). Use whichever makes the code easiest to read.

Practice exercises 💪

Easy Countdown. Write void countdown(int n) that prints n down to 1, then "Liftoff!", recursively. Base case at n == 0.
Medium Sum to n. Write int sum_to(int n) returning 1 + 2 + … + n (sum_to(n) = n + sum_to(n - 1), base case sum_to(0) = 0). Check it against n*(n+1)/2.
Stretch Power. Write int power(int base, int exp) recursively: power(base, 0) = 1, otherwise base * power(base, exp - 1). Trace power(2, 4) on paper and confirm you get 16.

🎓 What to remember

  • Recursion is a function calling itself on a smaller input.
  • Two parts: a base case that stops, and a recursive case that moves toward it.
  • Calls stack up and wait, then unwind once the base case returns — each call keeps its own variables.
  • Put return in front of the recursive call when you need its result.
  • No reachable base case → stack overflow. A loop can always do the same job — pick the clearer one.
