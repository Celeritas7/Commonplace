🛠️ PART I · SETTING UP

Choosing and Setting Up an Editor

You could write C in Notepad. You shouldn't. The right editor catches mistakes as you type, jumps to definitions, runs your code on one keypress. We pick VS Code, get it installed, and tick off a 6-item setup checklist.

1 Why this matters

What an editor actually does for you 💪

A good editor turns hours of frustration into minutes of progress. It's the difference between "I think this might be right" and "I can see it's right before I run it." That difference compounds over every lesson in this book.

Notepad can hold C source code, sure — but it's blind. It can't tell you that you mis-spelled printf as prinft, that you forgot a semicolon, or that you're calling a function with the wrong arguments. A real code editor catches all of that as you type. Today we install one and configure it specifically for C.

2 The superpowers you're about to gain

Six things your editor will do for you ✨

You don't need to understand any of these yet. You just need to know they're coming, so when they happen during the next lessons you'll know why.
🎨
Syntax highlighting Keywords, strings, numbers each get their own color so code is scannable at a glance.
💡
IntelliSense (autocomplete) Start typing print and the editor suggests printf, with its expected arguments.
🚨
Live error squiggles Red underlines appear under mistakes as you type, not just when you compile.
🔧
One-key build and run F5 or Ctrl+Alt+N runs the file. No retyping gcc hello.c -o hello && ./hello.
🐛
Step-through debugger Pause execution at any line, inspect every variable. The way professionals find bugs.
🔗
Jump-to-definition Ctrl+click any function name to leap to where it's defined, even across files.
3 Why VS Code

The reasonable default, and what else exists 📝

We will use VS Code as our code editor to write our code...

Your notes already picked VS Code, and it's the right call. Here's why, and what the main alternatives are if you ever want to compare:

FREE · CROSS-PLATFORM · BEGINNER-FRIENDLY

Visual Studio Code

The most popular code editor in the world. Free, lightweight, runs on Windows/Mac/Linux. Huge extension ecosystem. Backed by Microsoft. Our choice for this book.

FREE · WINDOWS ONLY · HEAVIER

Visual Studio Community

Microsoft's full IDE (different product from VS Code). More powerful for huge projects but overkill for learning. Windows only.

FREE · CROSS-PLATFORM · MINIMAL

Vim / Neovim

Lightning fast, terminal-based, beloved by veterans. Steep learning curve. Worth trying after you've got a few months of C under your belt.

PAID · CROSS-PLATFORM · POLISHED

CLion (JetBrains)

Heavy-duty C/C++ IDE with stellar refactoring. Subscription required (free for students). Many pros use it. Way more tool than you need today.

If you already use one of the others happily, you can absolutely follow this book in it — but the rest of these instructions assume VS Code.

4 Install VS Code

Three steps, five minutes ⏱️

The installer is the same on all three operating systems. Download, run, open.
1
Download

Go to code.visualstudio.com. The site auto-detects your OS and offers the right installer on the big button. Click it.

2
Run the installer

Accept all the defaults. On Windows, do tick the boxes for "Add to PATH" and "Register Code as an editor for supported file types" when offered — they're convenient. On Mac, drag the app to Applications. On Linux, your distro's package install runs as usual.

3
Open it

Launch VS Code. You'll see a welcome screen with sample shortcuts. Close that — we'll explore as we go. The window has three main parts: a tall thin activity bar on the left (icons), a wider sidebar next to it (file tree, search, etc.), and the big editor area in the middle where your code lives.

5 The setup checklist

Six quick configurations 📋

Click each item below to expand. Tick the box when you've done it. Each finished item = +8 XP. Finish all 6 = bonus + confetti.
🛠️ Your VS Code setup checklist
0 / 6 done
✓
🧠
Install the C/C++ extension (Microsoft)The big one — provides IntelliSense, error squiggles, and the debugger.
▼

Click the Extensions icon in the activity bar (looks like four squares with one breaking off). In the search box at the top, type C/C++. The first result published by Microsoft (with millions of installs) is the one you want. Click Install.

That's the single most important extension. It alone gives you syntax highlighting, IntelliSense, code navigation, and the debugger. After installation, VS Code may ask if you want to install additional related extensions — click Yes.

✓
▶️
Install Code RunnerOne-click button to run the current file. Beginner-friendly shortcut.
▼

Same extensions panel, search Code Runner, install the one by Jun Han. After installing, a little triangle ▶️ appears at the top-right of every editor window. Click it → your C file compiles and runs in the terminal panel at the bottom. Magic.

(Eventually you'll graduate to proper VS Code tasks and the F5 debugger. Code Runner is your training wheels — useful for the first few lessons.)

✓
💾
Turn on Auto SaveSo you never lose work to a crash or forgotten Ctrl+S.
▼

Open File → Auto Save and tick it. Done. Now your file saves whenever you switch focus or wait a moment between keystrokes. No more "I forgot to save" debugging mysteries.

✓
🎨
Turn on Format on SaveYour code gets tidy whitespace automatically.
▼

Open File → Preferences → Settings (or Ctrl+,). In the search bar, type format on save. Tick the checkbox.

Now every time the file saves, VS Code reformats your code consistently — indentation, spacing around braces, etc. The "right" style is enforced for free, and you stop arguing with yourself about it. (We'll learn more about formatting rules later; the defaults are fine.)

✓
🚦
Install Error LensInline error messages right next to the line, instead of just a tooltip.
▼

Search the Extensions panel for Error Lens, install the one by Alexander. After install, errors and warnings appear inline at the end of the line they're on, in red or yellow text. No more hovering over squiggles to read the message.

This is hands-down the most quality-of-life extension for beginners. You'll know what's wrong without breaking your typing flow.

✓
📁
Install Material Icon ThemePretty file icons in the sidebar. Pure quality-of-life.
▼

Search Extensions for Material Icon Theme, install it. Then run Command Palette (Ctrl+Shift+P / Cmd+Shift+P) → type Icon Theme → pick Material Icon Theme.

Now every file type has a distinct, recognizable icon: .c files are blue, header files have a different look, folders are colorful. Tiny detail, but you'll notice when you switch back to the default — boring grey blobs everywhere.

Tick each item as you complete it. There are 6 in total.

7 Common mistakes

Three setup traps 🚧

🙅
"I installed VS Code AND Visual Studio. Why doesn't anything work?" Two completely different products with confusingly similar names. VS Code (a lightweight editor) is what we want. Visual Studio (a huge IDE) is something else. Check the icon — VS Code has a blue chevron </>; Visual Studio has a colored infinity symbol. Open VS Code (the blue one) and ignore the other.
🙅
"I installed an extension but nothing changed." Some extensions activate immediately; others need a window reload. If you don't see a change, run Command Palette → Developer: Reload Window (or just close and reopen VS Code). If still nothing, check the Extensions panel — there may be an "enable" toggle you missed.
🙅
"IntelliSense isn't suggesting anything when I type." Two common causes: (1) the file isn't saved with a .c extension yet (it has to know the language), or (2) the C/C++ extension can't find your compiler. Open Command Palette → C/C++: Edit Configurations (UI) and make sure the "Compiler path" field points at your gcc from lesson 1.1.
8 Your turn

Configure your editor 💪

Easy Do the checklist. Click through all 6 items above and actually install/configure each one. Screenshot the Extensions panel showing them all installed. (This is the warm-up for lesson 1.3 where you'll write your first program.)
Medium Open a folder, not a file. Create an empty folder somewhere on your machine called c-practice. In VS Code, choose File → Open Folder and pick it. Notice the sidebar now shows your folder's contents. Always open the folder, not individual files — VS Code is far smarter when it knows the project root.
Stretch Customize a key binding. Open File → Preferences → Keyboard Shortcuts. Find the action "Run Code" (from Code Runner) and check what key it's bound to. Try changing it to F6 (or whatever you like). Bonus: try the same for "Format Document". Customizing your tools is a programmer habit you'll never regret.

🎓 Quick recap

  • VS Code is the reasonable default editor. Free, fast, cross-platform, huge ecosystem.
  • The one must-install extension is Microsoft's C/C++ extension — it gives you IntelliSense, error squiggles, and the debugger in one shot.
  • The 6-item checklist: C/C++ extension · Code Runner · Auto Save · Format on Save · Error Lens · Material Icon Theme.
  • Always open a folder, not a single file. VS Code is much smarter when it sees a project root.
  • You're now set up to write, edit, and run C code productively. Next lesson: hello.c, line by line.
