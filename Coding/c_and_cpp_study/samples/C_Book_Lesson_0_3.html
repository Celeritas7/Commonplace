Why C? Where C Lives in the Modern World

C is 50+ years old. Easier languages exist. So why learn it? Spoiler: because almost every "easier language" you could pick is itself built on top of C.

1 Why this matters

The honest question: should you even bother? 🤔

Maybe you've heard people say C is old, complicated, dangerous, or replaced by "better" languages. Some of that is true. Most of it misses the point. Before we spend the next 50+ lessons on this language, let's settle: what does C give you that nothing else does?

Short answer: C is the language closest to the hardware that you can still write comfortably as a human. Almost every other language you'll meet — Python, JavaScript, Java, Ruby — runs on top of something written in C. So when you learn C, you're not learning a niche tool. You're learning the layer the whole programming world sits on.

2 The case for C

Four reasons C is still standing 💪

Each of these is a reason C didn't fade away when newer languages arrived. Hover any card to see it lift.
1

It's fast

C compiles to native machine code with almost no overhead. There's no interpreter, no garbage collector slowing things down. When milliseconds matter — operating systems, games, trading systems — C is often the answer.

2

It's small

The whole language fits in a thin book. About 30 keywords. That makes it possible to fully understand — and to run on tiny chips with 64 KB of memory, where Python or Java can't fit.

3

It's portable

A well-written C program compiles on essentially every operating system and CPU architecture that exists — from your phone to a Mars rover. Few languages can claim that.

4

It's the foundation

The Python interpreter is written in C. So is Ruby's. So is JavaScript's V8 engine (mostly C++, but founded on C ideas). Linux is C. Most of macOS and Windows is C. C is the bedrock under "modern" programming.

3 A day in your life, secretly powered by C

Tap each moment to see what C is doing 👇

Each box below is a moment in a typical day. You probably don't think about C at any of them — but it's there, quietly running. Tap to reveal what specifically. Each reveal = +8 XP.
📅 A Tuesday in your life
0 / 8 revealed
7:00 AM
⏰ Your phone's alarm goes off ▼
The Linux kernel (in Android) and the XNU kernel (in iOS) are both written almost entirely in C. They're the layer that wakes your phone up, decides which app gets CPU time, and talks to the speaker that plays your alarm sound.
7:30 AM
☕ Microwave reheats your breakfast ▼
The microcontroller running your microwave — typically an 8-bit or 16-bit chip with kilobytes of memory — runs firmware written in C. It reads the buttons, counts the seconds, drives the display, and turns the magnetron on and off. Same story for your washing machine, coffee maker, and dishwasher.
8:15 AM
🚗 You start the car ▼
A modern car has between 50 and 150 little computers called ECUs (Engine Control Units). Almost all of them run firmware written in C — usually under MISRA-C, a strict subset designed for safety-critical systems. They manage fuel injection, anti-lock braking, airbag deployment, and the touchscreen on the dashboard.
9:00 AM
🐍 You open Python to do some data work ▼
You're writing Python — but CPython, the interpreter that runs your Python code, is written in C. When you call len(list) or numpy.sum(arr), you're calling C code under the hood. NumPy, SciPy, and pandas are all C (and Fortran) underneath. Python is fast for you to write, but it's only fast to run because C is doing the heavy lifting.
12:00 PM
📞 You make a video call ▼
The codecs that compress your voice and video on the fly — H.264, H.265, Opus, AAC — are written in C. They have to run in real time on whatever chip your laptop or phone has. That's a job only a language like C can do, year after year, on every device.
3:00 PM
💾 You save a file to disk ▼
The filesystem driver (ext4 on Linux, NTFS on Windows, APFS on Mac) is written in C. It's what knows how to chop your file into blocks, find free space on the disk, and remember where everything lives. If this code had a bug, you'd lose data — which is why it's been C for 40 years.
8:00 PM
📺 You stream a movie on your TV ▼
Your smart TV runs Linux (C). The video decoder chip's firmware is C. The network stack pulling bits down from Netflix's servers is C. The HDMI controller is C. Five different layers of C cooperate to put each frame on your screen, 60 times per second.
11:00 PM
🔋 You plug in your phone to charge ▼
Inside the charging brick is a tiny chip running C firmware that negotiates voltage with your phone. Inside the phone, a power management IC (PMIC) — also running C — decides how to safely route current to the battery without overheating it. The whole "fast charge" handshake is C, talking to C.
Reveal all 8 to count the C in your day. The honest count is usually way higher than people guess.
4 What learning C teaches you

The hidden bonus: a different brain 🧠

Even if you eventually spend your career writing Python or JavaScript, the things C makes you think about will improve every line of code you write in any language.
🔍
How memory actually works Python hides it. JavaScript hides it. C makes you face it. After C, you'll understand why your Python list operation is slow, or why your JavaScript app uses 2 GB of RAM.
⚡
Why fast code is fast C teaches you what the CPU actually likes — contiguous memory, predictable branches, small loops. That intuition is gold no matter what language you ultimately use.
🧩
How "high-level" languages are built You'll see how a string, a list, a hashmap, a garbage collector actually work, because in C you have to build them. Suddenly Python's dict stops being magic.
🛡️
Defensive programming habits C will bite you if you're sloppy. That teaches you to check your assumptions, validate input, and think about every edge case — habits that pay off in every language.

6 Common mistakes

Myths people tell about C 🚧

🙅
"C is obsolete." The Linux kernel alone is ~30 million lines of active, evolving C. New C standards (C11, C17, C23) keep coming. The TIOBE language ranking puts C in the global top 5 most years. "Obsolete" isn't the word.
🙅
"You need to be a hardware expert to learn C." You don't. You just need patience with the idea that the computer is dumb and literal. C makes that literalism visible. That's a teachable mindset, not a degree.
🙅
"Modern languages have replaced C for systems work." Rust is gaining ground for new systems projects. But the existing 50 years of operating systems, drivers, libraries, embedded firmware, and infrastructure is C. A "replacement" that ignores billions of lines of working code isn't really a replacement.
7 Your turn

Try these before moving on 💪

Easy Spot the C around you. Look around the room you're in right now. Count five things you can see that almost certainly have C running inside them (even if you can't see the chip). Write the list down.
Medium The C iceberg. Pick one piece of software you use daily (a browser, an editor, a game). Spend 10 minutes searching online for "is <that app> written in C?" or "what language is <that app>?" — and trace the dependencies one level down. (Example: VS Code is JavaScript/TypeScript… but it runs on Electron, which runs on Chromium, which is mostly C++/C…)
Stretch The bootstrap problem revisited. Read the Wikipedia entry on "self-hosting" in computing. How did the very first C compiler get built — before any C compiler existed? (You'll need to learn what "bootstrapping" means in compilers. It's a beautiful puzzle.)

🎓 Quick recap

  • C is still everywhere — operating system kernels, embedded firmware, codecs, language interpreters, network stacks. Most of it is invisible to end users.
  • Four reasons C is still standing: fast, small, portable, foundational.
  • Almost every "modern" language you might pick (Python, JavaScript, Ruby, PHP) has a C implementation under it.
  • Learning C teaches you how memory works, why fast code is fast, and how higher-level languages are built. Those mental models pay off in every language you'll ever use.
  • "C is dead" is a myth. The Linux kernel alone is millions of lines of active C, with no sign of slowing down.
