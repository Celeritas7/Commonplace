static Members

Most members belong to each object — every object has its own copy. A static member is different: it belongs to the class as a whole, so there's exactly one copy shared by every object. Perfect for a count of all instances or a shared constant.

1 Why this matters

Some data belongs to the class 🧠

Normal data members are per-object: each Dog has its own name, each Counter its own count. But sometimes a piece of data belongs to the type as a whole, not to any single object — like "how many Widget objects exist right now," or a shared configuration value all objects use. A static member is stored once for the whole class and shared by every object, so it's the right tool for class-wide data.

You've seen this idea in the library already: std::string::npos is a static member — one value that belongs to the string type itself, not to any particular string.

2 static data member

One copy, shared by all 🔗

Mark a data member static and there's a single shared copy for the whole class. Change it through any object (or the class) and every object sees the change:

struct Widget {
    static int count;   // ONE shared copy for all Widgets
    Widget() { count++; }   // each new Widget bumps the shared count
};
There's no count inside each Widget — there's one count that all of them touch. If Widget() does count++, then after three objects exist, count is 3 — a running tally of instances. Compare that with a normal member, where each object would have started at its own private 1.
3 Defining it

Declare inside, define once outside 📝

A static data member is only declared inside the class — you must define it once, outside, to give it actual storage (and an initial value):

struct Widget {
    static int count;   // declaration (no storage yet)
};

int Widget::count = 0;   // definition — once, in a .cpp file
Forgetting this line is a classic beginner error — the program compiles but won't link ("undefined reference to Widget::count"). Since C++17 you can skip the separate definition by writing inline static int count = 0; right in the class. Either way, the member is created once for the whole program, before main runs.
4 Accessing it

Widget::count — through the class 🏷️

Because it belongs to the class, you reach a static member with the class name and :: — no object required. It exists even when zero objects do:

std::cout << Widget::count;   // ✅ read it with no objects at all

Widget a, b;
std::cout << Widget::count;   // 2
std::cout << a.count;         // 2 — also works through an object (same shared value)
You can access it through an object (a.count), but Widget::count is clearer — it signals "this is class-wide, not part of a." Whether you read it via a or b or the class name, you're looking at the one shared value.
5 static member functions

Methods with no object ⚙️

You can also make a function static. A static member function belongs to the class, not an object — so it has no this and can only touch static members. Call it with Class::func():

struct Math {
    static int square(int x) { return x * x; }   // no object needed
};

std::cout << Math::square(5);   // 25 — called on the class itself
Since there's no object, a static function can't use ordinary (per-object) members or this — it can only work with static members and its parameters. Use static functions for operations that relate to the class but don't need a particular object: utility helpers, or a "factory" that creates and returns objects. It's the same idea as the static data member — belonging to the type, not an instance.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

🔗
Forgetting to define the static member Declaring static int n; inside the class isn't enough — define it once outside: int Widget::n = 0; (or use inline static in C++17). Otherwise you get a linker error.
🚫
Using this in a static function A static member function has no object and no this, so it can't touch ordinary per-object members — only static members and its parameters.
👥
Expecting per-object copies A static member is shared. Change it through one object and every object sees it. For data that should differ per object, use an ordinary (non-static) member.

Practice exercises 💪

Easy Count instances. Give a struct Robot a static int count, define it as 0, and do count++ in the constructor. Create three robots and print Robot::count — it should be 3.
Medium Rise and fall. Also do count-- in the destructor. Create some robots inside a { } block, print Robot::count inside and after the block, and watch it drop as they're destroyed.
Stretch A static helper. Add a static int howMany() that returns count, and a static utility like Math::square(int). Call them as Robot::howMany() and Math::square(6) — no object needed.

🎓 What to remember

  • A static data member is shared by all objects — one copy for the whole class, not per-object.
  • Declare it static inside; define it once outside (int C::n = 0;), or use inline static (C++17).
  • Access it via the class name: Class::member — it exists even with no objects.
  • A static member function has no this; it can only use static members. Call it Class::func().
  • Use static for class-wide data (an instance counter, a shared constant) and object-free helpers.
