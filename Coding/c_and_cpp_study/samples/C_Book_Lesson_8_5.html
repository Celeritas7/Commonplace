A First Data Structure: Linked List

Everything in Part VIII comes together here. A singly linked list is a chain of nodes — each a struct holding a value and a pointer to the next node. With struct, ->, and malloc/free, you'll build a structure that grows one element at a time. Your first real data structure, end to end.

1 Why this matters

A structure that grows ⛓️

An array is fixed and contiguous: to grow it you reallocate and copy (you saw this with the dynamic array in 7.5). A linked list takes a different approach — it stores each element in its own little struct (a node) scattered anywhere in memory, and chains them together with pointers: every node holds a pointer to the next one. To add an element you just allocate one node and link it in — no copying the rest. This is the moment struct, the -> operator, and malloc/free all combine into something genuinely useful, and it's the gateway to every pointer-based data structure (trees, graphs, hash tables).

The capstone of Part VIII: you now have every piece you need to build it.

2 The node

A struct that points to itself 🔗

A node holds the data plus a pointer to the next node. The struct refers to its own type — that's why the tag Node is needed inside:

typedef struct Node {
    int value;
    struct Node *next;    // points to the next node (or NULL at the end)
} Node;

Node *head = NULL;          // the whole list IS just a pointer to the first node
// head == NULL means the list is empty
A list is represented by a single head pointer — the address of the first node. Each node's next points to the following node, and the last node's next is NULL, which marks the end. An empty list is simply head == NULL. Notice the node is self-referential: struct Node *next uses the struct Node tag (the typedef name Node isn't fully defined yet at that line), exactly the pattern from 8.2's stretch exercise.
3 Adding a node

Prepend — link it at the front ⬅️

The simplest insert puts the new node at the front. Allocate it, set its value, point it at the old head, then make it the new head:

Node *push_front(Node *head, int v) {
    Node *n = malloc(sizeof *n);   // a fresh node on the heap
    n->value = v;
    n->next  = head;             // new node points at the old first node
    return n;                     // n is the new head
}

head = push_front(head, 3);     // list: 3
head = push_front(head, 7);     // list: 7 → 3
head = push_front(head, 10);    // list: 10 → 7 → 3
This is O(1) — constant time, no matter how long the list, because you only touch the front. Notice the order of the two assignments matters: point the new node at the old head first, then update head. Prepending builds the list in reverse of insertion order (the last one pushed ends up first) — which is fine, and something the interactive below makes vivid.
4 Walking the list

Traverse until NULL 🚶

To visit every node, start at head and follow next until you hit NULL:

for (Node *p = head; p != NULL; p = p->next) {
    printf("%d ", p->value);   // 10 7 3
}
// p starts at the first node and "hops" along the next pointers;
// the loop ends when p becomes NULL (past the last node)
This loop is the heartbeat of every linked structure: a walking pointer p that advances with p = p->next and stops at NULL. From it you can build everything else — counting nodes, summing values, searching for one, printing them. An empty list (head == NULL) just means the loop body never runs, which is exactly the behaviour you want.
5 Freeing the list

Free every node — carefully 🧹

Each node was malloc'd, so each must be free'd. The catch: once you free a node you can't read its next — so save it first:

Node *p = head;
while (p != NULL) {
    Node *next = p->next;   // SAVE next before freeing p
    free(p);                 // now p is gone
    p = next;                // hop to the saved next
}
head = NULL;                 // list is now empty

🚨 Don't read a node after you free it

The classic bug is free(p); p = p->next; — that's a use-after-free: you've already returned the memory, so reading p->next is undefined behaviour. Always grab next into a local before the free. And don't skip the freeing loop either: every node you allocated and never free is a leak. Run your list under AddressSanitizer or Valgrind (7.6) to confirm it's clean.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

🧹
Use-after-free while freeing free(p); p = p->next; reads freed memory. Save next into a local before free(p), then advance.
🏁
No NULL terminator If a node's next is never set to NULL, traversal runs past the end into garbage and crashes. Start head = NULL and make the last node's next NULL.
💧
Leaking the whole list Every node is a separate malloc. Dropping head without walking and freeing each node leaks all of them. Free the list when you're done.

Practice exercises 💪

Easy Build and print. Define the Node struct, prepend 1, 2, 3, then traverse and print. Confirm you get 3 2 1 (prepend reverses).
Medium Count and sum. Write int length(Node *head) and int sum(Node *head) using the traversal loop. Test them on a list and on the empty list (both should handle NULL).
Stretch Append and free. Write push_back (walk to the last node, link the new one there) and a free_list that frees every node safely. Run it under a sanitizer (7.6) and confirm zero leaks.

🎓 What to remember

  • A node is a struct holding a value and a next pointer to another node.
  • The list is a head pointer; the last node's next is NULL; empty means head == NULL.
  • Prepend (n->next = head; head = n;) adds at the front in O(1).
  • Traverse with for (p = head; p; p = p->next) — stop at NULL.
  • Free every node, saving next before each free — no use-after-free, no leaks.
