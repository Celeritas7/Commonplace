📦 PART V · ARRAYS & STRINGS

The <string.h> Toolkit

Since a C string is just a char array, you can't copy it with = or compare it with ==. The <string.h> library gives you the proper tools: strlen, strcpy, strcat, strcmp — and their safer, length-limited cousins.

1 Why this matters

Strings need functions, not operators 🧠

In most languages you copy a string with = and compare with ==. In C those seem to work but do the wrong thing: a C string is an array, so == compares addresses, not letters, and you can't assign one array to another at all. To actually work with text you call functions from <string.h>.

This is the standard string toolbox every C programmer uses daily: measure (strlen), copy (strcpy), join (strcat), and compare (strcmp). Each is simple, and each shares the same responsibility you learned in 5.4 — you must make sure the destination has room, including the '\0'. We'll cover the everyday four, plus the strn* versions that guard against overruns.

2 The everyday four

length, copy, join, compare 🧰

FunctionWhat it doesExample
strlen(s)length (letters before '\0')strlen("cat") → 3
strcpy(d, s)copy s into d (with its '\0')strcpy(d, "hi")
strcat(d, s)append s to the end of d"foot" + "ball"
strcmp(a, b)compare; 0 if equalstrcmp("a","a") → 0
 
#include <string.h>

char dest[20] = "foot";
strcat(dest, "ball");     // dest is now "football"
printf("%lu\n", strlen(dest)); // 8
strcpy(dest, src) reads "copy into dest" — the destination comes first (like dest = src). The same order holds for strcat(dest, src): it tacks src onto the end of dest. Both write a fresh '\0' at the end for you — but only if there's room.
3 strcmp — the surprising one

0 means equal ⚖️

Comparing strings trips up everyone, because strcmp returns 0 when the strings are equal — not 1, not "true." It returns a negative number if the first string comes before the second alphabetically, positive if after:

if (strcmp(a, b) == 0)    // THE way to check equality
    printf("same\n");

// strcmp("apple", "banana")  → negative ('a' < 'b')
// strcmp("cat",   "cat")     → 0        (equal)
// strcmp("dog",   "cat")     → positive ('d' > 'c')

🚨 Using == to compare strings

if (a == b) does not compare the text — it compares the two arrays' memory addresses, which are almost never equal, so it's nearly always false even for identical text. To compare contents you must write strcmp(a, b) == 0. Read it as "their difference is zero, so they're the same." This is one of the most common C bugs there is.

4 The destination must have room

You own the buffer 📦

These functions write into a buffer you provide, and they don't check its size. If the destination is too small to hold the result plus its '\0', they write past the end — a buffer overflow, the classic C security bug.

char small[4];
strcpy(small, "hello");   // 🚨 "hello" needs 6 bytes, small holds 4 — OVERFLOW
The array must be big enough for the final string plus the terminator. For strcat, that means room for both pieces joined together. The compiler won't warn you — sizing the buffer correctly is entirely your job, exactly like the bounds-checking from lesson 5.1. This risk is why the safer versions exist.
5 The safer strn* versions

Add a length limit 🛡️

Each risky function has an n cousin that takes a maximum number of characters, so it can't write more than you allow: strncpy, strncat, strncmp.

char dest[8];
strncpy(dest, src, 7);   // copy at most 7 chars…
dest[7] = '\0';          // …and terminate yourself, to be safe

if (strncmp(a, b, 3) == 0)  // compare only the first 3 characters
    printf("same start\n");
The limit caps how far they'll write or read, which prevents the overflow. One gotcha: strncpy does not always add a '\0' (if the source is as long as the limit, it stops without one), so it's good practice to place the terminator yourself afterward. As a beginner: prefer the strn* versions, and always remember the destination still needs room for the terminator.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

⚖️
Comparing with == (or expecting strcmp to return 1) a == b compares addresses, not text. And strcmp returns 0 for equal — so the check is strcmp(a, b) == 0, not == 1 or a bare if (strcmp(a, b)).
📦
A destination too small strcpy/strcat don't check the buffer size. The destination must hold the final string plus the '\0', or you overflow. Size it generously, or use the strn* versions.
🟰
Assigning strings with = dest = "hello"; doesn't copy text into an array (you can't assign to an array). Use strcpy(dest, "hello") to actually put the characters in.

Practice exercises 💪

Easy Greet by name. Read a name into a char buffer, then build a greeting in a second buffer using strcpy (copy "Hello, ") and strcat (append the name). Print it. Make sure the buffer is big enough.
Medium Password check. Read a word and compare it to a fixed password with strcmp(...) == 0, printing "correct" or "wrong." Then deliberately try if (word == password) and see that it doesn't work.
Stretch Roll your own strlen. Write your own my_strlen(const char *s) that counts characters until '\0', and your own my_strcmp that walks both strings comparing characters. Compare results with the real ones — you'll understand exactly what the library does.

🎓 What to remember

  • C strings need functions, not operators — include <string.h>.
  • The everyday four: strlen (length), strcpy (copy), strcat (append), strcmp (compare).
  • strcmp returns 0 when equal — check with strcmp(a, b) == 0; never use ==.
  • You own the buffer: the destination must have room for the result plus the '\0'.
  • Prefer the strn* versions (strncpy, strncat, strncmp) with a length limit to avoid overflows.
