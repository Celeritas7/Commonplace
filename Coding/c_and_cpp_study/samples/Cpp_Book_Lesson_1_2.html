Editor + Turning On Modern C++

VS Code is the standard pick. Add a couple of extensions, flip one critical flag (-std=c++20), and you're configured for the entire rest of the book.

1 Why this matters

An editor that understands modern C++ 💪

A good editor catches mistakes as you type — wrong types, undeclared identifiers, missing semicolons. For a language as nuanced as C++, that real-time feedback is the difference between flowing and floundering. And one critical flag (-std=c++20) tells your tools you're writing the modern stuff, not legacy C++98.

This lesson does two things: gets you a properly configured VS Code, and explicitly turns on the modern C++ standard so the next lessons work as written.

3 Why VS Code

The modern default, and what else exists 📝

Various C++ IDEs and compilers are available. We'll use a free tool called Code::Blocks, which includes both an IDE and...

Your notes' suggestion of Code::Blocks was reasonable a decade ago, but the C++ world has moved on. The modern free choices:

FREE · CROSS-PLATFORM · MODERN

Visual Studio Code

The dominant editor for new C++ projects in the last several years. Free, fast, runs everywhere, huge extension ecosystem. Plays nicely with g++/clang++. Our choice.

FREE · WINDOWS ONLY · POWERFUL

Visual Studio Community

Microsoft's full IDE (different product from VS Code). Best-in-class for big Windows C++ projects, but heavy for learning. Free for personal use.

PAID · CROSS-PLATFORM · POLISHED

CLion (JetBrains)

Premium C++ IDE with stellar refactoring, native CMake support, and a beautiful debugger. Subscription needed (free for students). Many professionals swear by it.

DEPRECATED · LEGACY

Code::Blocks

The old recommendation. Hasn't seen a major update since 2020. Still works, but won't help you learn modern C++ habits. Skip in favour of VS Code.

If you're determined to use Visual Studio or CLion you can absolutely follow this book — but the next instructions assume VS Code.

4 Install VS Code

Three steps, five minutes ⏱️

1
Download

Go to code.visualstudio.com. The big button auto-detects your OS. Click it.

2
Run the installer

Accept all defaults. On Windows, tick "Add to PATH" and "Register Code as an editor for supported file types" when offered. On Mac, drag the app to Applications.

3
Open it

Launch VS Code, close the welcome tab. The interface is three parts: the tall activity bar on the left, a wider sidebar next to it (file tree, search, etc.), and the big editor area in the middle.

Install the C/C++ extension (Microsoft)The big one — IntelliSense, error squiggles, debugger.
▼

Click the Extensions icon in the activity bar (four squares with one breaking off). Search C/C++. The first result by Microsoft (millions of installs) is what you want. Click Install.

This single extension gives you syntax highlighting, IntelliSense, navigation, and the debugger. After installing, VS Code may offer to install the "C/C++ Extension Pack" — accept it; it brings the CMake Tools too, which is useful from lesson 1.4 onward.

✓
▶️
Install Code RunnerOne-click run for any file. Beginner-friendly shortcut.
▼

Same extensions panel; search Code Runner, install by Jun Han. A ▶️ button appears at the top-right of every editor window. Click it → your .cpp file compiles and runs in the terminal panel.

Code Runner uses default flags, which may not include -std=c++20 — we'll fix that explicitly in item #7.

✓
💾
Turn on Auto SaveNever lose work to a crash.
▼

Open File → Auto Save and tick it. Done. The file now saves automatically.

✓
🎨
Turn on Format on SaveTidy code, every save, automatically.
▼

Open File → Preferences → Settings (or Ctrl+,). Search format on save. Tick the checkbox.

The C/C++ extension ships with the LLVM clang-format style by default, which is industry-standard. You can switch styles (Google, Mozilla, Microsoft) later by setting C_Cpp.clang_format_style, but the default is great.

✓
🚦
Install Error LensInline error text right beside the code.
▼

Search Extensions for Error Lens, install by Alexander. Now errors and warnings appear inline at the end of the line, in colored text. C++ error messages can be hundreds of characters long, but Error Lens shows the first line right where the problem is, which is enough 90% of the time.

✓
📁
Install Material Icon ThemeDistinct icons for .cpp, .h, etc.
▼

Search Extensions for Material Icon Theme, install it. Then run Command Palette (Ctrl+Shift+P / Cmd+Shift+P) → type Icon Theme → pick Material Icon Theme. Source files, headers, and CMake files now all have distinct visual identities.

✓
⚡
Set the C++ standard to C++20The single most important setting in this lesson.
▼

Without this, your tools will probably default to C++14 or C++17, and some lessons in the book won't compile. Two places to set it:

1. For IntelliSense (so the editor understands modern C++):
Open Command Palette → C/C++: Edit Configurations (UI). Find C++ standard and set it to c++20. Save.

2. For Code Runner (so your runs use C++20):
Open Settings, search code-runner.executorMap, click Edit in settings.json. Find the "cpp" entry and add -std=c++20 -Wall to the g++ command. Example:

"cpp": "cd $dir && g++ -std=c++20 -Wall $fileName -o $fileNameWithoutExt && $dir$fileNameWithoutExt"

(-Wall turns on warnings — extra hints from the compiler about questionable code. We'll see them in action in lesson 1.5.)

Why this matters: features like structured bindings, std::optional, ranges, concepts — half this book — require C++17 or C++20. Without the flag, your compiler reports them as errors. With the flag, they just work.

7 Common mistakes

Three setup traps 🚧

🙅
"VS Code and Visual Studio — same thing, right?" Two different products. VS Code (blue chevron icon) is a lightweight editor. Visual Studio (colored infinity icon) is a massive IDE. They share a brand and that's it. For this book, we use VS Code.
🙅
"My editor still flags modern C++ features as errors after I set the standard." You probably set it in only one of the two places. The C/C++ extension's IntelliSense and Code Runner have separate standard settings. Both need to say C++20 (item #7). Otherwise the editor shows red squiggles while your runs actually work — or vice versa. Set both.
🙅
"IntelliSense isn't suggesting std::cout." Two common causes: (1) you haven't #include <iostream> at the top of the file — IntelliSense needs the include to know about std::cout. (2) the C/C++ extension can't find your g++ — open Command Palette → C/C++: Edit Configurations (UI) and set the Compiler path to your g++ from lesson 1.1.
8 Your turn

Configure your editor 💪

Easy Run the checklist. Click through all 7 items and actually install/configure each. Screenshot the Extensions panel showing them installed. (You need this for lesson 1.3 where we write our first program.)
Medium Open a folder, not a file. Create an empty folder called cpp-practice. In VS Code: File → Open Folder → pick it. Always open the folder, not individual files — VS Code is much smarter when it sees the project root. From here on we'll always work this way.
Stretch Try clangd. Many serious C++ developers prefer clangd over Microsoft's C/C++ extension because it's faster on large projects. Search Extensions for clangd (by LLVM), install it, and follow its prompts to set up compile_commands.json. (You may want to disable Microsoft's IntelliSense to avoid duplication.) Compare the two; pick the one that feels right.
