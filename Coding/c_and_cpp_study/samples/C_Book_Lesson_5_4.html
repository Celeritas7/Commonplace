C Strings

C has no built-in string type. A "string" is just a char array with a special marker — a '\0' (null) byte — sitting after the last letter. That tiny zero is how every string function knows where your text ends.

1 Why this matters

Text is just characters in a row 🧠

Other languages give you a string type. C doesn't — and once you know how it represents text, a lot of "weird" string behaviour suddenly makes sense. A C string is a char array holding the letters, followed by one extra byte: '\0', the null terminator. It's the value zero, and it means "the string ends here."

Because the array itself doesn't store a length (lesson 5.2), C needs some way to know where a string stops — and the '\0' is that marker (it's the sentinel idea from 5.2 in action). Functions like printf("%s", …) and strlen simply walk the characters until they hit the '\0'. Get that one idea and C strings stop being mysterious.

2 The null terminator

"cat" is four characters 🔚

Write a string literal and C automatically appends the '\0' for you. So the three-letter word "cat" actually occupies four char slots:

char word[] = "cat";
// in memory:  'c'  'a'  't'  '\0'
// index:       0    1    2    3
The '\0' is a real character with the value 0 — written '\0' (or just 0). It is not the digit '0' (which is the value 48). Every proper C string carries this invisible terminator, which is why a string of N visible letters needs an array of at least N + 1 bytes. Forgetting that +1 is the number-one C string bug.
3 Declaring strings

Three ways to make one ✍️

char a[] = "hi";          // size 3: 'h' 'i' '\0' (C counts for you)
char b[10] = "hi";        // size 10: 'h' 'i' '\0' then 7 spare '\0's
char c[3] = {'h', 'i', '\0'}; // the long way, by hand
char a[] = "hi" lets C size the array — it makes it exactly big enough for the text plus the terminator (here, 3). char b[10] = "hi" reserves extra room (useful if the string will grow), and the unused tail is filled with '\0'. All three forms produce a proper, null-terminated string. Note that double quotes make a string (with a '\0'); single quotes make one char.
4 Using a string

Print it, index it, measure it 📏

Print a whole string with %s, reach a single character with an index, and find its length with strlen — which counts characters up to (but not including) the '\0':

#include <string.h>   // for strlen
char word[] = "cat";

printf("%s\n", word);          // cat
printf("%c\n", word[0]);        // c   — one character
printf("%lu\n", strlen(word));    // 3   — letters before '\0'
printf("%lu\n", sizeof(word));    // 4   — array bytes, INCLUDING '\0'
Notice the difference: strlen gives the text length (3), while sizeof on the array gives the storage size (4, because it counts the '\0'). To loop over the characters yourself, walk until you hit the terminator: for (int i = 0; word[i] != '\0'; i++). That condition — "stop at the null" — is the manual version of what strlen does.
5 The trap

No terminator, no end 🚧

If a char array isn't null-terminated, string functions don't know where to stop — they keep reading past your data into whatever memory follows, printing garbage or crashing. The '\0' is not optional; it is the string.

🚨 An array too small for the terminator

char s[3] = "cat"; is a bug: "cat" needs 4 bytes ('c' 'a' 't' '\0') but the array holds only 3, so the '\0' doesn't fit. Now s isn't a valid string — printf("%s", s) runs off the end. Always size a string array for the text plus one. The same risk appears when you build strings by hand or copy them (next lesson covers the safe ways).

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

➕
Forgetting the +1 for the terminator A string of N letters needs an array of at least N + 1 bytes. char s[3] = "cat" has no room for the '\0' — size it char s[4] (or just char s[]).
0️⃣
Confusing '\0' with '0' '\0' is the null terminator, value 0. '0' is the digit zero, value 48. They look similar but mean completely different things — only '\0' ends a string.
📏
Mixing up strlen and sizeof strlen(s) is the text length (no terminator); sizeof(s) on a char array is the storage size (with terminator). For "cat": strlen = 3, sizeof = 4.

Practice exercises 💪

Easy Length by hand. Without using strlen, count the characters in char s[] = "hello" by looping until you reach '\0'. Print the count and compare with strlen.
Medium Print each character. Loop over a string one char at a time (stop at '\0') and print each on its own line with its index. Notice the loop never prints the terminator.
Stretch Uppercase it. Walk a string and, for each lowercase letter, convert it to uppercase by subtracting 32 from its char value ('a' − 32 = 'A'). Stop at '\0' and print the result. (You'll see why the terminator matters — overrun it and you'd "uppercase" random memory.)

🎓 What to remember

  • A C string is a char array ending in '\0' — the null terminator (value 0).
  • A literal adds the '\0' for you: "cat" is 4 bytes ('c' 'a' 't' '\0').
  • Size for text + 1: N letters need at least N + 1 bytes.
  • Functions stop at the '\0': %s and strlen read until they hit it.
  • strlen = text length (no terminator); sizeof on the array = storage (with terminator).
