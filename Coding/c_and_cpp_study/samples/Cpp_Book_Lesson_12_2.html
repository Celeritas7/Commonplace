Capstone: A JSON-like Parser

For the final build, we'll parse a JSON-like format — objects, arrays, strings, numbers, true/false/null — into an in-memory tree. It shows off two big ideas at once: a std::variant value that can hold any of several types, and recursive descent, where the parser calls itself to handle nesting of any depth.

1 What we're building

From text to a tree 🌲

JSON is everywhere — config files, web APIs, save data. A parser reads that flat text and turns it into a structured tree your program can walk. We'll handle the core of JSON: objects { "key": value }, arrays [ value, … ], strings, numbers, and the literals true, false, null. The build rests on two ideas you've met: a std::variant (Part IX–ish modern C++) to make one value type that can be any of those kinds, and recursive descent — a set of small functions that call one another (and themselves) to match the grammar. It's the perfect finale: a real, useful program that leans on the type system and recursion together.

Paste some JSON into the interactive below and watch it become a tree; then walk through the five pieces of the C++ that does it.

2 The value type

One type, many shapes — std::variant 🎭

A JSON value can be a string, a number, a bool, null, an array, or an object. A std::variant lets a single type hold exactly one of those at a time:

#include <variant>
#include <string>
#include <vector>
#include <map>

struct JsonValue;                          // forward declaration (recursive)
using Array  = std::vector<JsonValue>;
using Object = std::map<std::string, JsonValue>;

struct JsonValue {
    std::variant<std::nullptr_t, bool, double,
                 std::string, Array, Object> data;   // exactly one kind
};
std::variant is a type-safe union: a JsonValue stores one of the six listed types, and the variant remembers which. The clever part is that Array and Object are themselves built from JsonValue — so the type is recursive, exactly mirroring how JSON nests values inside values. (The struct JsonValue; forward declaration lets us name the type before it's fully defined, which the recursion needs.) Later you read a value with std::get<T>(v.data), or check the kind first with std::holds_alternative<T>(v.data).
3 The parser skeleton

Walk the string with an index 🚶

The parser holds the input and a current position, plus tiny helpers. The heart is value(), which looks at the next character and dispatches:

class Parser {
    const std::string& s_;
    size_t i_ = 0;                       // current position
public:
    Parser(const std::string& s) : s_(s) {}
    JsonValue parse() { JsonValue v = value(); skipWs(); return v; }
private:
    void skipWs() { while (i_ < s_.size() && std::isspace(s_[i_])) ++i_; }
    char peek() { skipWs(); return i_ < s_.size() ? s_[i_] : '\0'; }

    JsonValue value();                   // dispatch on peek()
    std::string parseString();
    Array  parseArray();
    Object parseObject();
};
This is the shape of a recursive-descent parser: one object that scans the input left to right via the index i_, never backing up. skipWs jumps over spaces and newlines; peek reports the next meaningful character without consuming it. Each grammar rule — a value, a string, an array, an object — becomes its own function. Keeping the position in a member means every function shares one cursor as they hand control back and forth.
4 Recursion is the trick

Containers call value() again 🔁

value() picks a rule from the first character. The container rules then call value() right back — that mutual recursion is what handles nesting:

JsonValue Parser::value() {
    char c = peek();
    if (c == '{') return { parseObject() };
    if (c == '[') return { parseArray() };
    if (c == '"') return { parseString() };
    if (c == 't' || c == 'f') return { parseBool() };
    if (c == 'n') { i_ += 4; return { nullptr }; }   // null
    return { parseNumber() };
}

Array Parser::parseArray() {
    ++i_;                                // consume '['
    Array arr;
    if (peek() == ']') { ++i_; return arr; }
    while (true) {
        arr.push_back(value());          // <-- recursion!
        if (peek() == ',') { ++i_; continue; }
        if (peek() == ']') { ++i_; break; }
        throw std::runtime_error("expected , or ]");
    }
    return arr;
}
Here's the key insight of the whole capstone: parseArray calls value() for each element, and value() might call parseArray (or parseObject) again — so an array of objects of arrays just works, to any depth, with no special handling. The grammar is recursive, so the code is recursive. parseObject is the same idea: read a string key, expect a :, call value() for the value, repeat on , until }. The scalar parsers (parseString, parseNumber, parseBool) just scan characters — they're the base cases that stop the recursion.
5 Using it

Parse, then read the tree 🌳

A small free function kicks it off, and you read results out of the variant with std::get:

JsonValue parse(const std::string& text) {
    return Parser(text).parse();
}

int main() {
    JsonValue v = parse(R"({"name":"Ada","tags":["math","code"]})");

    Object& o = std::get<Object>(v.data);          // it's an object
    std::cout << std::get<std::string>(o["name"].data);   // Ada

    Array& tags = std::get<Array>(o["tags"].data);
    std::cout << tags.size();                       // 2
}

🚨 Check the kind before you std::get the wrong type

std::get<T>(v.data) throws std::bad_variant_access if the variant doesn't currently hold a T — so reading std::get<double> from a value that's actually a string is a runtime error (the exception machinery from Part X catches it). When you're not certain of the kind, test first with std::holds_alternative<T>(v.data), or use std::visit to handle every alternative in one place. The other thing to remember: this is a teaching parser. A production JSON library also handles string escapes (\n, \", \uXXXX), exponent numbers, deep-nesting limits, and precise error positions. The shape here is real; the edge cases are where the work is.

8 Mistakes, extensions, recap

Wrap up 📚

Three mistakes to avoid 🚨

🎭
std::get on the wrong alternative std::get<T> throws std::bad_variant_access if the variant holds a different type. Check with std::holds_alternative<T> first, or handle all cases with std::visit.
🚶
Forgetting to consume a character Each parse function must advance i_ past what it matched (the [, the ,, the closing "). Forget one and the parser loops forever or misreads — step the index deliberately.
🕳️
No error on malformed input A parser that silently accepts garbage hides bugs. Throw (or return an error) on an unexpected character, a missing : or ], or trailing junk after the value — and report where.

Extend it 💪

Easy Pretty-print it. Write a function that takes a JsonValue and prints it back out as indented JSON — a recursive walk that mirrors the parser (use std::visit or check each alternative).
Medium String escapes. Extend parseString to handle \", \\, \n, and \t inside strings, turning the escape sequences into the real characters.
Stretch Good errors. Make parse failures report the line and column (not just the byte index), and write a handful of malformed inputs to confirm each gives a clear, located message.

🎓 What you built

  • A recursive-descent JSON parser turning text into an in-memory tree of values.
  • std::variant gives one JsonValue type that holds any of six kinds; Array/Object reference it, making the type recursive.
  • The parser walks the string with an index; value() dispatches on the next character to a per-rule function.
  • Recursion is the engine: container rules call value(), which calls them back — so nesting of any depth just works.
  • Read results with std::get/std::holds_alternative; check the kind first, and report errors on bad input.
