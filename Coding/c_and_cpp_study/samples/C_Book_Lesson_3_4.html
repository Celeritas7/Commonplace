for Loops

The for loop is C's counting workhorse. It takes the three parts every loop needs — initialize, condition, update — and gathers them into one compact header, so the whole loop's shape is visible at a glance.

1 Why this matters

The loop you'll write most 🧠

Last lesson you met while, where the three loop ingredients — initialize, condition, update — were scattered around. The for loop is the same machinery, but it puts all three in one place: the header. When you're counting a known number of times or walking through an array by index, for is the loop you'll reach for nine times out of ten.

The reason it's so popular: everything that controls the loop sits on the first line, where you can see it at a glance. No hunting through the body to find where the counter changes — it's right there in the header. That makes for loops easy to read, easy to write, and harder to get wrong (you can't forget the update when it has its own slot). Let's break down the header.

2 Anatomy of a for loop

Three slots, one line 🎯

for (i = 0; i < 5; i++) {
    printf("%d ", i);
}
// prints: 0 1 2 3 4

The header has three slots, separated by semicolons — for (init; condition; update):

SlotExampleWhen it runs
Initializei = 0Once, at the very start
Conditioni < 5Before each pass — true keeps going
Updatei++After each pass through the body
This for is exactly equivalent to a while loop — same three pieces, just collected into the header instead of spread out: 
int i = 0;          // init
while (i < 5) {       // condition
    printf("%d ", i);
    i++;             // update
}
Anything you can write as a for, you can write as a while, and vice versa. Use for when you're counting.
3 Execution order

What runs when 🔀

The order the three slots fire in is the thing beginners get wrong, so make it concrete. For for (i = 0; i < 5; i++):

  • init runs once — i = 0.
  • Then the cycle repeats: check the condition → if true, run the body → run the update → back to the condition.
  • The moment the condition is false, the loop stops — without running the body that final time.
Two things people get backwards: the update runs after the body, not before. And the condition is checked before the very first body run — so if it's false to begin with, the body runs zero times, just like a while. The for-Stepper below lights up each slot as it fires so you can watch the cycle.
4 Common for patterns

The shapes you'll reuse 🧰

PatternHeaderRuns
Count up (0-based)for (i = 0; i < n; i++)n times: 0…n-1
Count up (1-based)for (i = 1; i <= n; i++)n times: 1…n
Count downfor (i = n; i > 0; i--)n times: n…1
Step by 2for (i = 0; i < n; i += 2)0, 2, 4, …
Walk an arrayfor (i = 0; i < len; i++)each index 0…len-1
The zero-based count-up — for (i = 0; i < n; i++) — is the single most common loop in all of C, because arrays are indexed from 0 to len - 1. Burn that pattern into muscle memory: start at 0, condition is < len (not <=), step by 1. You'll write it thousands of times.
5 Gotchas and odd corners

Where for trips you up ⚠️

🚨 The off-by-one

Starting at 0, i < n runs n times; i <= n runs n + 1. With an array of length len, i <= len reads one slot past the end — a real bug that can crash or corrupt data. For 0-based array loops, the condition is almost always i < len.

🚨 The stray semicolon

for (i = 0; i < 5; i++); — that semicolon makes the body empty. The code you meant to repeat sits below the loop and runs once. It compiles cleanly, so the compiler won't save you.

A couple of legal oddities you'll see in real code: all three slots are optional. for (;;) is an intentional infinite loop (no condition means "always true"). And you can update more than one variable with a comma: for (i = 0, j = 9; i < j; i++, j--). You don't need these often, but now they won't surprise you.

7 Predict the output

Quick check 🎯

What does this print? 
int i;
for (i = 0; i < 3; i++) {
    printf("%d", i);
}
printf("|%d", i);
Hint: the loop prints 0,1,2. But what value does i hold after the loop — the one that made the condition false?
8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

📏
Off-by-one in the condition For a 0-based array of length len, use i < len, not i <= len. The <= version reads one element past the end — undefined behavior.
❗
A semicolon right after for (...) for (i = 0; i < 5; i++); gives the loop an empty body. Whatever you meant to repeat runs only once, below the loop. Watch for that stray ;.
🔀
Misremembering the execution order It's init → (condition → body → update), repeating. The update fires after the body, and the condition is rechecked before every body run, including the first.

Practice exercises 💪

Easy Times table. Use a for loop to print the 7× table from 7 x 1 = 7 to 7 x 10 = 70, one line each. Pick the condition so it stops exactly at 10.
Medium Countdown and sum. Use a counting-down for loop to print 10 9 8 … 1, and at the same time accumulate their sum. Print the total (should be 55).
Stretch FizzBuzz. For i from 1 to 20, print "Fizz" if i is divisible by 3, "Buzz" if by 5, "FizzBuzz" if by both, otherwise the number. Combine a for loop with the if / else chains and the % operator from earlier lessons.

🎓 What to remember

  • for (init; condition; update) gathers all three loop parts into one header — ideal for counting.
  • Execution order: init once, then condition → body → update, repeating. The update fires after the body.
  • A for is equivalent to a while — pick for when you're counting a known number of times.
  • The 0-based pattern for (i = 0; i < n; i++) is the most common loop in C — memorize it.
  • Watch off-by-one (< vs <=) and the stray semicolon that empties the body.
