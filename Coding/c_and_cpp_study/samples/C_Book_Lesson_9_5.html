Command-Line Arguments

When you type gcc -o hello hello.c, how does gcc know the filename and the options? Through command-line arguments. Give main two parameters — argc and argv — and the words a user typed after your program's name land right in your hands. This is how every real command-line tool reads its input.

1 Why this matters

Input before main runs ⌨️

So far your programs got their input from scanf or files. But command-line tools take input as you launch them: cp source.txt backup.txt, grep error log.txt, gcc -O2 main.c. Those words after the program name are command-line arguments, and C hands them to you through two parameters on main. This is what makes a program scriptable and composable — it can be told what to do without stopping to ask. Combined with the streams and files from this Part, argc/argv are the last piece for building a genuine Unix-style tool.

It's a tiny bit of syntax that unlocks every program you've ever run from a terminal.

2 main's two parameters

argc and argv 🔢

Declare main with two parameters and the system fills them in for you:

int main(int argc, char *argv[]) {   // argc = count, argv = the strings
    printf("got %d arguments\n", argc);
    return 0;
}
  • argc (argument count) — how many words were on the command line, including the program name.
  • argv (argument vector) — an array of strings (char *), one per word.
char *argv[] means "an array of char *" — an array of strings, since a C string is a char * (you saw this back in the strings and pointers parts). The system splits the command line into words and hands you the count in argc and the words themselves in argv. You don't allocate or free any of it — it's set up before main starts and cleaned up after.
3 The layout

argv[0] is the program name 0️⃣

The very first entry, argv[0], is the program's own name — the real arguments start at argv[1]. After the last argument, argv[argc] is NULL:

// run as:   ./greet Ada 3
// argc == 3
argv[0]  // "./greet"   ← the program name
argv[1]  // "Ada"       ← first real argument
argv[2]  // "3"         ← second real argument
argv[3]  // NULL        ← marks the end
So if a user passes N real arguments, argc is N + 1 (the extra one is the program name at index 0). That off-by-one is the thing to keep straight: a tool expecting one filename should check argc == 2, not argc == 1. The trailing NULL at argv[argc] is a convenience — you can loop until you hit it, though looping with argc is more common.
4 Looping over them

Visit each argument 🔁

To process the real arguments, loop from 1 (skipping the program name) up to argc:

for (int i = 1; i < argc; i++) {   // start at 1, stop before argc
    printf("arg %d: %s\n", i, argv[i]);
}
Start at i = 1 to skip argv[0] (the program name) and stop at i < argc so you never read past the last argument. Each argv[i] is a plain C string, so everything you know about strings applies: compare with strcmp (e.g. if (strcmp(argv[i], "-v") == 0) to detect a flag), measure with strlen, and so on. This little loop is the skeleton of every option parser.
5 Arguments are strings

"42" is text, not a number 🔤

Every argument arrives as a string, even if it looks like a number. To do math with it, convert first — atoi for a quick int, or strtol for proper error checking:

// run as:   ./repeat 3
int n = atoi(argv[1]);   // "3" (text) → 3 (int)
for (int i = 0; i < n; i++) puts("hi");

🚨 Reading argv[i] without checking argc first

If your program uses argv[1] but the user ran it with no arguments, argv[1] is the NULL terminator — and using it (e.g. atoi(NULL) or printf("%s", argv[1])) is undefined behaviour, often a crash. Always check argc before reading an argument: if (argc < 2) { fprintf(stderr, "usage: %s <n>\n", argv[0]); return 1; }. And remember the values are text: comparing argv[1] == "3" compares pointers, not contents — use strcmp, and convert with atoi/strtol before doing arithmetic.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

🧨
Not checking argc before reading argv[i] If the user gave too few arguments, argv[i] may be NULL — using it is undefined behaviour. Check argc and print a usage message first.
🔢
Forgetting argv[0] is the program name The first real argument is argv[1], and argc counts the program name too. A tool needing one argument wants argc == 2.
🔤
Treating an argument as a number argv[i] is text. Use atoi/strtol to convert, and strcmp (not ==) to compare — == compares pointers, not the characters.

Practice exercises 💪

Easy Echo the args. Print argc, then loop from 1 to argc printing each argv[i] on its own line. Run it with different words and confirm the count and order.
Medium Sum the numbers. Treat every argument as a number with atoi and print their total — ./sum 2 3 10 → 15. First check argc and print a usage line if none were given.
Stretch A tiny cat. Take a filename from argv[1], open it with "r" (Part IX!), and print it line by line — handling both "no filename given" (check argc) and "file won't open" (check fopen). Add a -n flag that numbers the lines.

🎓 What to remember

  • int main(int argc, char *argv[]) receives the command line: argc is the count, argv the strings.
  • argv[0] is the program name; real arguments start at argv[1]; argv[argc] is NULL.
  • Loop for (int i = 1; i < argc; i++) to visit each argument as a string.
  • Arguments are text — convert with atoi/strtol, compare with strcmp, never ==.
  • Check argc first — reading a missing argument is undefined behaviour.
