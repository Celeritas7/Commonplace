Nested Loops

A loop inside another loop. The outer loop takes one step, and the inner loop runs all the way through — then the outer takes its next step, and the inner runs again. It's how you sweep a grid, build a table, or draw a pattern.

1 Why this matters

When one loop isn't enough 🧠

A single loop walks a line of things: items 0 through n. But lots of data is two-dimensional — a grid of pixels, a spreadsheet of rows and columns, a chessboard, a multiplication table. To touch every cell, you need a loop for the rows and a loop for the columns. Put one inside the other and you've got a nested loop.

The idea is simple once you see the rhythm: the outer loop is the slow hand of a clock, the inner loop is the fast hand. For each single tick of the outer loop, the inner loop spins all the way around. It's the same loop machinery you already know — just one loop living inside another's body. The Grid Sweeper later in this lesson makes the rhythm visible.

2 The mental model

Outer slow, inner fast 🕐

for (i = 0; i < 3; i++) {        // outer: rows
    for (j = 0; j < 4; j++) {    // inner: columns
        printf("(%d,%d) ", i, j);
    }
    printf("\n");              // after each full row
}
(0,0) (0,1) (0,2) (0,3)
(1,0) (1,1) (1,2) (1,3)
(2,0) (2,1) (2,2) (2,3)
Trace the order: i starts at 0, then the inner loop runs j = 0, 1, 2, 3 — a whole row. Only then does i become 1 and the inner loop start over from j = 0. The inner loop completes fully on every single pass of the outer loop. That one sentence is the whole concept — say it out loud and the rest follows.
3 Counting the work

Outer × inner 🧮

How many times does the inner body run in total? If the outer loop runs N times and the inner runs M times each, the body runs N × M times altogether. A 3-row, 4-column grid runs the inner body 3 × 4 = 12 times.

That multiplication is worth respecting. Two loops of 1,000 each isn't 2,000 steps — it's a million. Nesting multiplies the work, so a deeply nested loop over big data can get slow fast. It's a perfectly normal tool; just keep the multiplication in mind when the numbers are large. (You'll meet this idea again as "big-O" much later — for now, just remember: nesting multiplies.)
4 A worked example

The multiplication table ✖️

Here's a classic that puts it together — a 1-to-5 times table. The outer loop picks the row number, the inner loop multiplies it by each column:

for (i = 1; i <= 5; i++) {
    for (j = 1; j <= 5; j++) {
        printf("%4d", i * j);   // width 4, lines up neatly
    }
    printf("\n");                  // new line after each row
}
   1   2   3   4   5
   2   4   6   8  10
   3   6   9  12  15
   4   8  12  16  20
   5  10  15  20  25
Notice where the printf("\n") sits: after the inner loop, inside the outer loop. That's what ends each row and starts the next. A super-common beginner slip is putting the newline in the wrong place — inside the inner loop (one number per line) or outside the outer loop (no row breaks at all). The newline that breaks rows belongs between the two loops.
5 break, continue, and variables

What jumps reach, what names collide 🎯

  • break and continue hit only the innermost loop. A break in the inner loop ends just that inner pass — the outer loop carries on to its next step. (From lesson 3.5: to escape both, restructure or use a flag.)
  • Use different loop variables. The outer and inner counters must be different names — i and j by convention. Reusing i for both would make the inner loop clobber the outer's progress.
  • The inner loop restarts every time. Its counter is re-initialized (j = 0) at the top of each outer pass — that fresh start is exactly why the inner loop sweeps the full range again and again.

🚨 Reusing the same variable for both loops

If you write for (i = 0; i < 3; i++) and then for (i = 0; i < 4; i++) inside it — both using i — the inner loop runs i up to 4, which also ends the outer loop early or makes it behave unpredictably. Always give nested loops their own counters: i outside, j inside.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

🔤
Reusing one variable for both loops The outer and inner loops need separate counters (i and j). Sharing one makes the inner loop trample the outer's progress.
↩️
Putting the row break in the wrong place The printf("\n") that ends each row goes after the inner loop but inside the outer. Inside the inner loop you get one item per line; outside the outer you get no line breaks at all.
🐢
Forgetting that nesting multiplies Two loops of N each run the inner body N² times. That's fine for small N, but it grows fast. Be aware when N is large or you nest three or more levels deep.

Practice exercises 💪

Easy Rectangle of stars. Use nested loops to print a 4-row, 6-column rectangle of *. The outer loop counts rows, the inner prints the stars in a row, and a newline ends each row.
Medium Right triangle. Print a triangle where row i has i stars (row 1 has one star, row 5 has five). Hint: the inner loop's condition depends on the outer counter — for (j = 1; j <= i; j++).
Stretch Find a pair. Given an array of integers, use nested loops to check whether any two different elements add up to a target value. Print the first pair you find, then stop both loops (remember: break only exits the inner one — use a flag or a helper to leave both).

🎓 What to remember

  • A nested loop is a loop inside another loop. The inner loop runs fully on every single pass of the outer.
  • The body runs N × M times — nesting multiplies the work, so watch out when N is large.
  • Use separate counters (i outside, j inside); the inner counter resets each outer pass.
  • Row breaks go between the loops: the printf("\n") sits after the inner loop, inside the outer.
  • break/continue affect only the innermost loop — escaping both needs a flag or a helper function.
