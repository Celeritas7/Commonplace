🎯 PART VI · POINTERS

Pointers: & and *

You know what an address is. A pointer is simply a variable that holds one. Two operators do all the work: & gets an address to store, and * follows a stored address to the value it points at. Master these two symbols and you've mastered the mechanics of pointers.

1 Why this matters

A box that holds a box number 🧠

Last lesson you learned that every variable has an address. A pointer is just a variable whose value is one of those addresses — it doesn't hold a number like 30, it holds "the location where 30 lives." With a pointer you can reach a variable indirectly: follow the address and you arrive at the data. That indirection is the engine behind pass-by-reference, dynamic memory, and every data structure in C.

It comes down to two operators you'll use constantly: & to get an address (you met it in 6.1), and * to follow one. Let's wire them together.

2 Declaring a pointer

int *p; — "pointer to int" 📦

You declare a pointer by putting a * between the type and the name. The type says what it points to:

int  x = 30;
int *p;        // p is a "pointer to int" — it can hold the address of an int
Read int *p; as "p is a pointer to an int." It's a variable like any other, but the kind of value it stores is an address — and specifically the address of an int. Right now p doesn't point anywhere useful yet; we have to aim it.
3 Aiming it with &

p = &x; makes p point at x 🎯

To make p point at x, store x's address in it — that's exactly what & gives you:

int  x = 30;
int *p = &x;    // p now holds the address of x → "p points to x"

printf("%p\n", (void *)p);   // the address (same as &x)
Now p and &x are the same value — both are "where x lives." We say p points to x. You can declare and aim in one line (int *p = &x;) or separately (int *p; p = &x;). Notice the * only appears in the declaration here — assigning the address doesn't use it.
4 Following it with *

*p reaches the value 🔗

Once p points at x, the dereference operator *p means "the thing p points to." You can read it or write through it:

int  x = 30;
int *p = &x;

printf("%d\n", *p);  // 30  — read the value at p's address
*p = 99;            // write through p…
printf("%d\n", x);   // 99  — …and x changed!
This is the magic: *p = 99 didn't touch p — it followed p to x and changed x. Because p points at x, reading or writing *p is just reading or writing x by another route. That's how a function can change a caller's variable: hand it a pointer, and it writes through *p.
5 The two faces of *

Declare vs dereference ✨

The * symbol does two different jobs depending on where it appears — this trips up every beginner, so name it clearly:

WhereWhat * meansExample
in a declaration"this is a pointer"int *p;
in an expression"follow the pointer"*p = 99;
And keep p and *p straight: p is the address (where it points), while *p is the value (what's there). Likewise & and * are opposites — &x goes from a variable to its address; *p goes from an address back to the variable.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

✨
Mixing up the two uses of * In int *p; the * declares a pointer. In *p = 5; it dereferences — follows the pointer. Same symbol, two completely different jobs.
💥
Dereferencing before aiming int *p; *p = 5; writes to a random, garbage address — a crash waiting to happen (a "wild" pointer). Always point p at something real first, or set it to NULL.
🔀
Confusing p with *p p is the address it holds; *p is the value at that address. p = 99; would point p at address 99; *p = 99; stores 99 in the target.

Practice exercises 💪

Easy Four prints. Declare int x = 7; and int *p = &x;. Print x, &x, p, and *p. Confirm that p equals &x, and *p equals x.
Medium Change it from afar. With p pointing at x, do *p = 100; and then print x. Watch x become 100 — you changed it without naming it directly.
Stretch Two pointers, one variable. Make int *p = &x; and int *q = &x; — both aimed at the same x. Change *p, then read *q. They see the same value, because they point at the same box (they alias it).

🎓 What to remember

  • A pointer is a variable that stores an address: int *p; is a "pointer to int."
  • &x gives an address; p = &x; aims p at x ("p points to x").
  • *p dereferences — reads or writes the value at the address p holds.
  • The * in int *p declares; the * in *p follows — same symbol, two roles.
  • p is the address, *p is the value — and always aim a pointer before you dereference it.
