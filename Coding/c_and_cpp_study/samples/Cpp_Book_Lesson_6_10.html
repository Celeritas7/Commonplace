Operator Overloading

Built-in types use +, ==, and << — and so can your own. Operator overloading lets you define what these operators mean for your classes, so a + b, a == b, and cout << a just work. Used well, it makes your types read like the language's own.

1 Why this matters

Your types, the language's syntax 🧠

You write 3 + 4 and x == y without thinking — those operators are built into the language for numbers. But for a Vector2 or a Money type you designed, the compiler has no idea what + or == should do. Operator overloading lets you teach it: you define a function for the operator, and from then on a + b on your type calls your code. It's how std::string supports s1 + s2 and cout << s — and you can do the same for your classes.

This is mostly about readability: a + b is far clearer than a.plus(b). We'll cover the three you'll reach for most — +, ==, and <<.

2 Arithmetic: operator+

Define + for your type 🧮

To make a + b work, write a member function named operator+. It takes the right-hand operand and returns a new object — it doesn't change either input:

struct Vector2 {
    int x, y;
    Vector2 operator+(const Vector2& rhs) const {
        return { x + rhs.x, y + rhs.y };   // a new Vector2
    }
};

Vector2 a{1, 2}, b{3, 4};
Vector2 c = a + b;   // c == {4, 6} — calls a.operator+(b)
a + b is shorthand for a.operator+(b): a is the object, b is the parameter rhs. Take the parameter by const& (no copy) and mark the function const (it only reads). Returning a brand-new object is what lets + behave like normal addition — the operands are unchanged.
3 Comparison: operator==

Make a == b meaningful ⚖️

By default the compiler doesn't know when two of your objects are "equal." Define operator== returning bool and you decide:

struct Vector2 {
    int x, y;
    bool operator==(const Vector2& rhs) const {
        return x == rhs.x && y == rhs.y;
    }
};

Vector2 a{1, 2}, b{1, 2};
if (a == b) std::cout << "equal";   // equal
Now a == b compares the way you defined — here, both components must match. (In C++20 you can often write bool operator==(const Vector2&) const = default; and the compiler generates it, even giving you != for free. Defining it by hand, as above, works in every version.)
4 Output: operator<<

Print with cout << a 🖨️

To make std::cout << a work, you overload operator<<. This one is special: its left operand is the stream, not your object, so it must be a non-member function — and a friend (6.8) if it needs to read private fields:

struct Vector2 {
    int x, y;
    friend std::ostream& operator<<(std::ostream& os, const Vector2& v) {
        return os << "(" << v.x << ", " << v.y << ")";
    }
};

std::cout << a;   // (1, 2)
It takes the stream by reference and the object by const&, then returns the stream — that's what lets you chain cout << a << b << "\n". This is the exact pattern you glimpsed in the friend lesson; now you can see why it's a non-member: the stream has to be on the left.
5 Use it well

🚨 Don't surprise your users

The golden rule of operator overloading: make operators do what people expect. + should add, == should compare, << should print. An operator+ that secretly mutates its operands, or an operator== that compares only half the fields, makes code that lies about itself. Overload an operator only when its meaning is obvious; otherwise, a plainly-named method (a.merge(b)) is clearer. And remember you can only overload existing operators — you can't invent new symbols like **.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

🎭
Making an operator do something surprising Keep meanings intuitive: + adds, == compares, << prints. An operator that mutates its operands or compares half the fields produces code that misleads.
🖨️
Trying to make operator<< a member Its left operand is the stream, not your object, so it can't be a member. Write it as a non-member (a friend if it reads private fields), taking (std::ostream&, const T&) and returning the stream.
📋
Forgetting const / by-reference Read-only operators should be const members taking const& parameters — that avoids copies and lets them work on const objects and through const&.

Practice exercises 💪

Easy Add two vectors. Give struct Vector2 { int x, y; } an operator+ that adds componentwise. Add two vectors and print the result's x and y.
Medium Compare them. Add operator== comparing both components. Test it on two equal vectors and two unequal ones, printing the boolean result.
Stretch Print directly. Add a friend operator<< so std::cout << v prints (x, y). Then print a + b in one line — std::cout << (a + b) — combining two of your overloads.

🎓 What to remember

  • Operator overloading defines what operators mean for your own types — operator+, operator==, operator<<.
  • Arithmetic/comparison operators are usually const members taking a const& and returning a new value (or bool).
  • operator<< is a non-member (often a friend) taking (ostream&, const T&) and returning the stream.
  • Keep overloaded operators intuitive — + adds, == compares; don't surprise your users.
  • You can only overload existing operators — you can't invent new symbols.
