enum — Named Constants

Code full of bare numbers — state == 2, if (color == 1) — is a puzzle to read. An enum gives a set of related integer values real names: RED, GREEN, BLUE instead of 0, 1, 2. Same numbers underneath, but now your code says what it means.

1 Why this matters

Names beat magic numbers 🧠

Imagine reading if (s == 2) ... — what's 2? You'd have to hunt for a comment. Now imagine if (s == PAUSED) — instantly clear. An enum defines a group of named integer constants, so you write the name and the compiler fills in the number. It makes code self-explanatory, keeps related values together in one place, and pairs perfectly with switch for handling each case. It's the third of your custom-type tools, after struct and typedef.

A tiny feature that quietly makes a whole program easier to read.

2 Declaring an enum

Auto-numbered from zero 🏷️

List the names inside braces. By default the first is 0 and each one counts up by one:

enum Color { RED, GREEN, BLUE };
//            0      1       2     ← assigned automatically

enum Color c = GREEN;     // c holds 1
printf("%d\n", c);       // prints 1
printf("%d\n", BLUE);    // prints 2
Each name inside the braces is an enumerator — a named integer constant. RED is just a readable spelling of 0, GREEN of 1, and so on. You can use them anywhere a number fits: assign them, compare them, print them. The type's full name is enum Color (you'll shorten that with typedef shortly).
3 Choosing the values

Set your own numbers 🎯

You can give any enumerator an explicit value. Ones after it keep counting up from there:

enum Status { OK = 200, REDIRECT = 300, NOT_FOUND = 404 };

enum Signal { LOW, HIGH, ERR = 10, FATAL };
//            0     1    10        11   ← FATAL resumes from ERR+1
Explicit values are handy when the numbers mean something (HTTP codes, hardware registers) or when you want a clean mapping. The key rule to remember: assigning one enumerator doesn't reset the counter — the next unassigned name is simply the previous value plus one. So in Signal, FATAL becomes 11, not 2.
4 They're just ints

Perfect partners with switch 🔁

An enum value is an ordinary integer, so it works in comparisons, arithmetic, and especially switch — the natural way to handle each case by name:

enum Color c = GREEN;
switch (c) {
    case RED:   puts("stop");  break;
    case GREEN: puts("go");    break;   // ← runs this
    case BLUE:  puts("cool");  break;
}
Reading case GREEN: instead of case 1: is the whole win — the switch documents itself. Because enumerators are real compile-time constants, they're also valid as array sizes and in other places that need a constant. Under the hood it's all just int, but on the page it reads like meaning.
5 typedef + enum

Drop the enum keyword 🧱

Just like with structs, a typedef lets you skip writing enum every time:

typedef enum { MON, TUE, WED, THU, FRI } Day;

Day today = WED;       // no "enum" needed — today holds 2

🚨 An enum variable isn't range-checked

In C, an enum is just a dressed-up int — there's no type safety. Nothing stops you writing Day d = 99; even though no enumerator has that value; it compiles fine. Enums improve readability, not correctness. (They also share the normal identifier namespace, so two enums can't both define a name like RED — the names would collide.)

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

🔢
Expecting the counter to reset After an explicit value, the next unassigned enumerator is the previous plus one — not back to 0 or to the original sequence. { LOW, HIGH, ERR=10, FATAL } gives FATAL = 11.
⚠️
Assuming type safety An enum is just an int in C. Day d = 99; compiles even though it's not a valid day. Use enums for clarity; validate values yourself if it matters.
💥
Name collisions Enumerators live in the ordinary namespace, so two enums can't both define RED or OK. Prefix them (COLOR_RED, HTTP_OK) when clashes are likely.

Practice exercises 💪

Easy Days of the week. Write enum Day { MON, TUE, WED, THU, FRI, SAT, SUN }; and print the value of SAT. Confirm it's 5.
Medium A tiny state machine. Define enum State { IDLE, RUNNING, PAUSED, STOPPED }; and write a switch that prints a message for each state. Drive it with a variable and watch the right case fire.
Stretch Explicit codes. Make enum Http { OK = 200, MOVED = 301, NOT_FOUND = 404, ERROR = 500 };. Print each, then add one unassigned enumerator right after OK and predict its value before running.

🎓 What to remember

  • An enum names a set of integer constants — RED, GREEN, BLUE instead of 0, 1, 2.
  • Auto-numbering starts at 0 and counts up; an explicit value sets the count, and the next name resumes from there + 1.
  • Enumerators are plain ints — great in switch, comparisons, and as constants.
  • typedef enum { … } Name; lets you drop the enum keyword.
  • No type safety in C — any int fits an enum variable; the value is clarity, not correctness.
