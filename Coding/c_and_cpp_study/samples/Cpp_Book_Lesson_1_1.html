🛠️ PART I · SETTING UP

Installing a Modern C++ Compiler

One tool, three OS paths. By the end of this lesson, you'll have a modern C++ compiler installed and verified — ready for everything from hello.cpp to smart pointers in Part VIII.

1 Why this matters

The tool that makes C++ real 🛠️

Your .cpp source code is just text. It does nothing on its own. The tool that turns it into an executable your CPU can actually run is the compiler. Without one installed, you can't follow along with anything past this lesson.

Three compilers dominate the C++ world:

  • g++ — the GNU C++ compiler, part of GCC. Open source, ships with most Linux distros, the default in MSYS2 on Windows.
  • clang++ — the LLVM compiler. Also open source. Apple's default on macOS (often hiding behind the g++ name there).
  • MSVC (cl.exe) — Microsoft Visual C++. Excellent on Windows; less portable.

For this book we'll use g++ (or clang++ on Mac — they're interchangeable for our needs). It's free, widely supported, and we can write commands that work the same on Windows, Mac, and Linux. We'll mention MSVC briefly but won't depend on it.

One non-negotiable: we need a compiler that understands C++17 or newer. Older compilers won't accept some of the modern syntax we'll use from Part V onward. The instructions below all install something current enough.

2 What you need

The two pieces of your setup 🧩

Various C++ IDEs and compilers are available.

True — and the choice has gotten simpler over the years. Today, the standard beginner-friendly setup is:

  • A text editor — we'll set up VS Code in the next lesson (1.2). For now, any text editor will do.
  • A C++ compiler — what we install today.
Older C++ tutorials (and your existing notes) often mention Code::Blocks, an all-in-one IDE+compiler from the early 2010s. It still works, but the modern stack — VS Code + g++/clang++ — is what professionals use, is better maintained, and what we'll line up with in this book.
3 Pick your operating system

Three paths, one destination 🎯

Click your operating system below. Each path is well-trodden by millions of developers. Each command has a copy button — paste it into your terminal exactly as shown.
1
Download MSYS2

Go to msys2.org and click the installer link. Run it; accept the defaults. (MSYS2 is the modern, maintained successor to MinGW — they're the same family.)

2
Install GCC (which includes g++)

From the Start menu, open MSYS2 MINGW64. Paste this:

$ pacman -S --needed mingw-w64-x86_64-gcc

Type Y when prompted. The gcc package brings g++ with it — they're siblings.

3
Add g++ to your Windows PATH

Open Settings → System → About → Advanced system settings → Environment Variables. Under User variables, edit Path, click New, paste:

C:\msys64\mingw64\bin

Click OK everywhere. Close any open terminals and open a fresh one.

4
Verify it works

Open a normal Windows command prompt (Win+R → cmd → Enter):

> g++ --version
g++.exe (Rev1, Built by MSYS2 project) 13.2.0
Copyright (C) 2023 Free Software Foundation, Inc.

Version 11 or later supports everything in this book. 🎉

1
Open Terminal

Press Cmd + Space, type Terminal, hit Enter.

2
Install Xcode Command Line Tools

This installs Apple's clang++ — fully C++20 capable. Paste:

$ xcode-select --install

Click Install in the dialog. Wait 5–10 minutes. When the terminal prompt returns, the install is done.

3
Verify it works
$ g++ --version
Apple clang version 15.0.0 (clang-1500.3.9.4)
Target: arm64-apple-darwin23.0.0

You'll see Apple clang — that's the system clang++ wearing a g++ hat. Perfectly fine. 🎉

(If you want "real" g++ alongside clang, run brew install gcc after installing Homebrew — but it's not required.)

1
Open a terminal

Press Ctrl+Alt+T or find Terminal in the application menu.

2
Install g++ via your package manager
# Ubuntu / Debian / Mint
$ sudo apt update && sudo apt install build-essential
# Fedora / RHEL
$ sudo dnf install gcc gcc-c++ make
# Arch / Manjaro
$ sudo pacman -S base-devel

The build-essential / base-devel packages bring both g++ and the rest of the GNU build tools.

3
Verify it works
$ g++ --version
g++ (Ubuntu 13.2.0-23ubuntu4) 13.2.0
Copyright (C) 2023 Free Software Foundation, Inc.

Look for version 11 or later for full C++20 support. 🎉

4 The "did it work?" check

One command, one answer ✅

The single command that tells you whether installation succeeded is g++ --version. Run it in any terminal. The output should look something like this:

✅ Expected output

g++ (Ubuntu 13.2.0-23ubuntu4) 13.2.0
Copyright (C) 2023 Free Software Foundation, Inc.

As long as you see "g++" (or "clang") and a version number, you have a working compiler. Version 11 or later covers C++20 features we'll touch on; 9 or later covers C++17. Older than that, you'll want to update.

If you see 'g++' is not recognized as an internal or external command on Windows, or command not found on Mac/Linux — the install didn't finish or PATH isn't picking it up. Most commonly, just closing and reopening the terminal fixes it.

5 Plan B — the online fallback

If install gives you trouble 🚧

Installation works for most readers, but not all — corporate laptops, locked-down environments, network blocks. You can still follow this whole book without a local install, using online C++ compilers.

Three solid options for C++:

  • godbolt.org (Compiler Explorer) — the favorite of professional C++ programmers. Pick any compiler (g++ 13, clang 18, MSVC, etc.), see the assembly output side-by-side. Free, no signup.
  • wandbox.org — clean, minimal interface. Supports dozens of C++ compiler versions. Great for "try a snippet" experiments.
  • onlinegdb.com — full compiler with debugger. Slower to start but a good all-in-one.

For Part I–IV, any of these works. From Part V onward (the STL deep dive), having a local install becomes more convenient — but it's still optional.

7 Common mistakes

What trips beginners here 🚧

🙅
"I added it to PATH but it still doesn't work." Almost always: you didn't close and reopen the terminal. Environment variables are read at terminal startup. Close every terminal window, open a new one, retry.
🙅
"My compiler is too old to do modern C++." Possible — if you installed g++ 7 or earlier, lambdas-with-init from C++14 and structured bindings from C++17 will fail. Run g++ --version and aim for version 11+ for full C++20 (or version 9+ for C++17). When in doubt, upgrade via your package manager.
🙅
"I need to enable the C++ standard with a flag." True, and we'll cover this in 1.2. The compiler picks an older default (often C++14 or C++17). For full modern code, you'll pass -std=c++20 to g++ when you compile. We'll wire it into your editor settings.
8 Your turn

Do the install 💪

Easy Install it. Follow the OS picker above. Run g++ --version. Screenshot the output — first proof of a working C++ dev environment.
Medium What standard? Run g++ -dM -E -x c++ /dev/null | grep cplusplus (Linux/Mac) or echo "" | g++ -dM -E -x c++ - | findstr cplusplus (Windows). The number it prints tells you which C++ standard your compiler defaults to. 201703L = C++17, 202002L = C++20.
Stretch The Compiler Explorer trick. Open godbolt.org, type a single line of C++: int x = 1 + 2; into the left panel. Set the compiler dropdown to g++ 13.2. Look at the assembly on the right. Notice what the compiler did with your three-operation source: it just stored the literal 3 — the addition happened at compile time. That's the kind of magic C++ does that you'll appreciate more and more.

🎓 Quick recap

  • You need a C++ compiler installed on your machine to turn .cpp files into runnable programs. We're using g++ (or clang++ on Mac — interchangeable).
  • Three install paths by OS: MSYS2 on Windows · Xcode Command Line Tools on macOS · build-essential (or equivalent) on Linux.
  • One verification command: g++ --version. If it prints a version number, you're good. Aim for version 11 or later for full C++20 support.
  • On Mac, g++ is an alias for Apple's clang++ — same C++ for our purposes.
  • Don't fight a broken install — godbolt.org and wandbox.org are perfectly good for the first several parts of this book.
