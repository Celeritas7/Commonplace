File Streams

Reading and writing files in C++ uses the same streams you already know. An ofstream works like cout but writes to a file; an ifstream works like cin but reads from one. Best of all, they're RAII objects — the file opens when the stream is created and closes itself when the stream goes out of scope.

1 Why this matters

Files, the C++ way 📂

You've used std::cout and std::cin since lesson one. File I/O reuses exactly the same machinery: a file stream is a stream object, like cout, that happens to be connected to a file instead of the console. So everything you know — << to write, >> to read, std::getline for whole lines — works unchanged. There are three types, all in <fstream>: std::ofstream (output — write to a file), std::ifstream (input — read from a file), and std::fstream (both). And because they follow RAII (Part VIII), you almost never call close() by hand: opening happens in the constructor, closing happens automatically in the destructor.

If you've seen C's fopen/fclose (the C book's Part IX), this is the same idea — but the stream cleans up after itself, so there's no handle to forget to close.

2 Writing a file

std::ofstream — like cout, to disk ✍️

Construct an ofstream with a filename and write to it with <<, exactly as you would to cout:

#include <fstream>

std::ofstream out("notes.txt");   // opens (creates/truncates) the file
if (!out) {                      // check it actually opened
    std::cerr << "cannot open file\n";
    return;
}
out << "Hello\n" << 42 << '\n';   // write just like cout
// no close() needed — `out`'s destructor closes it here
Opening an ofstream on a path creates the file if it doesn't exist and truncates it (empties it) if it does — so by default you start with a blank file. Always check the stream right after opening: if (!out) (or out.is_open()) tells you whether the open succeeded — it can fail for a bad path, a read-only directory, or permissions. After that, writing is identical to console output. When out goes out of scope its destructor flushes and closes the file for you; that's the RAII payoff.
3 Reading a file

std::ifstream — like cin, from disk 📖

An ifstream reads with >> (whitespace-separated tokens) or std::getline (whole lines):

std::ifstream in("notes.txt");
if (!in) { /* open failed — file missing? */ }

std::string word;
in >> word;            // reads one whitespace-separated token

std::string line;
std::getline(in, line); // reads the rest of the current line
>> skips leading whitespace and reads a single token (a "word", or a number into an int/double); std::getline(in, line) reads everything up to the next newline. Each extraction can fail — because the data didn't match (a letter where a number was expected) or because you hit end-of-file. When that happens the stream goes into a fail state, and crucially, the stream tests as false in a boolean context. That single fact is what makes the read loop on the next step work correctly.
4 The read loop

Loop on the read itself 🔁

The idiomatic way to read a whole file is to put the extraction in the loop condition. The loop ends the moment a read fails (including at EOF):

// word by word:
std::string w;
while (in >> w) { use(w); }          // stops when extraction fails / EOF

// line by line:
std::string line;
while (std::getline(in, line)) { use(line); }

🚨 Don't loop on !in.eof()

The classic beginner bug: while (!in.eof()) { std::getline(in, line); use(line); }. The problem is that eof() only becomes true after a read has already failed by hitting end-of-file — so the loop runs one extra time, using a line that wasn't actually read (a stale or empty value). Test the read, not eof(): while (std::getline(in, line)) or while (in >> w) checks whether the extraction succeeded, so it stops at exactly the right moment. Same idea you saw with cin — it's just a file now.

5 Open modes

Append, binary, and friends 🎛️

A second constructor argument sets the open mode — combine flags with |. The common ones:

FlagMeaning
std::ios::appappend — keep existing contents, write at the end
std::ios::trunctruncate — empty the file first (the ofstream default)
std::ios::binarybinary mode — no newline translation (for non-text data)
std::ios::in / outread / write (an fstream can use both at once)
 
// append to a log instead of overwriting it:
std::ofstream log("app.log", std::ios::app);
log << "started\n";   // added at the end; old lines kept
Each stream type has a sensible default mode, so you often don't pass one: ofstream defaults to write+truncate, ifstream to read. Reach for an explicit mode when you need to add to a file rather than replace it (std::ios::app), when you're handling non-text data (std::ios::binary — see the C book's binary-I/O lesson for why newline translation matters), or when you want a single fstream open for both reading and writing (std::ios::in | std::ios::out). For everyday text, the defaults are exactly what you want.

8 Mistakes, exercises, recap

Wrap up 📚

Three mistakes to avoid 🚨

🚪
Not checking if the file opened Opening can fail (bad path, permissions). Writing to or reading from a failed stream silently does nothing. Always test if (!in) / if (!out) right after construction.
🔁
Looping on eof() while (!in.eof()) runs one extra iteration with stale data, because eof() turns true only after a read fails. Loop on the extraction: while (in >> x) or while (getline(in, line)).
♻️
Truncating when you meant to append Opening an ofstream normally empties the file. To add to it instead, open with std::ios::app. Forgetting this silently wipes existing contents.

Practice exercises 💪

Easy Write then read. Open an ofstream on "hello.txt", write three lines, let it close. Then open an ifstream on the same file and print each line with while (std::getline(in, line)).
Medium Count words. Read a text file token by token with while (in >> word) and print the total count. Then add a check: if the file didn't open, print an error to std::cerr and exit.
Stretch Append a log. Write a function log(const std::string& msg) that opens "app.log" with std::ios::app, writes msg plus a newline, and lets the stream close. Call it several times and confirm earlier lines are kept, not overwritten.

🎓 What to remember

  • File streams reuse cout/cin: ofstream writes with <<, ifstream reads with >>/getline, fstream does both. All in <fstream>.
  • RAII: the file opens in the constructor and closes itself in the destructor — no manual close() needed.
  • Always check the open with if (!stream); opening an ofstream creates/truncates the file.
  • Loop on the read — while (in >> x) or while (getline(in, line)) — never on !eof().
  • Open modes (std::ios::app, ::binary, ::in|::out) tune behaviour; the per-type defaults suit everyday text.
