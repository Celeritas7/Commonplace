Your Very First Program — hello.c

Six lines. We'll write them, decode every part, compile them into a working program, and run it. By the end of this lesson you'll have made a computer print "Hello, world!" — and you'll know why.

1 Why this matters

The rite of passage 🚀

Every programmer's first C program is some variation of "Hello, world!" — a one-line greeting on the terminal. It looks trivial, but every single piece of it teaches you something about how C is structured. After this lesson, you'll have read, understood, and run real C code.

The phrase comes from Brian Kernighan and Dennis Ritchie's 1978 book The C Programming Language — yes, the same Dennis Ritchie who created C. They used "hello, world" as the very first example, and now every language teaches the same thing first. You're about to join a tradition that's older than the web.

2 The program

Six lines — meet your first C 💻

Below is the complete program. Don't worry about understanding every part yet — we'll click through every piece in the next section. Just read it once and notice the shape.
hello.c #include <stdio.h> int main() { printf("Hello, world!\n"); return 0; }

Six lines (or four, if you don't count the blank line and the lonely closing brace). That's the smallest legal C program that does something visible. You'll write this shape — #include at top, int main(), statements inside, return 0; at the end — for every single C program in this book. Get comfortable with it.

4 Compile and run it

From text file to running program ⚡

Save the program above as hello.c (the filename matters — must end in .c). Open a terminal in the folder where you saved it, then run these two commands: 
$ gcc hello.c -o hello

That's the compiler. It reads hello.c, produces a new file called hello (on Linux/Mac; on Windows it'll be hello.exe). If there are no errors, the command finishes silently — no news is good news.

$ ./hello

That runs the executable. (On Windows cmd, you can just type hello without the ./.) You should see:

✅ Expected output

Hello, world!

That's the rite of passage. You made a computer print "Hello, world!" — and you wrote, compiled, and ran C to do it.

6 Common mistakes

What trips beginners on their first program 🚧

🙅
Forgetting the semicolon at the end of a statement. Every C statement ends with ;. Drop one and you'll see a cryptic error like "expected ';' before something". The fix is usually on the line above the one the compiler points at — the compiler kept reading hoping to find one and gave up later.
🙅
"Smart quotes" instead of plain quotes. If you copy code from a webpage or Word document, you might end up with " and " (curly) instead of " and " (straight). The compiler will reject them. Always type quotes by hand inside your editor, or paste into a code-aware editor (like VS Code) that won't transform them.
🙅
Saving the file as hello.txt instead of hello.c. The .c extension matters. Some editors (especially Notepad on Windows) silently add a .txt on top. If gcc hello.c complains "no such file or directory", do a directory listing — your file is probably hello.c.txt. Rename it.
7 Your turn

Make this program your own 💪

Easy Personalize it. Change the string inside printf from "Hello, world!" to "Hello, Aniket!" (or your own name). Recompile and run. Take a screenshot of the output — it's the first program you wrote, even if the change is small.
Medium Print three lines. Use three separate printf calls (or one call with two \ns) to make the output look like: 
Hello, world!
This is line two.
And this is line three.
Run it. Now try removing all the \ns — what happens? (You'll see all three messages run together on one line. That's why \n exists.)
Stretch Break the program on purpose. Try four variations, one at a time, recompile, and read the error: (a) remove the semicolon after printf(...). (b) remove #include <stdio.h>. (c) change main to Main (capital M). (d) change return 0; to return "zero";. Don't fix them yet — just read each error message and try to guess what the compiler is complaining about. This is how you build error-reading skill, which is lesson 1.5's headline topic.

🎓 Quick recap

  • Every C program follows the same shape: #include what you need at the top, int main(), statements inside, return 0; at the end.
  • #include <stdio.h> gives you access to printf and friends — the standard input/output library.
  • main() is the function the operating system runs when you launch your program. It's the only entry point.
  • printf("...") prints text. \n inside the string means "go to a new line."
  • return 0; tells the operating system the program finished successfully. Non-zero means "something went wrong."
  • Two commands turn your source code into a running program: gcc hello.c -o hello compiles, ./hello runs.
