  • And how to correctly reference different worksheets and workbooks when a macro spans more than one.
  • This is where you start typing your own code from scratch.
  • Follow along with the S4..._Start macro-free file. Open the VBE (Alt+F11), Insert → Module (named e.g. Lessons Ranges), and type Sub ReferToCells — brackets and End Sub appear automatically.
  • Option Explicit may sit at the top by default; not important yet — leave it or remove it.
  • Two key terms: ActiveCell (the single active cell) vs Selection (everything currently selected).
  • Test properties live in the Immediate window (View → Immediate, or Ctrl+G) with a question mark.
  • Refer to cells either by A1 style — Range("A1") — or by row/column numbers — Cells(1, 1). The Cells style is easier once you start looping.
' query a property live in the Immediate window
?ActiveCell.Address      ' → $A$1

Sub ReferToCells()
    Range("A1").Value = "Hello"   ' A1 style
    Cells(2, 1).Value = "World"      ' row 2, column 1  → A2
End Sub

Range objects have many members — most you'll rarely need. These are the ones that earn their keep:

MemberWhat it does
.ValueDefault property — the underlying value in the cell.
.CellsReturns a cell/range within a range — great for jumping to the bottom or right of a range.
.OffsetReturns a new range offset by a number of rows & columns from the start.
.CountNumber of cells in a range.
.Column / .RowPosition — note .Column returns a number, not a letter.

.EntireColumn / .EntireRowe.g. Range("A1").EntireRow.Delete deletes row 1.

💡 Get the full list any time from the Microsoft help page for the Range object, or the Object Browser (F2).
  • Finding where a range ends is the tricky part — and it matters most when you later loop over data.
  • Pick whichever fits your dataset; sometimes several work and you go with the one you're comfortable with.
' 1 · from the top, jump down to the last filled cell
lastRow = Range("A4").End(xlDown).Row

' 2 · from the very bottom of the sheet, jump up (most robust)
lastRow = Cells(Rows.Count, 1).End(xlUp).Row

' 3 · count the rows in the contiguous region
lastRow = Range("A4").CurrentRegion.Rows.Count

' 4 · the last used cell on the sheet
lastRow = ActiveSheet.UsedRange.SpecialCells(xlCellTypeLastCell).Row
💡 SpecialCells won't necessarily find your last data row unless it's also the last used row on the sheet — keep that caveat in mind.
  • The Copy method takes the destination as an argument — no need for two lines, and no need to select anything.
  • For values-only, use PasteSpecial.
  • Resize changes a range's dimensions before you copy — perfect for copying the current region without the header row.
' one-line copy — destination is an argument
Range("A4:E10").Copy Destination:=Range("J4")

' copy the data region but skip the header row, then paste values
With Range("A4").CurrentRegion
    .Offset(1).Resize(.Rows.Count - 1).Copy
End With
Range("J4").PasteSpecial xlPasteValues
  • So far we didn't reference a sheet at all — VBA assumed the active sheet. You could write ActiveSheet.Range(...), but it's unnecessary if you always want the active sheet.
  • Careful: if you run the macro from the VBE, make sure you're on the correct sheet first. A button on a specific sheet avoids this.
  • Reference a specific sheet with Worksheets("Sheet1").
  • UsedRange is a property of the worksheet, not a Range — you must specify the parent. ?UsedRange.Address alone errors ("object required"); use ?ActiveSheet.UsedRange.Address.
Worksheets("Sheet1").Range("A1").Value = 10
?ActiveSheet.UsedRange.Address   ' parent required — UsedRange is a sheet property
  • Excel assumed the active workbook until now — you only need to reference workbooks when a macro works with more than one at a time (e.g. copying data from a file you just opened).
  • Reference the current file with ActiveWorkbook; a useful property is .Name.
  • .FullName returns the name and the full path.
  • Beyond typing tests with ?, you can use Debug.Print inside running code to write to the Immediate window.
?ActiveWorkbook.Name        ' → Sales.xlsm
Debug.Print ActiveWorkbook.FullName   ' → C:\Data\Sales.xlsm (while code runs)

Build a macro that saves a hard-coded backup — no formulas or links, just values. New name: HC_ followed by the workbook name.

  • Save the current workbook first.
  • Select all sheets, copy, and Paste Special → Values over all cells.
  • Save As (be careful — not Save) into the same directory.
  • Name it HC_ + the current name.
  • Bonus 1: add today's date as yymmdd → e.g. HC_yymmdd_S4_ReferencingRanges.
  • Bonus 2: save the copy without macros (a normal .xlsx).
💡 Instructions also live in the S4_ReferencingRanges_Start file's Activity tab. The solution (incl. both bonuses) is in the next video and the complete workbook's ProjectActivity module — it uses some VBA built-in functions covered in Section 7.
  • The macro converts all formulas to their values (Ctrl+C → Paste Special → Values) across the whole workbook, then saves under a new name.
  • Always lay out the steps first — whether recording or writing by hand:
  • 1 · Save the current workbook (before hard-coding anything).
  • 2 · Select all sheets (as if right-click → Select All Sheets).
  • 3 · Select all cells, Copy, then Paste Special → Values.
  • 4 · Save As (not Save — Save would wreck the live workbook), adding the HC_ prefix.
  • Bonus: append the date for a dated backup; save the copy as a macro-free .xlsx.
  • Reference cells by A1 style or by Cells (numbers). Use Cells when you need to loop through columns.
  • Reference many cells at once with a comma or a colon — keep the quotation marks around your start/end cells.
  • Find the last row with xlUp / xlDown, by counting CurrentRegion.Rows, or with SpecialCells (with its caveat).

C Test your knowledge

Q1.What is the default property of a Range?
.Value is the default (and most common) property — the underlying value held in the cell.
Q2.Which is the most robust way to find the last data row in column A?
Jumping from the very bottom of the sheet up (Cells(Rows.Count,1).End(xlUp).Row) survives blank cells inside the data, unlike End(xlDown) which stops at the first gap.
Q3.Range("B5").Column returns…
.Column returns a number (B = 2), not the column letter — a common gotcha.
Q4.Why can you copy in a single line with Range("A4:E10").Copy Destination:=Range("J4")?
The Copy method accepts Destination as an argument, so you don't need to select anything or write a separate paste line.
Q5.UsedRange is a property of…
UsedRange belongs to the worksheet. Calling it bare errors with "object required" — write ActiveSheet.UsedRange (or Worksheets("…").UsedRange).
Q6.In the backup activity, why must you use Save As rather than Save?
After pasting values over every formula, a plain Save would write that destroyed version back onto your original. Save As writes a new HC_ file and leaves the live workbook intact.
