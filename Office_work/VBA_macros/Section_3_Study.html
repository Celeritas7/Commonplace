The Object Model — Understanding the Dot

The make-or-break concept of VBA. Once objects, properties and methods click — and you can read the . in any line of code — you can build (or fix) almost anything, whether you recorded it, wrote it, or found it on a forum.

💡 Even people who "jumped in" should take a step back here — coming up with the correct syntax is the single most difficult part of writing VBA, and this section is what fixes it.
  • Writing VBA is basically manipulating objects on the screen. Every object has a code word so VBA knows what you mean.
  • These code words live in a reference library. Excel exposes several: the Excel library, the Office library, and the VBA library.
  • The macro recorder mostly records from the Excel library. For logic and "smart" behaviour you reach into the VBA library — the brain of the macro.
  • To control other applications (Word, PowerPoint, Outlook) you can activate a reference to their object library via Tools → References. Not strictly required, but it gives you Intellisense and makes the code far easier to write.
  • Browse any library in the Object Browser (F2 in the VBE). Items are grouped into classes — "compartments" of related code.
  • A VBA module is made of Sub procedures. Anything the recorder produces sits inside a Sub, and most code you write will too.
  • There are also Function procedures — your way of writing your own formulas and returning values to Subs (covered later).
  • Colour cues: blue = keywords; VBA also auto-capitalises recognised references as confirmation you typed them correctly; red = something's wrong; green = comments (start the line with a single quote ').
  • Whitespace is free — use it to organise and indent for readability.
  • Line-continuation: a space + underscore _ tells VBA "this line and the next belong together" — purely for readability.
' green comment — starts with a single quote
Range("A1").Value = _
    "split across two lines for readability"
  • VBA is object-oriented: before any action you must say which object you're acting on. Telling a colleague "please close" is meaningless — door? window? PC? You must name the object.
  • The VBA word order is object first, then action — like light.on rather than "on the light".
  • Objects sit in a hierarchy (home → district → city). In Excel: Application → Workbook → Worksheet → Range, each level a collection of the next.
  • The dot separates the levels. If you don't name the parent, VBA assumes the active one.
' full hierarchy, spelled out
Application.Workbooks("Book1").Worksheets("Sheet1").Range("A1").Value = 10

' parent omitted — acts on the ACTIVE workbook & sheet
Range("A1").Value = 10
  • A property is what an object has — its characteristics.
  • Just like real objects have predefined properties, so do Excel objects.
  • Properties come after the object hierarchy, separated by dots — and a property can itself return an object with its own properties (e.g. Car.Interior.Color).
Car · colour · size · type · engine · age
Shoe · colour · size · gender · material · season
Range("A1").Value = "Total"
Range("A1").Font.Bold = True
Range("A2").Interior.Color = vbYellow   ' Interior is a property that returns an object
  • A method is what an object does — an action. A car can start, stop, crash; a shoe you put on or take off.
  • Methods can take arguments (extra information) — e.g. start the car quickly or slowly.
  • Syntax: Object.Method, then a space before the first argument, arguments separated by commas (like a worksheet formula). Or use named arguments with := for clarity.
' positional arguments: space, then comma-separated
Workbooks.Open "C:\Data\Sales.xlsx", , True

' named arguments with :=  (clearer, order-independent)
Workbooks.Open Filename:="C:\Data\Sales.xlsx", ReadOnly:=True
  • Macro recorder — the easiest way to get syntax you can then clean up.
  • Object Browser (F2) — dig into any library's objects, properties and methods.
  • F1 Help — opens Microsoft's page for the active keyword, usually with examples.
  • Intellisense — let VBA suggest valid members; force the dropdown with Ctrl + Space.
  • Immediate window — test code or query a property on the spot.
  • The internet & forums — a great starting point you then customise.
💡 Even experienced coders forget exact properties/methods — Leila uses a combination of these every day. Knowing how to look it up beats memorising.
  • Refer to an object through its position in the hierarchy; the dot is the separator.
  • If you don't specify the parent, VBA assumes the active object.
  • You don't need to select objects to manipulate them (exceptions apply).
  • Objects have specific properties and methods — some shared between objects, some unique.
  • A property can return another object: Range("A2").Interior.Color — Interior is a property that returns an Interior object, whose Color is a property. (Think of a shoe's heel: a property of the shoe, and an object with its own properties.)
  • Macros/VBA live in Sub procedures (functions in Function procedures). To find correct syntax: recorder → Object Browser → MSDN/F1 → Intellisense → Immediate window.

C Quiz 2 — Test your knowledge of the Object Model

Q1.In VBA, what is the word order for acting on something?
VBA is object-oriented: you name the object first, then the action — Object.Method. Leila's analogy is light.on, not "on the light".
Q2.What's the difference between a property and a method?
A property is a characteristic the object has (colour, size, value); a method is an action the object does (open, select, clear).
Q3.If you write Range("A1").Value = 5 without naming a workbook or sheet, VBA acts on…
When the parent is omitted, VBA assumes the active object — here the active workbook and active worksheet.
Q4.Which key opens the Object Browser in the VBE?
F2 opens the Object Browser. F5 runs code, Alt+F8 is the Macros dialog, and Ctrl+R is the Project Explorer.
Q5.How do you force the Intellisense dropdown to appear?
Ctrl + Space forces the Intellisense list (or completes the word). Ctrl+G opens the Immediate window; F1 opens Help.
Q6.In Range("A2").Interior.Color, what is Interior?
A property can return another object. Interior is a property of the range that returns an Interior object, whose Color is in turn a property — like a shoe's heel being both a property and an object.
