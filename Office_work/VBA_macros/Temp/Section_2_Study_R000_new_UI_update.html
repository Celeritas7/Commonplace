Section 2 · Your First Macro

Recording & Running Macros

The hands-on heart of the course begins: record a macro, read the VBA it generates, learn the Visual Basic Editor, and discover the seven ways to run your code — plus the all-important difference between a macro and VBA.

⏱ ~45 min 👩‍🏫 Leila Gharani 🎯 9 lessons · practice checklist · quiz

▶ Try it now — the Visual Basic Editor

Recording generates VBA for you — but reading and running it is where the learning happens. Pick a macro (or edit the code), then press F5 to Run or F8 to Step one line at a time and watch it act on the worksheet. MsgBox pops a real dialog; Debug.Print writes to the Immediate window — exactly like the real editor.

A Study the concepts

Click any card to expand it — opening one marks it read and bumps your progress at the top.

  • A macro is a stored set of instructions that automates an Excel task — anything you do by hand can usually be recorded or coded.
  • Every macro is written in VBA (Visual Basic for Applications) and stored inside a module.
  • The Visual Basic Editor (VBE) is the separate window where you view, write and edit that code. Open it with Alt + F11 or Developer → Visual Basic.
  • Recording a macro writes the VBA for you automatically — one of the best ways to discover Excel's object model and correct syntax.
💡 Record first to see the code, then edit it to understand it. That loop is how recorded actions turn into real VBA fluency.
  • Developer → Record Macro (or View → Macros → Record Macro).
  • Give it a name — no spaces, and it can't start with a number. Optionally set a shortcut key, choose where to store it (usually This Workbook), and add a description.
  • Perform your actions, then click Stop Recording.
  • Open the VBE (Alt + F11) and look in Module1 to read the generated Sub.
  • Run it via Developer → Macros → Run (or Alt + F8), or with the shortcut you assigned.
Sub MyFirstMacro()
    ' Everything you did while recording lands here
    Range("A1").Value = "Hello"
    Range("A1").Font.Bold = True
End Sub
  • Project Explorer (Ctrl + R) — the tree of open workbooks, their sheets, ThisWorkbook, modules and userforms.
  • Properties window (F4) — properties of the selected object.
  • Code window — where your Subs live; double-click a module to open it.
  • Immediate window (Ctrl + G) — run a line on the spot and see Debug.Print output.
  • Add a fresh standard module with Insert → Module.
  • You may see Option Explicit at the top — it forces you to declare variables; covered fully later.
  • Recording a fixed selection like Range("A1:A20") is fragile — it breaks the moment your data grows or shrinks.
  • SpecialCells selects cells by type instead of address: xlCellTypeBlanks, xlCellTypeConstants, xlCellTypeFormulas, xlCellTypeLastCell.
  • CurrentRegion grabs the whole contiguous block around a cell — perfect for "select all my data".
  • Together they make one macro work on a table of any size.
Sub FillBlanks()
    ' Select every empty cell in the data block, then fill it
    Range("A1").CurrentRegion.SpecialCells(xlCellTypeBlanks).Value = "n/a"
End Sub
  • Absolute recording (the default) captures exact addresses — a macro recorded at A1 always acts on A1, wherever you are when you run it.
  • Relative recording — toggle Use Relative References on the Developer tab — records movements relative to the active cell, so the macro runs wherever you currently are.
  • Choose based on intent: fixed report cells → absolute; "do this starting from wherever I'm standing" → relative.
  • This is the key idea behind copy/consolidate macros that paste data into the next free spot.
💡 The Relative toggle's state is remembered while recording — check it before you hit Record, not after.
  • Macros dialog — Developer → Macros or Alt + F8, pick and Run.
  • Keyboard shortcut assigned when recording (or via the dialog's Options).
  • Form Control button placed on the worksheet.
  • Shape or icon with a macro assigned (right-click → Assign Macro) — great for modern-looking buttons.
  • Quick Access Toolbar button.
  • Custom Ribbon tab/group.
  • From the VBE with F5 / Run — plus event triggers like opening the workbook.
  • In Microsoft 365, the classic cell "Comments" were renamed "Notes" — and "Comments" now means the new threaded comments.
  • In VBA the old .Comment object and methods like .ClearComments still correspond to what the UI now calls Notes.
  • Keep this in mind so your add/delete-comment macros target the right thing on newer Excel.
  • Record (or write) a macro that strips every comment/note from the sheet.
  • The recorded result is usually a one-liner — clean and reusable.
  • Try it, then read the code to confirm you understand each piece.
Sub DeleteAllComments()
    ' Clears notes/comments from every cell on the active sheet
    Cells.ClearComments
End Sub
  • Macro = the stored procedure — the automation itself (often one you recorded).
  • VBA = the programming language that macro is written in.
  • A recorded macro is simply VBA you didn't have to type. Writing VBA yourself unlocks logic, loops and decisions that recording can't capture.
  • Recording teaches you syntax; editing and writing VBA is where real automation power lives — the rest of the course builds exactly that.

C Test your knowledge

Q1.Which shortcut opens the Visual Basic Editor?
Alt + F11 opens the VBE. Alt + F8 opens the Macros dialog, and Ctrl + G jumps to the Immediate window.
Q2.What does turning on Use Relative References change while recording?
Relative recording captures moves relative to wherever you start, so the macro runs at the active cell instead of always at the originally recorded address.
Q3.Which macro name is valid?
Macro names can't contain spaces or start with a number (and avoid hyphens). CleanData is fine.
Q4.SpecialCells(xlCellTypeBlanks) selects…
It returns just the blank cells in the range — handy for filling or formatting only the gaps.
Q5.What is the difference between a macro and VBA?
A macro is the automation (often recorded); VBA is the programming language behind it. A recorded macro is simply VBA you didn't type.
Q6.In Microsoft 365, the classic cell "Comments" are now called…
The old Comments became Notes; "Comments" now refers to the new threaded comments. Your .Comment/ClearComments VBA targets what the UI calls Notes.
