  • Turn on the Developer tab: right-click the ribbon → Customize Ribbon → tick Developer → OK.
  • Close your personal/work files when using lecture files.
  • To run macros you must Enable Content when opening a file.
  • Check Developer → Macro Security. The recommended default is "Disable all macros with notification" — files open with macros off until you enable them.

D Test your knowledge

Q1.How do you make the Developer tab appear in Excel?
The Developer tab is off by default. Right-click the ribbon, choose Customize Ribbon, tick Developer, and click OK.
Q2.Which macro-security setting does Leila recommend (and use)?
"Disable all macros with notification" is the safe default — files open with macros disabled until you choose to Enable Content. "Enable all macros" is explicitly not recommended.
Q3.What does the _Start suffix on a project file mean?
Files ending in _Start are the macro-free versions — the starting point for each project so you can write the code yourself.
Q4.You finished a code-free file and added a macro. How must you save it?
Macros aren't saved in a standard .xlsx. Use File → Save As → Excel Macro-Enabled Workbook (*.xlsm).
Q5.What's the benefit of a Trusted Location folder?
Any workbook stored in a Trusted Location runs its macros automatically — no "Enable Content" needed each time. Add one via Macro Security → Trusted Locations.
Q6.According to Leila, who builds the most impressive automated Excel tools?
Her philosophy: use Excel functionality where it makes sense and VBA when you hit a limitation. Advanced Excel users who know VBA avoid over-programming.
