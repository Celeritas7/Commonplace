Looping through Collections & Making Decisions

The leap to efficient macros: instead of naming every sheet or cell, loop through the whole collection — and let your code make smart choices with If, Select Case and GoTo.

A Study the concepts

Click any card to expand it — opening one marks it read and bumps your progress at the top.

  • A collection is a group of similar objects that belong to the same parent — like stamps in a stamp album (each stamp an object, the album the collection).
  • Looping = "do this to each one." Rather than "stamp 1 → frame, stamp 2 → frame…", you say "move each stamp to the frame."
  • This section covers: With / End With for efficient code → For Each looping → If and Select Case for decisions → GoTo to change program flow.
  • With … End With lets you state the object once, then list its properties/methods with a leading dot — easier and faster to write and maintain.
  • Bonus: fewer dots → faster code. The fewer times VBA resolves the object reference, the quicker it runs.
  • Don't forget the leading dot before each property or method inside the block.
' before — the object is repeated every line
MyRange.Font.Name = "Arial"
MyRange.Font.Size = 12
MyRange.Font.Bold = True

' after — stated once with With
With MyRange.Font
    .Name = "Arial"
    .Size = 12
    .Bold = True
End With
  • For Each … Next runs code for every object in a collection — all worksheets, all cells in a range, every comment, anything that belongs to a collection.
  • You declare an object variable for one item and state which collection it belongs to (e.g. ThisWorkbook.Worksheets).
  • The loop keeps going as long as there are objects left in the collection — no need to know how many.
' protect every sheet, however many there are
Dim ws As Worksheet
For Each ws In ThisWorkbook.Worksheets
    ws.Protect
Next ws

' loop through each cell in a range
Dim c As Range
For Each c In Range("A1:A10")
    c.Value = c.Value * 2
Next c
  • The If…Then construct mirrors Excel's IF: depending on whether a condition is True or False, send the code one way or another.
  • You can nest as many conditions as you like (like nested IF formulas), and combine them with And / Or — exactly as in Excel.
  • Block If ends with End If; ElseIf adds extra branches.
' copy A1 across only if it isn't empty (leave prior value untouched)
If Range("A1").Value <> "" Then
    Range("C1").Value = Range("A1").Value
End If

If score >= 50 And attended = True Then
    result = "Pass"
ElseIf score >= 50 Or bonus = True Then
    result = "Review"
Else
    result = "Fail"
End If
  • When an If grows too big or complex, Select Case is the cleaner choice.
  • Start with Select Case + the thing you're evaluating (e.g. Range("B3").Value); close with End Select.
  • Example: read a number in B3 → write a status in C3 (1–200 = Good, 0 = empty, >200 = Excellent, <0 = Bad).
Select Case Range("B3").Value
    Case 1 To 200
        Range("C3").Value = "Good"
    Case 0
        Range("C3").Value = ""
    Case Is > 200
        Range("C3").Value = "Excellent"
    Case Is < 0
        Range("C3").Value = "Bad"
End Select
💡 If the workbook is protected from an earlier lesson, run your unprotect macro first — Select Case here writes to a cell.
  • GoTo jumps to a labelled section, skipping the lines in between.
  • Its main use is error handling: on an error, skip ahead to a clean-up section.
  • Syntax: GoTo Label, and the label is a name followed by a colon.
  • Often you'll want Exit Sub before the label, so normal runs don't fall into the error code.
  • Use sparingly — jumping around otherwise makes code confusing.
Sub Demo()
    On Error GoTo GetOut
    ' …risky code here…
    Exit Sub            ' stop before the label on a normal run
GetOut:
    MsgBox "Something went wrong"
End Sub
  • On the S6_Collections_With_IF_Start workbook's Activity sheet, write a macro to count how many cells in the used range contain formulas.
  • Put the answer in cell B6.
  • Detailed instructions are on the Activity tab; the finished macro is in the completed workbook S6_Collections_With_IF.
  • Loop through each cell of the used range (not the whole sheet), and when a cell has a formula, add to a running count.
  • Write the total to B6, then create a button and assign the macro.
  • (You could also solve this later with SpecialCells — but here we practise collections + If.)
Sub Count_If_Formula()
    Dim cell As Range
    Dim total As Long
    For Each cell In ActiveSheet.UsedRange
        If cell.HasFormula Then total = total + 1
    Next cell
    Range("B6").Value = total
End Sub
  • Hiding many sheets is easy; unhiding is painful — the Unhide dialog only lets you pick one sheet at a time.
  • A For Each loop over the workbook's sheets sets every sheet's Visible back on — instantly.
  • (Leila shows three variations, and how to run it even on plain Excel files with no macros.)
Sub Unhide_All()
    Dim ws As Worksheet
    For Each ws In ThisWorkbook.Worksheets
        ws.Visible = xlSheetVisible
    Next ws
End Sub
  • A collection is similar objects under one parent; For Each … Next acts on every member without counting them.
  • With … End With states an object once — cleaner code and fewer dots means faster execution.
  • Make decisions with If…Then / ElseIf / Else (combine conditions with And / Or), or switch to Select Case when there are many conditions.
  • GoTo jumps to a labelled section — reserve it mainly for error handling so code stays readable.

C Test your knowledge

Q1.What is a "collection" in VBA?
A collection is a group of similar objects under one parent — like Worksheets (sheets), or a range's Cells. For Each loops over them.
Q2.Besides less typing, why does With … End With help?
Stating the object once means VBA resolves fewer references (dots), which improves performance — a nice side effect of cleaner code.
Q3.Which statement runs code for every object in a collection?
For Each … Next iterates over every member of a collection (sheets, cells, comments…) without you knowing the count.
Q4.When is Select Case a better choice than If?
Select Case shines when you'd otherwise write a long, nested If…ElseIf chain — it's cleaner to read and maintain.
Q5.GoTo is mainly used for…
Its primary use is error handling (On Error GoTo Label). Overusing it for general flow makes code hard to follow.
Q6.To count formula cells, you loop the UsedRange and test which property of each cell?
cell.HasFormula returns True when the cell holds a formula — increment a counter each time it's True.
