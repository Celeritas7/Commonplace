Working with Variables

Give VBA a memory. Learn what variables are and why they save you time, pick data types that make code run faster, and master declaring, the Set statement, and scope — the last of the theory before you start building real tools.

⏱ ~55 min 👩‍🏫 Leila Gharani 🎯 8 lessons · checklist · quiz

A Study the concepts

Click any card to expand it — opening one marks it read and bumps your progress at the top.

  • Don't mix up assignment with equality. If A1 holds 4 and you run Range("A1").Value = 5, A1 becomes 5 — the = assigns the right-hand side to the left.
  • Variables are nicknames that let VBA memorise a value on the right-hand side so you can reuse it throughout your code.
  • You assign with =: e.g. myTitle = Range("A1").Value.
  • The payoff: if a value (like the title's cell) changes, you edit one assignment line instead of every place the value appears.
myTitle = Range("A1").Value   ' store it once…
Range("C1").Value = myTitle   ' …reuse it everywhere
Range("E1").Value = myTitle
  • VBA keeps things in memory and works through it — variables are how you use that memory.
  • By default a variable is a Variant (~16 bytes), which morphs to whatever you store (text, number, date…).
  • If you know a variable is, say, a small page number, declare it as Byte — one byte vs sixteen. The saving compounds on long, complex macros.
  • Every type has a range: smaller memory → smaller range. A Byte holds 0–255; try to store 300 and VBA errors (overflow).

  • Best practice: declare before you use. Declaring tells VBA to free up the right (usually smaller) amount of memory — code runs faster and is less error-prone.
  • Use the Dim keyword: Dim myText As String.
  • Naming rules: no spaces (use underscores), can't start with a number, and no . ! @ $ & etc. Keep names short and obvious.
  • Some follow a naming convention — prefix with the type's initials, e.g. strMyText, intCount. Optional; use what helps you.
Dim myText As String
Dim lastRow As Long
Const TaxRate As Double = 0.19   ' a constant — fixed value
  • Variables can hold objects too — workbook, worksheet, range.
  • Declaring one as a type (e.g. Dim NewSheet As Worksheet) restores Intellisense — which ActiveSheet alone doesn't give, because it could also be a chart sheet.
  • To assign an object you must use the Set keyword. NewSheet = ActiveSheet errors; Set NewSheet = ActiveSheet works.
  • Leila's memory trick: the moment your variable isn't a String/Long/Integer but a workbook/worksheet/range, it needs Set. (Ordinary assignments technically use the optional Let, which nobody types.)
Dim NewSheet As Worksheet
Set NewSheet = ActiveSheet      ' Set is required for objects
NewSheet.Range("A1").Value = "Hi"   ' now Intellisense works
  • Procedure scope (narrowest): Dim inside a Sub. The variable exists only while the procedure runs, then dies — what we've done so far.
  • Module scope: put Dim at the top of the code window (below Option Explicit, outside any procedure). Available to all Subs in that module, and its value stays in memory after a procedure ends.
  • Project scope (broadest): same idea but use Public instead of Dim, in any module. Available to all modules and procedures; value also persists.
Option Explicit
Dim moduleCount As Long     ' module scope — all Subs here
Public appTitle As String   ' project scope — every module

Sub Demo()
    Dim i As Long            ' procedure scope — dies with Demo
End Sub

Update the HC_ Save-As macro from Section 4, adding variables where they help. Leila's version (in the S5_Variables completed workbook, module ProjectActivity):

Sub Hardcoded_Workbook_copy()
    Dim FilePath As String
    Dim HCName As String

    FilePath = ThisWorkbook.Path & "\"
    HCName = "HC_" & VBA.Format(Date, "yymmdd") & "_" & ThisWorkbook.Name
    ThisWorkbook.Save
    Sheets.Select
    Cells.Copy
    Cells.PasteSpecial xlPasteValues

    Range("A1").Select
    Sheets(1).Select
    ThisWorkbook.SaveAs Filename:=FilePath & HCName
End Sub
💡 The two String variables hold the folder path and the new file name — built once, then reused in SaveAs. VBA.Format(Date, "yymmdd") stamps the backup with today's date.

C Quiz 3 — Test your knowledge

Q1.If A1 holds 4 and you run Range("A1").Value = 5, what does the = do?
The = is assignment, not equality — it puts the right-hand value into the left-hand target. A1 ends up as 5.
Q2.If you don't declare a variable's type, what does VBA use by default?
Undeclared variables default to Variant, which uses ~16 bytes and changes shape to fit whatever you store.
Q3.A variable declared As Byte can hold…
A Byte uses one byte and ranges 0–255. Storing 300 exceeds the range and triggers an error — pick Integer/Long instead.
Q4.To assign an object (worksheet, workbook, range) to a variable you must use…
Object variables require the Set keyword: Set sh = ActiveSheet. Forgetting Set is a classic error.
Q5.How do you make a variable available to every module and procedure?
Project (global) scope uses Public at the top of any module — available everywhere and its value persists.
Q6.What does turning on Option Explicit do?
Option Explicit requires every variable to be declared, catching typos and accidental new variables. Enable it via Tools → Options → "Require Variable Declaration".
